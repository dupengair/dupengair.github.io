    • 安全性：

    应用层安全(网络安全)：设备访问控制 会话管理 身份验证 用户授权 协议栈安全
    系统层安全(本地安全)：用户管理 权限管理 系统审计 异常检测 灾难恢复 入侵检测 系统漏洞检测 沙箱技术
    记录层安全(数据安全)：数据访问授权 数据保密性 数据完整性 数据可用性
    物理层安全(硬件安全)：网卡失效处理 分区损坏处理 启动失效处理 升级安全

    一、应用层安全(网络安全)

    1. 设备访问控制
      • IP过滤
        • 黑名单
        • 白名单
      • 终端访问
        • ssh取代telnet
        • 安全文件传输：scp取代tftp，tftp只在uboot下开放用于远程升级
      • URL过滤
        • CGI
        • SDK（rtsp…）
        • web service API
        • RESTful API
      • 开放端口过滤
        • 陌生端口甄别
        • 端口使用记录
    2. 会话管理
      • 连接数限制
      • 带宽限制
      • 异常活动检测
        • 端口嗅探
        • 僵尸链接
        • DoS攻击手段
    3. 身份验证
      • 基于证书保护
      • 证书格式
        • X509
      • 证书生成策略
        • 设备参数：IP、MAC..
        • 防止重放：时间
        • 唯一性：加盐
      • 密码复杂度约定，防止枚举攻击
    4. 用户授权
      • administrator权限集
      • operator权限集
      • guest权限集
      • user defined level
      • 特殊权限集：如智能，授权方式不同
    5. 协议栈安全
      • 应用层：
        • 网页访问默认https，提供http选项
        • http优先使用digest认证，而不是Base
        • rtsp支持digest和base
        • 使用明文传输的协议，对其中敏感消息内容使用PGP数字签名，如设备信息、设备配置内容等
        • 对文件传输，尽量使用加密处理，如升级文件、配置文件
        • 网页安全性(XSS攻击等)
      • 传输层：
        • 会话管理
          • 系统连接数
          • session分配与回收
        • 链接异常的处理
          • 资源释放(sessions、fd…)
          • 系统状态刷新
        • 定期检查DNS缓存，防止DNS中毒
      • 网络层：
        • 网络设置检查
          • 防止异常IP
          • IP冲突检测
        • 无dhcp服务处理：apipa
        • 定期检查和清理arp缓存，防止arp中毒
        • 防止广播风暴
      • 物理层：
        • 网卡失效
        • 断网检测
        • MAC失效

    二、系统层安全(本地安全)

    1. 用户管理
      • 用户配置与备份
      • 特殊用户
      • 影子密码
      • 默认密码策略
      • 密码遗失策略
    2. 权限管理
      • 设备功能集
      • 设备权限集
      • 权限级别定义
      • 特殊权限集的授权，如智能
    3. 系统审计(audit)
      • 系统资源监控（threads/processes、fd…）
      • 系统性能监控（cpu、memory、io、network thoughput…）
      • 用户活动监控（login/logout、sessions、key activities，…）
    4. 异常检测
      • 应用程序异常
      • 系统&内核异常
      • 网络异常
    5. 灾难恢复
      • 灾难分级
      • 备份手段
        • 自动备份
        • 事件触发备份(参数修改，系统状态变化)
      • 恢复手段
        • 进程内部处理
        • 进程外部重启
        • 重新加载模块
        • 重新加载驱动
        • 重启系统
        • 恢复出厂
      • 系统升级失败
        • 恢复原系统
        • 继续升级
    6. 入侵检测
      • 登陆用户扫描
      • 用户活动监控
      • 外部链接管理
      • 陌生端口扫描
    7. 系统漏洞检测
      • heartbleed
        • 检测openssl版本
      • shellshock
        • 通过脚本命令检测
    8. 沙箱技术sandbox
      • 修改busybox制作沙箱，应用程序引导完成进入sandbox
      • 用户登录后在sandbox环境下，受限运行
      • 系统应用程序在sandbox外运行，不受限

    三、记录层安全(数据安全)

    1. 数据访问授权
      • 设备配置导出导入
      • 设备配置访问
      • 获取日志
      • 录像下载
      • SQL查询
    2. 数据保密性
      • 对数据访问进行权限管理
      • 录像数据防拷贝
      • 防止SQL注入
    3. 数据完整性
      • 设备配置防篡改
      • 系统配置防篡改
    4. 数据可用性
      • 设备配置异常处理
        • 当前配置
        • 备份配置
        • 默认配置
      • 系统配置异常处理
        • 网络配置导致网络异常
        • 图像参数配置导致芯片异常
      • 重要文件异常处理
        • 启动脚本和系统脚本
        • 关键信息文件
          • 图像参数
          • 预置位参数
          • 机芯信息等
          • 硬件相关(如stm32、mcu)

    四、物理层安全(硬件安全)

    1. 网卡失效处理
    2. 分区损坏处理
    3. 启动失效处理
    4. 升级安全
