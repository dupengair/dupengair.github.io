网络性能工具篇(tcpdump)

基本用法

# tcpdump -i eth0 -nn icmp and host 10.82.16.74

# tcpdump -i eth0 -nn host 10.82.16.74 and host 10.82.16.73

# tcpdump -i eth0 -nn ‘host 10.82.16.74 and (port 23 or port 22)’

# tcpdump -i eth0 -nn ‘tcp[tcpflags] & (tcp-syn|tcp-fin) != 0 and not net 10.82.16.0/24’
// 不是16网段的tcp握手

# tcpdump -i eth0 ‘icmp[icmptype] != icmp-echo and icmp[icmptype] != icmp-reply’
// 不是ping包

tcp信息

  • 格式：

    src > dst: flags data-seqno ack window urgent options length
flags: S F P R W(ECN CWR) E(ECN-ECHO)

  • 捕获tcp握手过程的SYN

    # tcpdump -i eth0  tcp[13] = 2
    tcpdump: verbose output suppressed, use -v or -vv for full protocol decode
    listening on eth0, link-type EN10MB (Ethernet), capture size 65535 bytes
    18:39:57.408869 IP 10.82.16.71.1341 > 10.82.16.73.90: Flags [S], seq 1266894751, win 65535, options [mss 1460,nop,nop,sackOK], length 0

  • 捕获tcp握手过程的SYN 和 ACK+SYN (要用逻辑与&表示同时包含)

    # tcpdump -i eth0  'tcp[13] & 2 = 2'
    tcpdump: verbose output suppressed, use -v or -vv for full protocol decode
    listening on eth0, link-type EN10MB (Ethernet), capture size 65535 bytes
    18:45:08.146609 IP 10.82.16.71.1348 > 10.82.16.73.90: Flags [S], seq 1844972537, win 65535, options [mss 1460,nop,nop,sackOK], length 0
