二、uptime

# uptime 
 19:49:51 up  1:18,  1 user,  load average: 0.12, 0.12, 0.18
  1. 系统负载 —— 可运行态进程和不可中断态进程总数
  2. 包含当前时间、已运行时间、当前登陆用户数，系统过去1、5、15分钟的平均负载
  3. 可运行态：正在使用或等待cpu。 不可中断态：正在等待IO访问
  4. 平均负载为1意味着单个cpu总是有负载的，对于单cpu来说1或2是可接受的，这个数字等于cpu个数为最佳
  5. 可由uptime确定问题在自身还是在网络，如果cpu负载不高，问题可能和自身无关

三、ps

  1. 以当前有效用户ID运行

  2. 显示

    • 显示单个进程： ps -C FwLoader
    • 显示单个进程某列： ps -C FwLoader -o pid=
    • 显示线程信息：ps -L $(pidof FwLoader)

  3. 参数

    -s：显示信号格式
-u：显示面向用户格式
-v：显示虚拟内存格式
-e：所有进程
-f：全格式列表
-F：额外的全格式

  4. 状态

    D：不可中断睡眠
R：正在运行或可运行
S：可中断睡眠
T：已停止
Z：已死亡，还未被父进程回收
BSD格式：
<：高优先级，其它用户不能nice
N：低优先级，其它用户可以nice
L：内存中锁定分页，用于实时或定制IO
s：session leader
l：多线程
+：前台进程组

四、free

# free -lm
             total       used       free     shared    buffers     cached
Mem:           587        176        410          0          0        111
Low:           587        176        410
High:            0          0          0
-/+ buffers/cache:         65        522
Swap:            0          0          0
  1. shared表示/proc/meminfo中Shmem字段（实测有差别）

  2. 含义

    Mem: used = 已使用 + buffer + cache
     free = 空闲
-/+ buffers/cache: used = 已使用
                   free = 空闲 + buffer + cache
  3. 64bit系统不再使用high内存
  4. 使用/proc/buddyinfo确定每个区域可用的内存块。列代表可用的分页数量，第一列为0，每列的可用分页数量为当前数字乘以2的列数次幂，再乘以分页大小。如第0列的数字为4，分页数为：4 * 2^0 * 4KB = 16KB

  5. buffer与cache

    • buffer: buffer cache，块设备写缓冲区

    • cache : page cache，文件系统读缓冲区，大说明cache住的文件多，如果频繁访问的文件被cache，磁盘IO效率低下

      A buffer is something that has yet to be “written” to disk.

      A cache is something that has been “read” from the disk and stored for later use.

  3. 输出项

    CPU：cpu编号
%usr：用户态下cpu使用率
%nice：使用nice优先级执行的cpu使用率
%sys：内核态下cpu使用率，不含硬软中断的时间
%iowait：等待磁盘IO造成的cpu空闲率
%irq：硬中断的cpu时间
%soft：软中断的cpu时间
%steal：hypervisor为虚拟cpu服务所占用的cpu
%guest：cpu运行虚处理器所占用的cpu
%gnice：cpu运行nice的虚拟机占用的cpu
%idle：cpu空闲率
