Linux内核初探 之 内存管理(六) —— per cpu data

一、竞争与抢占

  1. 不用担心竞争问题，因为数据对当前处理器来说唯一，因此不会被别的处理并发访问，所以多处理器间可以不用锁而安全访问
  2. 需要小心内核抢占
    • 当处理器抢占时，数据指向错误的cpu因而失效
    • 因为抢占，造成了同一处理器并发访问的竞争条件

二、变量定义与声明

  1. 定义与声明

    DECLARE_PER_CPU(type, name)    
DECLARE_PER_CPU_SECTION(type, name, "")

    • 内核实现：

      /*
 * Normal declaration and definition macros.
 */
#define DECLARE_PER_CPU_SECTION(type, name, sec)            \
    extern __PCPU_ATTRS(sec) __typeof__(type) name

#define DEFINE_PER_CPU_SECTION(type, name, sec)                \
    __PCPU_ATTRS(sec) PER_CPU_DEF_ATTRIBUTES            \
    __typeof__(type) name
#endif

/*
 * Variant on the per-CPU variable declaration/definition theme used for
 * ordinary per-CPU variables.
 */
#define DECLARE_PER_CPU(type, name)                    \
    DECLARE_PER_CPU_SECTION(type, name, "")

#define DEFINE_PER_CPU(type, name)                    \
    DEFINE_PER_CPU_SECTION(type, name, "")

  2. 操作

    • 获取当前处理器上的变量，并禁止抢占

      get_cpu_var(name)

    • 恢复处理器抢占

      put_cpu_var(name)

    • 获得别的处理器上的数据

      per_cpu(name, cpu)
      • 注意这个函数不禁止抢占，不提供任何形式的锁保护

三、运行时per cpu数据

  1. 动态创建

    <linux/percpu.h>
#define alloc_percpu(type)    \
(typeof(type) __percpu *)__alloc_percpu(sizeof(type), __alignof__(type))    

<mm/percpu.c>
/**
 * __alloc_percpu - allocate dynamic percpu area
 * @size: size of area to allocate in bytes
 * @align: alignment of area (max PAGE_SIZE)
 *
 * Allocate zero-filled percpu area of @size bytes aligned at @align.
 * Might sleep.  Might trigger writeouts.
 *
 * CONTEXT:
 * Does GFP_KERNEL allocation.
 *
 * RETURNS:
 * Percpu pointer to the allocated area on success, NULL on failure.
 */
void __percpu *__alloc_percpu(size_t size, size_t align)
{
    return pcpu_alloc(size, align, false);
}
    • alloc_percpu给每个cpu创建一个指定类型的对象，是对原始宏__alloc_percpu的封装，封装后按给定类型的自然边界对齐
    • 原始宏__alloc_percpu接收两个参数：
      • size：要分配的实际字节数
      • align：按多少字节数对齐
        • __alignof__是gcc的内置函数，返回指定类型或lvalue需要的对齐字节数，语义类似sizeof
    • 返回一个指针，间接引用动态创建的per cpu data

  2. 释放

    <linux/percpu.h>
/**
 * free_percpu - free percpu area
 * @ptr: pointer to area to free
 *
 * Free percpu area @ptr.
 *
 * CONTEXT:
 * Can be called from atomic context.
 */
void free_percpu(void __percpu *ptr)

四、per cpu data的优势

  1. 减少数据锁定，每个cpu访问自身的数据
  2. 减少缓存失效，per cpu data按cache-align对齐，确保数据只在当前处理器的cache line上
  3. 唯一的安全要求是禁止内核抢占，代价比上锁小很多，封装的接口能方便完成，在上下文中很安全
  4. 注意，不能在访问过程中睡眠，否则可能被抢占
