一、ODR原则(One Definition Rule)

  1. C++标准规定“同一个程序其用到的每个非内联函数，在所有文件中只能有一个定义，而对于类和内联函数，独立编译单元最多只能定义一次”
  2. 要确保相同实体的所有定义都是相同的
  3. 对内联函数和模板，通常定义在头文件，以便于在编译单元中生成定义

  4. 在不同的编译单元中，当且仅当两个定义中的每个符号都相同，并且符号在两个编译单元有相同意思时，这两个符号定义才完全相同：

    示例1：符号不同，不符合ODR
//1.cpp
int getVal();
//2.cpp
typedef int l;
l getVal();

示例2：意思不同
//1.cpp
typedef long l;
l getVal();
//2.cpp
typedef int l;
l getVal();

二、ODR的约束

  1. 外部链接与内部链接

    • 外部链接
      • 指在名字空间作用域声明，或全局作用域声明的且非static
    • 内部链接
      • 在未命名的名字空间作用域声明(局部于文件)，或全局作用域使用static声明

  2. 应用程序范围的等价性约束——程序内部只能有一处定义

    • 非内联函数和非内联成员函数(static?)
    • 具有外部链接的变量
    • 静态成员变量
    • 非内联的函数模板、成员函数模板和类模板的非内联成员(声明时没有export)
    • 类模板的静态成员变量(声明时没有export)

  3. 编译单元范围的等价性约束——编译单元内只能有一处定义

    • 所有头文件要加#include guard的原因
    • 使用前必需定义类类型的场合：
      • 创建类对象
      • 声明类的成员变量
      • 对类对象应用sizeof或typeof
      • 显示或隐式访问类成员
      • 在类对象上发生显示或隐式类型转换
      • 对类对象进行复制
      • 定义和调用参数含有或返回值为类对象的函数(仅仅声明就不需要)
    • 对类模板，实例化点(POI，point of instantiation)需要定义
    • 注意：对内联函数，定义可以位于使用点之后

  4. 跨编译单元的等价性约束

    • 跨编译单元的多个定义常带来不匹配的问题，C++标准约定，编译器要给出“未经定义行为”的信息
    • 跨编译单元的约束：
      • 当一个实体在两个位置都进行定义时，这两个位置必须由完全相同的标记序列组成：
        • 包括关键字、运算符、标识符和预处理后的标记
      • 这些位置不同的标记在不同的上下文中，必需指定相同的对象(指定义本身要是相同的)
    • 对于可能多个编译单元中被定义的实体，通常放在头文件中定义，保证定义完全一样
    • 有种例外情况是，如果用到的对象时具有相同值的常量，并且不适用结果表达式的地址，那么标记就被认为是相同的
      • 这些情况下的const隐含static语义，通常只是用于定义编译期常量，而不是运行期的某个存储位置
