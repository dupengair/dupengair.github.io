#一、相等与等价

  1. 相等
    • 基于operator==或用户自定义判断式
    • 如find算法，判断值是否相同
    • 对象x和y有相等的值并不意味着所有它们的成员有相等的值，取决于operator==方法的实现

  2. 等价

    • 基于operator<或用户自定义判断式
    • 如insert成员函数，判断元素存在以及大小

    • 两个值如果没有哪个在另一个之前（关于某个排序标准），那么它们等价（按照那个标准）

      !(x < y) && !(y < x)

    • 基于在一个有序区间中对象值的相对位置，一般在标准关联容器（set、multiset、map和multimap）的排序顺序方面有意义

#二、差别

  1. 实现一个函数对象，ciStringCompare，它对set进行了忽略大小写比较

    // 用于忽略大小写字符串比较的类        
struct CIStringCompare: public binary_function<string, string, bool> 
{ 
    bool operator()(const string& lhs,const string& rhs) const {
        return ciStringCompare(lhs, rhs); 
    } 
}

  2. 对于一个给定的对象插入“Persephone”和“persephone”，只会执行一次，此时体现等价

    set<string, CIStringCompare> ciss;
ciss.insert("Persephone"); // 一个新元素添加到set中
ciss.insert("persephone"); // 没有新元素添加到set中

  3. 使用set的find成员函数搜索字符串“persephone”，搜索会成功

    if (ciss.find("persephone") != ciss.end())...    // 这个测试会成功

  4. 用非成员的find算法，搜索会失败，因为find算法体现相等

    if (find(ciss.begin(), ciss.end(),
    "persephone") != ciss.end())...             // 这个测试会失败
    • 体现了等价但不相等
    • 因此，应当优先选择成员函数
