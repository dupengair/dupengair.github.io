title: 三十七、绝不重新定义继承来的缺省参数值
date: 2016-05-13 17:02:01
categories: [阅读笔记, 《Effective C++》]
tag: 《Effective C++》
toc: true

#一、前提

  • 有了前面的条款36，即只应该重新定义继承的virtual函数，所以这里讨论局限于“继承一个带有缺省参数的virtual函数”

#二、实例
类型定义：
class Shape {
public:
enum ShapeColor { Red, Green, Blue };
virtual void draw(ShapeColor color = Red) const = 0;
…
};

class Rectangle: public Shape {
public:
    virtual void draw(ShapeColor color = Green) const = 0;        //重新定义继承的缺省参数
    ...
};

class Circle: public Shape {
public:
    virtual void draw(ShapeColor color) const = 0;                  //重新定义继承的缺省参数
    ...
};

对象定义：
Rectangle r;                //r的静态类型为Rectangule，  无动态类型
Circle c;                    //c的静态类型为Circle，      无动态类型
Shape *pr = new Rectangle;    //pr的静态类型为Shape*，     动态类型为Rectangule*
Shape *pc = new Circle;     //pr的静态类型为Shape*，     动态类型为Circle*

调用：
r.draw();        //默认参数来自派生类：Green，静态绑定
pr->draw();        //默认参数继承自基类：Red，  动态绑定
c.draw();        //error! 静态绑定的参数来自派生类，不提供默认参数
pc->draw();        //默认参数继承自基类：Red，  动态绑定
  • 由此可见，静态绑定下的参数来自派生类，动态绑定下参数继承自基类，如果重新定义了继承来的缺省参数，会导致同一个对象不同绑定方式下的表现不一致。

#二、动态绑定与静态绑定

  1. virtual函数为动态绑定，缺省参数却是静态绑定
  2. 对象的动态类型（dynamic type）是指“目前所指对象的类型”，即动态类型可以表现出一个对象将会有什么样的行为
  3. 动态类型可以在程序执行过程中改变
  4. virtual函数为动态绑定，在调用时的实现方式由调用对象的动态类型决定，即pointer或reference所指的对象。
  5. 缺省参数为静态绑定，意思是在调用一个定义于derived class内的virtual函数时，可能使用base class所指定的缺省参数
  6. 动态绑定，包括pointer或reference的方式

#三、inside C++技术内幕

  1. C++的virtual函数与其默认参数的绑定方式不一致的原因，在于运行期效率。
  2. 如果缺省参数是动态绑定，编译器就必须有办法在运行期~为virtual函数决议适当的默认参数值，这比virtual函数本身采用的滞后联编方式更复杂。为了程序执行速度和编译器实现的简易度，C++语言做了一定的取舍，结果就是提高效率但采用不一致的绑定方式。

#四、同时提供默认参数给base和derived class

class Shape {
public:
    enum ShapeColor { Red, Green, Blue };
    virtual void draw(ShapeColor color = Red) const = 0;
    ...
};

class Rectangle: public Shape {
public:
    virtual void draw(ShapeColor color = Red) const = 0;      //提供相同的缺省参数
    ...
};
...
  • 可以看出，代码重复且有依赖性，如果base class内的缺省参数发生变化，derived class也必须改写

#五、替代设计——NVI手法

class Shape {
public:
    enum ShapeColor { Red, Green, Blue };
    draw(ShapeColor color = Red) const             //non-virtual
    {
        doDraw(color);                            //调用一个virtual
    }
private:
    virtual void doDraw(ShapeColor color) const = 0;        //完成真正的工作
    ...
};    

class Rectangle: public Shape {
private:
    virtual void doDraw(ShapeColor color) const;              //不须指定缺省参数
    ...
};
  • 由于条款36，non-virtual函数绝不应该被改写，所以draw函数的缺省参数color总是为Red，同时派生类可以定义自己的实现方式。

#六、总结
绝不要重新定义一个继承来的缺省参数，因为缺省参数值都是静态绑定，virtual函数才是唯一应该改写的东西——它是动态绑定的。
