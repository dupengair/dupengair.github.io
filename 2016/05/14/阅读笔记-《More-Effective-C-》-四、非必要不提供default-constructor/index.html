title: 四、非必要不提供default constructor
date: 2016-05-13 17:02:13
categories: [阅读笔记, 《More Effective C++》]
tag: 《More Effective C++》
toc: true

default constructor的意思是在没有任何外来信息的情况下初始化对象。对于必须具有外来信息才能生成对象的classes，则不必拥有default constructors

#一、缺乏default constructors带来的限制

  1. 无法产生数组，一般来说没有任何方法可以为数组中的对象指定constructor自变量。

    class Equipments {
public:
    Equipments( int ID );        //使编译器无法合成default constructor
    ……
};

Equipments bestPieces[10];        // error! 无法调用Equipments ctor
Equipments *bestPieces = new Equipments[10]        // error!

  2. 不适用于template-based container classes。因为对于这种class，被实例化的目标类型必须有一个default constructor。这是一个普遍的共同需求，因为这些container内几乎总是会产生一个以“template”类型参数为类型的数组。

    template<class T>
class Array {
public:
    Array (int size);
    …
private:
    T *data;
};

template<class T>
Array<T>::Array(int size)        //构造一个T类型是数组用来放置data
{
    data = new T[size];        //需要对每个元素调用 T::T()
}
    • 如果谨慎设计，可以消除对default constructor的需求。

  3. 对于Virtual base classes来说，如果其缺乏default constructors，将不是一个好的设计，因为其构造函数的参数必须要有派生层次最深的class提供。于是要求所有的derived class都了解其设计。

#二、解决缺乏default constructor的情况下无法创建数组
办法有2: （书上说有3，找不到第三种办法）

  1. 使用non-heap数组，在定义数组时就初始化每一个元素

    int ID1, ID2, …
Equipments bestPieces[] = {
    Equipmemts(ID1),
    Equipments(ID2),
    …
};

  2. 更一般化的做法是使用指针数组，而不是对象数组

    typedef Equipmemts *PEP;

PEP bestPieces[10];            // yes! 不需要调用ctor
PEP *bestPieces == new PEP[10];        // 也不需要调用ctor

for(int i=0; i<10; i++)
    bestPieces[i] = new Equipmemts(IDn);
    • 这个办法有两个缺点：
      • 必须记得将数组所指的所有对象删除，否则会memory leak
      • 必须额外空间来保持指针，加上objects的内存总量比较大

#三、通过placement new避免过度使用内存

  1. 可以先为数组分配raw memory。然后使用placement new在已分配的内存上构造objects

    void *rawMemory = operator new[] (10*sizeof(Equipmemts));
Equipmemts *bestPiece = static_cast<Equipments*>(rawMemory);
for(int i=0; i<10; i++)
    new (&bestPieces[i]) Equipmemts(IDn);        //placememt new

  2. 通过placement new允许在缺乏default constructor的情况下产生对象数组，在数组内对象生命结束时，需要手动调用其constructors，最后还得调用operator delete[] 的方式释放raw memory。

    for(int i=9; i>=0; i--)
    bestPiece[i].~Equipments[];            //以相反的顺序析构对象
 operator delete[] (rawMemory);            //释放 raw memory

  3. 千万不要对raw memory采用一般的数组删除语法，删除一个不以new operator获得的指针，其行为没有定义。

    delete [] bestPieces;

#四、滥用default constructors

  1. 几乎总是造成class内的其它member functions的复杂，因为这使得object变得很容易产生出来，其member functions必然要添加判断条件已确保产生出来的objects在逻辑上正确。
  2. 添加无意义的default constructors也会影响classes的效率。同上，需要在member function中添加大量的条件判断。
  3. 不使用default constructors带来的好处是可以预期它们所产生的对象总是被完全初始化，实现上富有效率。
