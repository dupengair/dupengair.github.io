  1. Common/objects

    • 每种类型都有隐式转换至其自身，和一般用法相反，why？作用？
    • 每种类型的运算操作符被定义为成员函数，无法进行类型转换？见E24

  2. Common/Enum.h

    • ChildrenEnum 定义的各个成员函数，先调用Enum将自己类型的objects从owner中查找出来（通过dynamic cast），然后遍历查找结果dynamic cast成自己类型的objects，这是两个重复的动作？Enum返回的结果已经是cast过后的对象了。

  3. Base/Config.cpp

    • Base::ChangesThread()内临时变量changes没有用到，可以放到Base::ChangesThread_Iteration()内部处理，从而减少其一个参数。（果然在EdgeOS空间下的派生类中就没有第二个参数了）
    • Base::ChangesThread()每S轮询对比一次配置变化？

  4. System/Event.h

    • 目前event回调会将一个event对象上添加的所有函数对象调用一遍，无法调用指定的函数，可以用函数对象模板来包装callback函数，并运用traits的思想来识别类型
    • 用户定义自己的回调函数时，通过基类operator+来完成赋值过程，实际上可以将任意参数的函数赋值，因为回调函数为了兼容任意参数形式，定义为：typedef void (*Callback)(void); 而添加过程定义为：event_args::operator+=((const Callback&)func);

  5. EdgeOS/resource.h

    • 目前给用户注册的的显示当前操作进度的回调函数是一个event_1args对象，只接受一个int型的参数，表示当前进度的百分比，供用户自定义进度显示过程，最好是传回max和count两个值，显示当前的总任务量和已完成任务量，这样用户操作更自由。
