  1. 对于private继承的class，编译器不会自动将derived class对象转换为base class对象，这意味着不能使一个基类指针或引用指向一个private继承的派生类

  2. private继承无法阻止基类的virtual函数接口被派生类重定义。

  3. virtual dtor 声明为private，表明只用作基类

  4. 利用dynamic_cast判断当前对象的类型

  5. 利用自动类型推导为lambda函数取名：auto func = [] () { cout << “Hello world”; };

  6. 内联函数只能静态链接

  7. 只有以by value方式传递参数，或是当对象被传递给一个reference-to-const参数时，隐式类型转换才会发生，如果对象被传递给一个reference-to-non-const参数，转换并不会发生

  8. 相比#pragma， _Pragma是一个标识符，所以可以用在宏里面

  9. c++11支持long long，至少64位

  10. __cplusplus被定义为一个整数，c++11中为201103L，可用于代码中判断支持c++11

  11. 调用boost::make_shared要求类不能使抽象基类，因为boost代码中要建立该类的对象

  12. 智能指针用作函数参数时，用值传递，不要用引用传递

  13. 定义std::ostringstream str;时出现”aggregate incomplete type, cannot be defined”错误，将头文件换成

  14. 编码将inline宏定义，调试时使用普通函数，这样才能调试。

  15. 函数参数为字符串的情况下， const char 比 std::string安全，因为 const char 隐式转换为bool的优先级更高，如果两个重载函数的同一个参数分别为bool和string，调用函数是参数为”string”（const char*），会优先匹配为bool，此时可以在调用时强制类型转换为string（const std::string&）

  16. C++有可能在栈上存在内存泄漏，如constructor中抛出异常，如果对象还未构造完毕，析构函数不会被调用，栈上的对象就不会被析构，或者destructor中抛出异常，造成无法正常析构，此时对象会驻留内存

  17. 动态类型语言（dynamic typing language）是指类型检查发生在运行期间（run-time）的语言。静态类型语言（static typing language）是类型检查发生在编译期间（compile-time）的语言。
