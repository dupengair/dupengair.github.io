四十、多重继承使用须知

May 14, 2016 | 阅读笔记《Effective C++》 | Hits
文章目录

#一、钻石型多重继承

  • 被派生类直接继承的base classes在继承体系中有更高级的base classes，并且它们中的一些可能会是同一个base class。

    class File {…};
class IFile : public File {…};
class OFile : public File {…};
class IOFile : public IFile, public OFile {…};

#二、问题

  1. C++解析重载函数调用的规则：在确定可用的重载函数前，先确认此函数为调用的最佳匹配，找到最佳匹配函数后才检验可访问性。
  2. 即使在不同的基类中有访问性不同的同名函数，也会导致编译器寻找最佳匹配的二义性。钻石型继承的二义性问题就更严重了。
  3. 为了解决二义性，必须明白指出要调用哪一个base class中的函数。

#三、virtual inheritance

  • C++继承的缺省做法是复制基类，如果不希望这样，可以让base class成为一个virtual base class，只需要让直接继承它的classes 采用 virtual 继承就行。

#四、virtual inheritance带来的问题

  1. 为了避免继承带来的成员重复，编译器执行的特殊操作户导致使用virtual继承的class对象体积增大，访问成员变量的速度变慢。
  2. 对virtual base class 的初始化规则更为复杂，且不直观。virtual base class的初始化由继承体系最底层（most derived）class 负责，这导致：
    • 不管继承层次多深，最底层class都要了解最顶层的virtual base class，带来了复杂性还丧失了一部分封装性。
    • 当新的derived class继承这个体系时，需要承担对virtual base class的初始化。

#五、对virtual inheritance的取舍

  1. 非必要不使用virtual bases，平常尽量使用non-virtual继承
  2. 如果使用virtual继承，尽可能避免里面有数据成员

#六、实例

class IPerson {                            //指出需要实现的接口
public:
    virtual ~IPerson();
    virtual std::string name() const = 0;
    virtual std::string birthDate() const = 0;
};

clase DataBaseID {…};                            //辅助实现，不重要

class PersonInfo {                                //可以用来实现IPerson接口
public:
    explicit PersonInfo(DatabaseID pid);
    virtual ~PersonInfo();
    virtual const char* theName() const;
    virtual const char* theBirthDate() const;
    virtual const char* valueDelimOpen() const;
    virtual const char* valueDelimClose() const;
    …
};

class CPersion : public IPerson, private PersonInfo {        //多重继承
public:
    explicit CPerson(DateBaseID pid):PersonInfo(pid) {}
    virtual std::string name() const            //实现IPerson成员函数
    { return PersonInfo::theName(); }
    virtual std::string birthDate() const        //实现IPerson成员函数
    { return PersonInfo::theBirthDate(); }
private:
    const char* valueDelimOpen() const { return ""; }        //重新定义继承来的virtual函数
    const char* valueDelimClose() const { return ""; }
