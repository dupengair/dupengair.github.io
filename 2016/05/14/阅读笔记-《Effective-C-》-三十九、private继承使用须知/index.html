title: 三十九、private继承使用须知
date: 2016-05-13 17:02:01
categories: [阅读笔记, 《Effective C++》]

#一、private继承的意义

  1. private继承意味着implemented-in-terms-of（根据某物实现出）。
  2. private继承纯粹只是一种实现技术，private继承意味着只有实现部分被继承，接口部分应该略去。
  3. private继承在软件设计层面上没有意义，只在软件实现层面有意义(不能改变，只能用)。

#二、影响

  1. 对于private继承的class，编译器不会自动将derived class对象转换为base class对象，这意味着不能使一个基类指针或引用指向一个private继承的派生类（private继承不具备多态特性！）

    类定义：
class Base {…};
class Derived : public Base {…};
class Derived2 : private Base {…};
调用：
    Derived d;
    Derived2 d2;
    Base* pb;

   pb=&d;        //ok
     pb=&d2;        //error!

  2. 由private继承来的base class成员，在derived class中都会变成private属性，对外不可访问。

#三、设计准则

  1. 尽可能使用组合，必要时使用private
  2. 当private/virtual函数相关时，或考虑程序体积时使用private

#四、组合与private

  1. 使用private方式为类添加timer计数器的实例：

    class Timer {
public:
    explicit Timer(int tickFrequency);
    virutal void onTick() const;        //客户class需要的接口
    ……
};

class MyClass: private Timer {
private:
    virutal void onTick() const;        //客户定义自己的接口实现
    ……
};
    • 这里采用private继承，是因为Timer类的方法和接口只需要在MyClass中实现和调用，不需要被继承或外部访问。如果把它们放入public接口会导致客户端调用。这样做的缺陷是，如果MyClass被继承，那么它的派生类仍然可以重定义基类的virtual 函数，但无法调用，见条款35。即private继承无法阻止基类的virtual函数接口被派生类重定义。

  2. 使用组合的方式

    class MyClass {
private:
    class MyTimer : public Timer {
    public:
        virtual void onTick() const;
        ……
    };
    MyTimer timer;
    ……
};
    • 这里通过一个嵌套类继承Timer并实现需要的基类接口，将这个嵌套类的访问属性设为private，使MyClass的派生类无法访问这个继承的MyTimer成员，更不可能修改成员的接口，因为MyClass的派生类与MyTimer成员的接口之间不存在继承关系。
    • 由此可见，直接继承无法阻止基类的public virtual接口被重新定义，可以用组合来实现。效果类似于Java的final和C#的sealed。
    • 并且使用组合的方式能降低类的编译依赖性，可以将MyTimer定义到独立的源文件，在MyClass中只使用其指针成员，这样在MyClass类的定义文件中只需要MyTimer类的向前引用声明，不需要#include头文件，编译时就不需要MyTimer的源文件。

#五、private与程序体积

  1. 这种情况只适用于类中无：

    • non-static成员
    • virtual函数 （会带来vptr）

    • virtual base class

      class Empty {};        //一个空白类

class AnInt {
private:
    int x;        //只有一个int 成员占体积
    Empty e;    //一个独立的空白类对象，是否占用内存？
};

结果：sizeof(AnInt) > sizeof(int)

  2. C++规定：凡是独立（非附属）对象都必须有非零大小

  3. 对于独立（非附属）对象，编译器器会安插一个char空间到空对象中

  4. 处于内存对齐的考虑，额外空间可能被占用

    class AnInt : private Empty {
private:
    int x;        //只有一个int 成员占体积        
};    

结果：sizeof(AnInt) == sizeof(int)
    • 此时，空白类成为derived class对象内的base class 成分，不再是一个独立（非附属）对象。

  5. 此即为：EBO（empty base optimization）空白基类最优化

  6. EBO一般只在单一继承下可行，多继承下不起作用
  7. 大多数基类不可能为空白类，所以EBO很少成为private基础的正当理由

#六、总结

  1. private继承意味着implemented-in-terms-of（根据某物实现出）,当derived class需要访问protected base class的成员或者重新定义继承来的virtual函数时，是合理的。
  2. 和组合不同，private继承可以最空白类实施EBO，缩小对象尺寸，但用途有限。
