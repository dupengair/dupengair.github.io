十九、了解临时对象的来源

May 14, 2016 | 阅读笔记《More Effective C++》 | Hits
文章目录
  • 局部对象不是临时对象。
  • 临时对象是不可见的——不会在代码中出现，只要产生了一个non-heap object而没为它命名，便产生了一个临时对象。通常产生于两种情况：调用函数时参数的隐式类型转换和函数返回对象时

#一、调用函数时参数的隐式类型转换

  1. 发生于传递对象给某个函数，而且类型与函数参数不同时。
  2. 有两个做法可以消除转换：重新设计代码，使转换不再发生或不再需要

  3. 只有以by value方式传递参数，或是当对象被传递给一个reference-to-const参数时，转换才会发生(即此时参数不可修改，所以可产生临时变量)，如果对象被传递给一个reference-to-non-const参数，转换并不会发生

    void uppercast(string& str);
char subtleBook[] = "effective c++";
uppercast(subtleBook);            // wrong!

  4. 当程序员期望“非临时对象”被修改，此时如果编译器对reference-to-non-const对象进行隐式类型转换，会允许临时对象被修改，这种修改是没有意义的。所以C++禁止为reference-to-non-const参数产生临时对象。reference-to-const参数因为本身不可修改，所以不存在问题。

#二、返回对象

  1. 当函数返回对象时，因为返回对象没有名称，所以是一个临时对象。
  2. 返回值优化 return value optimization

#三、临时对象

  1. 临时对象很耗成本，所以应尽可能避免
  2. 更重要的是识别可能产生临时对象的地方
  3. 任何时候只要看到reference-to-const参数，就可能产生临时对象绑定至该参数。
  4. 任何时候只要看到函数返回一个对象，就会产生临时对象（并于稍后销毁）
