C++11提供一个精度中立的库——chrono，同时提供基本的C和POSIX接口

#一、chrono

  1. chrono的设计，是希望处理timer和clock在不同系统中可能不同的问题，同时强化时间精度。
  2. STL的目标是提供一个精度中立的概念，把时间段duration和时间点timepoint从特定的时钟clock中区分开来。
  3. 基本概念
    • 时间段duration：某时间单位上一个明确的tick数，如1.5个1/3秒，1.5为tick数，1/3秒为时间单位
    • 时间点timepoint：以某个时钟clock为参数，一个duration和一个起始点epoch的组合。表示经历某个epoch之前或之后一段duration所得时间，其中epoch取决于某个clock。
    • 时钟clock：不同的clock有不同的起始点epoch，当操作系统处理多个timepoint时，必须采用相同的epoch/clock，任何clock都提供产生当前timepoint的函数。

#二、duration

  1. duration是一个数值（表现tick数）和一个分数（表现时间单位，以秒记）的组合，分数由class ratio描述

    std::chrono::duration<int>             tenSec(10);    // 10 seconds, 单位秒    
std::chrono::duration<double, std::ratio<60>>     halfMin(0.5);    // 半分钟, 单位60秒
std::chrono::duration<long, std::ratio<1, 1000>> milliSec(1);    // 1ms, 单位1/1000秒

  2. 利用C++库的定义

    std::chrono::seconds        aMinute(60);
std::chrono::hours        aDay(24);
std::chrono::milliseconds    aSec(1000);

  3. 算术运算

    • 可以进行四则混合运算、加减tick和比较大小

    • 运算涉及的两个duration单位类型可以不同，所得结果为两个操作数的单位的最大公约数

      chrono::seconds d1(10);
chrono::milliseconds d2(10);
chrono::duration<int, ration<1, 3>> d3(1);
chrono::duration<int, ration<1, 5>> d4(1);
d1 - d2 的结果为9990个ms, d3 - d4 为2个1/15s

    • 其它操作

      count()获取tick数

    • 支持隐式类型转换，但只能转换为精度高的类型，反之不行，因为会损失精度，除非强制类型转换

      chrono::minute aMin;
chrono::seconds aSec;
chrono::seconds s1 = aMin;        // OK
chrono::minutes m1 = aSec;        // Error, 不能从高精度转为低精度
chrono::minutes m1 = chrono::duration_cast<chrono::minutes>(sec);    //OK

#三、Clock和Timepoint

  1. 定义
    • Clock定义一个起始点epoch和一个周期tick
    • Timepoint表现出某个时间点，关联至某个clock的某个duration值
  2. 标准库提供三个clock
    • system_clock：表现的timepoint关联至现行系统的即时时钟real-time clock
    • steady_clock：保证绝不会被调整
    • high_resolution_clock：表现当前系统中带有最短tick周期的clock

  3. 4个特定的timepoint

    Epoch：clock的timepoint的default构造函数
Current Time：clock的now()产生
Min timepoint：min()
Max timepoint：max()

  4. clock的to_time_t()转换为POSIX的time_t类型

#四、C和POSIX的Date/Time函数

  1. 这些函数定义于<ctime>内，宏CLOCKS_PER_SEC定义了clock()的单位类型，返回以1/CLOCKS_PER_SEC计的elapsed time

#五、线程阻塞

  1. Duration和timepoint可用于线程或程序的阻塞，可以指定时间段或者时间点
  2. 以_for()结束的函数使用duration，表示阻塞一段时间，以_until()结束的函数用到timepoint，表示阻塞到某个时间点
