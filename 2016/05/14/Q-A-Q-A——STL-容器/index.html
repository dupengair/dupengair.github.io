title: Q&A——STL @ 容器
date: 2016-05-13 17:01:58
categories: Q&A
tag: Q&A
toc: true

Array
1、Array是否能由别的容器初始化？ （of course）
2、empty和size哪个效率更高？ （empty）
3、下面操作有什么问题？应如何修改？
    printf("%s\n", a.begin());    （不能以迭代器体现首个元素地址，迭代器的类型取决于实现，并不是一个pointer）
    （printf("%s\n", a.data()); 或 printf("%s\n", &a[0];）
4、array的内存是否是连续相邻的？怎么看出来？（yes，适用于数组操作&a[i] == &a[0] + i;）


Vector
5、vector的容量是怎么分配的？是否可以缩小？（分配更多的，不能，可以通过交换容器的方式间接缩小）
6、如何删除vector中为某个值的元素？ （无成员函数，需要用算法v.erase(remove(v.begin(), v.end(), val), v.end());）
7、有什么方法可以控制vector内存大小？（创建时初始化，或者reserve，无法缩小）


Deque
8、deque的内存实现和vector有什么不同？为何能向在两端快速扩展？（deque实现通常为一组独立区块，头尾不同的区块向不同方向扩展，因此能在头尾两端快速安插和删除元素）
9、在效率、容量和内存使用上和vector有什么不同？（访问稍慢，多一些间接过程，容量更大，由内存区块构成，重分配效率优于vector，大小可自动缩减，不允许用户控制）
10、和vector在使用场合上的区别？（适用于元素增长频繁，需要在头部插入的情况）


List:
11、特性？（安插删除快，随机访问慢）
12、Forward list的安插、替换与擦除和别的容器有什么不同？（在指定pos后，别的在指定pos前）


Set&Map:
13、关联型容器默认的排序准则是什么？定义排序准则有哪两种方式？有什么不同？（less<T>，template参数在编译期定义或构造函数在运行期定义，前者是容器类型的一部分，比较容器时会被认为是不同的容器）
14、关联容器的查找算法复杂度是什么？普通容器？（对数复杂度，线性复杂度）
15、下面插入map的做法有什么问题？（key为常量，coll.insert(std::pair<const std::string, float>("otto", 1));）
    coll.insert(std::pair<std::string, float>("otto", 1));


Unordered:
16、unordered容器基于什么实现？哪方面性能突出？（hash table，元素查找为常量时间复杂度）


容器使用：
17、如何做到对一组元素同时采用不同的排序准则？（多个不同排序准则的容器通过reference语义共享元素）
18、需要对元素自动排序时，关联容器是否一定是最优选择？（关联式容器可以自动排序，但不意味着效率高，因为每安插一个元素就要排序一次，效率不如序列式容器的做法：先安插所有元素，然后进行一次完全排序）
