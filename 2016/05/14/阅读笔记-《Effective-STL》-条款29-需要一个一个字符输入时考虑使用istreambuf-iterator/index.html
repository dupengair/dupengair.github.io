title: 条款29 需要一个一个字符输入时考虑使用istreambuf_iterator

#一、从文件拷贝字符串

  1. 把inputFile读入fileData

    ifstream inputFile("interestingData.txt");
string fileData((istream_iterator<char>(inputFile)), istream_iterator<char>());
    • 这种方法无法把文件中的空格拷贝到字符串中。那是因为istream_iterators使用operator>>函数来进行读取，而operator>>函数在默认情况下忽略空格。

  2. 想保留空格，就要覆盖默认情况，只要清除输入流的skipws标志

    ifstream inputFile("interestingData.txt");
inputFile.unset(ios::skipws); // 关闭inputFile的忽略空格标志
string fileData((istream_iterator<char>(inputFile)), istream_iterator<char>());

#二、效率

  1. istream_iterators依靠operator>>函数进行的格式化输入：
    • 为每个operator>>调用进行建立和清除活动的特殊的iostream对象
    • 必须检查流标志
    • 必须进行全面的读取错误检查
    • 而且如果遇到问题，必须检查流的异常掩码来决定是否该抛出异常

#三、使用istreambuf_iterator

  1. stream_iterator<char>对象使用operator>>来从输入流中读取单个字符，istreambuf_iterator<char>对象进入流的缓冲区并直接读取下一个字符

  2. 即一个istreambuf_iterator<char>对象从一个istream s中读取会调用s.rdbuf()->sgetc()来读s的下一个字

    ifstream inputFile("interestingData.txt");
string fileData((istreambuf_iterator<char>(inputFile)), istreambuf_iterator<char>());
    • ，istreambuf_iterator不忽略任何字符。它们只抓取流缓冲区的下一个字符，因此不需要考虑skipws标志
