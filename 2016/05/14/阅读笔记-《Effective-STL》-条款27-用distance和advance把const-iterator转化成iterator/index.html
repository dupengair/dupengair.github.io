title: 条款27 用distance和advance把const_iterator转化成iterator
date: 2016-05-13 17:02:06

tag: 《Effective STL》

#一、适用情形

  1. 如果只有一个const_iterator，而要在它所指向的容器位置上插入新元素呢，也就是需要把const_iterator转化为iterator，但并不存在从const_iterator到iterator之间的隐式转换

    typedef deque<int>::iterator Iter;
deque<int>::const_iterator ci;
Iter i(ci);                     // 错误！没有从const_iterator到iterator隐式转换的途径
Iter i(const_cast<Iter>(ci));     // 仍是个错误！不能从const_iterator映射为iterator！

  2. list、set、multiset、map、multimap甚至散列表容器会失败，使用映射也许在vector或string的代码时能够编译，这两者特殊

  3. 映射的代码不能通过编译的原因在于，对于这些容器而言，iterator和const_iterator是完全不同的类
  4. 对于vector或string，大多数实现都会采用真实的指针作为那些容器的迭代器。
    • vector<T>::iterator是T*的typedef，而vector<T>::const_iterator是const T*的typedef，string::iterator是char*的typedef，而string::const_iterator是const char*的typedef。
    • 在这种实现的情况下，用const_cast把const_iterator映射成iterator当然可以编译而且没有问题，const_iterator与iterator之间的const_cast映射被最终解释成const T*到T*的映射

#二、从const_iterator获得iterator

  1. 要得到与const_iterator指向同一位置的iterator，首先将iterator指向容器的起始位置，然后把它向前移到和const_iterator距离容器起始位置的偏移量一样的位置即
  2. distance返回两个指向同一个容器的iterator之间的距离；advance则用于将一个iterator移动指定的距离

  3. 如果i和ci指向同一个容器，那么表达式advance(i, distance(i, ci))会将i移动到与ci相同的位置上

    typedef deque<int>::iterator         Iter;
typedef deque<int>::const_iterator     ConstIter;
deque<int>     d;
ConstIter     ci;

Iter i(d.begin());                         // 初始化i为d.begin()
advance(i, distance<ConstIter>(i, ci)); // 把i移到指向ci位置，注意强制适用const类型
    • 注意：当遇到distance调用时，编译器要根据使用的实参类型推断出迭代器类型。这里显式的指明distance调用的模板参数类型，从而避免编译器自己得出它们的类型

#三、效率
1.效率取决于你所转换的究竟是什么样的迭代器。

* 对随机访问的迭代器（比如vector、string和deque的），是常数时间的操作
* 对双向迭代器（也就是，所有其它容器和包括散列容器的一些实现）而言，是线性时间的操作
