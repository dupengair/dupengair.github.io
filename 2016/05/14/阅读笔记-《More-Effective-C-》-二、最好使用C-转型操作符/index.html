二、最好使用C++转型操作符

May 14, 2016 | 阅读笔记《More Effective C++》 | Hits
文章目录

一、旧式转型操作符

  1. 几乎允许将任何类型转为任何其它类型
  2. 难以辨识，小括号和对象名称在任何地方都有可能被使用

二、static_cast

  1. 基本和旧式转型功能相同，限制也一样。

  2. 注意static_cast不能移除表达式中的常量性

    class Widget;
class SpecialWidget: punlic Widget {};
void update(SpecialWidget *psw);
SpecialWidget sw;
const SpecialWidget &csw = sw;

update(&csw);                                    //error! 不能把const 对象传递给非const参数
update( const_cast<SpecialWidget*>(&csw) );     // yes!
update( (SpecialWidget*) csw);                    // 旧式转型

三、const_cast

  1. 只能用来改变表达式中的常量性和易变性

    Widget *pw = new SpecialWidget;
update(pw);                                        //error! 类型不对
update( const_cast<SpecialWidget*>(pw) );        // error! 不能用来转换类型

四、dynamic_cast

  1. 用来执行继承体系中“安全的向下转型或跨系统转型动作”。也就是可以将指向base class objects的pointers或references转型为指向derived class objects的pointers或references，并得知转型是否成功。
  2. 如果转型失败，会一个以null指针或者exception体现出来
  3. 还有一个用途是找出被某对象占用的内存地址(将指针转型为void*)

  4. 无法应用在缺乏虚函数的类型上，也不能改变常量性。

    int firstnumber, secondNumber;
dynamic_cast<double>(firstnumber)/secondNumber;     //error!

五、reinterpret_cast

  1. 这个转型结果和平台相关，不具有移植性，应尽量避免

  2. 最常用的用途是转换函数指针的类型

    typedef void (*FuncPtr)();
FuncPtr funcArray[10];

int dosth();
funcArray[0] = &dosth();                            // error! 类型不匹配
funcArray[0] = reinterpret_cast<FuncPtr>(&dosth);    // yes!
