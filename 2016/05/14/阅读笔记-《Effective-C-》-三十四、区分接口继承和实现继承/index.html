title: 三十四、区分接口继承和实现继承
date: 2016-05-13 17:02:02
categories: [阅读笔记, 《Effective C++》]
tag: 《Effective C++》
toc: true

#一、继承的方式

  1. 只继承成员函数的接口（pure virtual）
  2. 同时继承接口和实现，又希望能覆盖继承的实现（virtual）
  3. 同时继承接口和实现，不允许覆盖任何东西（non-virtual）

#二、成员函数的接口总是会被继承

  • public意味着is-a，所以对base class为真的任何事情一定也对其derived class为真。如果某个函数可实行于某class身上，一定也可实行于derived class身上。

#三、pure virtual函数

  1. pure virtual函数的目的是为了让derived class只继承函数接口。
  2. 可以为简朴的virtual函数提供更平常安全的缺省实现

#四、简朴的virtual函数

  1. 简朴的virtual函数目的是为了让derived class继承该函数的接口和缺省实现

  2. 同时指定函数声明和缺省行为，有可能造成忘记在derived class中重定义的危险（派生类中不用声明）

    class Airplane {
public:
    virtual void fly() {…}                //在base class中缺省的实现方式
};
class ModelA: public Airplane {};            //采用默认的fly实现
class ModelB: public Airplane { virtual void fly(); };    //自定义的fly实现
class ModelC: public Airplane {};            //本该自定义，却忘记，以至于默认

  3. 为避免上述危险，可切断virtual函数接口和其缺省实现间的联系

    class Airplane {
public:
    virtual void fly() = 0;        //声明接口继承的方式
protected:
    void defaultFly() {…}        //在base class中缺省的实现方式,与接口无关
};    
class ModelA: public Airplane {
public:
    virtual void fly() { defaultFly(); }    //注意virtual不会被inl
};

  4. 以同一个函数提供缺省实现——依赖pure virtual函数必须在derived class中重新声明，当也可以拥有自己的实现（派生类中必须声明）

    class Airplane {
public:
    virtual void fly() = 0;        //声明接口            
};
void Airplane::fly() {…}         //缺省实现pure vitual函数

class ModelA: public Airplane {
public:
    virtual void fly() { Airplane::Fly(); }        //采用缺省的fly
};
class ModelB: public Airplane {
public:
    virtual void fly() {…}        //自定义的fly实现
};    
class ModelC: public Airplane {
public:
    virtual void fly() {…}        //因为pure virtual函数必须在derived class中声明并实现，
};                                        所以不会忘记实现
    • 现在fly成员函数被分割成两个基本要素，声明部分表现出的是接口，定义部分表现出缺省行为，并且丧失了两个函数享有不同保护级别的机会，protected成了public。

#五、non-virtual函数

  1. 声明non-virtual函数是为了继承接口和强制的实现方式
  2. non-virtual函数意味着并不打算在derived class中有不同的行为，因此绝不该在derived class中被重新定义。
  3. 一个non-virtual成员函数所表现的不变性高于其特异性

#六、易犯的错误

  1. 将所有函数声明为non-virtual，这使derived class 没有空余空间做特化工作。non-virtual析构函数尤其会带来问题。
  2. 将所有成员声明为virtual。对与Interface class是正确的。但不该在derived class中被重新定义的函数应为non-virtual
