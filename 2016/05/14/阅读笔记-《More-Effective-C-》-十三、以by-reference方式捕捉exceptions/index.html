十三、以by reference方式捕捉exceptions

May 14, 2016 | 阅读笔记《More Effective C++》 | Hits
文章目录

exception传递有3种方式：by pointer、by value、by reference，应该如何选择呢？

#一、Catch by pointer

  1. throw by pointer 是唯一在传递异常时不需要复制对象的一种做法。

  2. 必须保证exception objects离开抛出异常函数时仍然存在，只有Global对象和Static对象能满足要求。

    void Func()
{
    exception ex;                // Bomb!
    ...    
    throw &ex;                    // local var
}

try {
    Func();
}
catch (exception *ex) {            // 可能捕捉到空指针
    ...
}

  3. 抛出一个指向新 heap object 的指针

    void Func()
{
    ...    
    throw new exception;        // local var
}
    • 这样的做法会使得catch子句不知道是否应该delete 这个heap 资源，因为其它地方可能传递的是exception object，而不是heap object，这里并不能判断

  4. 4个标准exceptions，bad_alloc、bad_cast、bad_typeid、bad_exception统统都是对象，不是对象指针，所以无论如何不该采用这种方式。

#二、Catch by value

  • 不但会复制两次，还会引起对象slicing 问题

    class exception {
public:
    virtual const char* what() throw();
    ...
};

class runtime_error: public exception {...};        // 标准exception

class Validation_error: public runtime_error {
public:
    virtual const cahr *what() throw();
    ...
};

void Func()
{
    if(...)
        throw Validation_error();
}

void do_something()
{
    try {
        Func();
    }
    catch (exception ex) {            // Slicing!
        cerr << ex.what();            // 只能调用exception类的what
        ...
    }
}

#三、Catch by references

  1. 没有切割（slicing）问题，而且exception objects只会被复制一次
  2. catch到的对象可以以多态的方式运行

#四、总结
catch by reference 可以带来的好处：

  1. 避开对象删除问题
  2. 避开exception object的slicing问题
  3. 保留捕捉标准exception 的能力
  4. 约束了exception objects 被复制的次数
  5. 是否能move方式？
