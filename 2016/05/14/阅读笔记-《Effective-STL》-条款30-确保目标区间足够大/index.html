title: 条款30 确保目标区间足够大
date: 2016-05-13 17:02:07

#一、问题

  1. STL容器能自动扩容

  2. 问题常出现在向容器中插入对象但并没有告诉STL他们所想

    int transmogrify(int x);    // 这个函数从x产生一些新值
vector<int> values;            // 把数据放入values
vector<int> results;        // 返回的values附加到results
transform(values.begin(), values.end()， results.end(), transmogrify);    // Bomb!
    • *results.end()没有对象，调用transform是错误的，因为它会给不存在的对象赋值

#二、正确的做法

  1. 把transform的结果放入容器的结尾”的方式是调用back_inserter来产生指定目标区间起点的迭代器

    transform(values.begin(), values.end(), back_inserter(results), transmogrify);

  2. 在内部，back_inserter返回的迭代器会调用push_back，所以可以在任何提供push_back的容器上使用back_inserter（也就是任何标准序列容器：vector、string、deque和list）

  3. 在前面插入，可以使用front_inserter。在内部, front_inserter 利用了 push_front，所以front_insert只和提供那个成员函数的容器配合（也就是deque和list）

    list<int> results; // results现在是list
transform(values.begin(), values.end(), front_inserter(results), transmogrify); // 插入transform的结果
    • 不太常用：
      • results中的对象顺序会和values中对应的对象顺序相反
      • 另一个原因是vector不提供push_front，所以front_inserter不能用于vector

  4. 如果要transform把输出结果放在results前端，但你也要输出和values中对应的对象顺序相同，只要以相反的顺序迭代values

    list<int> results; 
transform(values.rbegin(), values.rend(), front_inserter(results), transmogrify); // 保持相对的对象顺序

  5. 是inserter允许强制算法把它们的结果插入容器中的任意位置

    vector<int> values; 
vector<int> results;
transform(values.begin(), values.end(), inserter(results, results.begin() + results.size()/2),transmogrify);

#三、效率

  1. transform每次对目的区间的插入只完成一个对象，如果容器是vector或string时，预先调用reserve，避免重新分配内存

    vector<int> values;
vector<int> results;
results.reserve(results.size() + values.size()); 
transform(values.begin(), values.end(), inserter(results, results.begin() + results.size() / 2), transmogrify); // 没有任何重新分配操作

  2. 警惕错误：即使预先reserve内存，也不能再容器尾部执行transform算法，必需配合inserter

    vector<int> values;
vector<int> results;
results.reserve(results.size() + values.size()); 
transform(values.begin(), values.end(), results.end(), transmogrify); // Bomb！

    • 正确做法：

      transform(values.begin(), values.end(), back_inserter(results), transmogrify);

#四、覆盖现有容器的元素

  1. 不需要插入迭代器，但仍然需要确保你的目的区间足够大

    vector<int> values;
vector<int> results;
...
if (results.size() < values.size()){    // 确保results至少和values一样大
    results.resize(values.size());
}
transform(values.begin(), values.end(), results.begin(), transmogrify);    // 覆盖values.size()个results的元素

  2. 或者可以清空results然后用通常的方式使用插入迭代器

    ...
results.clear();                         // 销毁results中的所有元素
results.reserve(values.size());         // 保留足够空间
transform(values.begin(), values.end(), pack_inserter(results), transmogrify);

#五、总结

  1. 要使用一个要求指定目的区间的算法，必需确保目的区间已经足够大或者在算法执行时可以增加大小
  2. 如果要增加大小，就使用插入迭代器
