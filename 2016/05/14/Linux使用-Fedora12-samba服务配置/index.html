title: Fedora12 samba服务配置
date: 2016-05-13 17:01:51
categories: Linux使用
tag: Linux使用
toc: true

  1. 建立共享文件目录并设置其权限为777

  2. 编辑samba配置文件：

    vi /etc/samba/smb.conf
[global]
security = share//安全级别，user需要用户名和密码，share级别则不要
[共享目录名]
path = 共享文件目录绝对路径
read only = no
browseable = yes
guest ok = yes
writable = yes
public = yes

  3. 关闭seLinux：

    vi /etc/sysconfig/selinux
#SELINUX=enforcing
SELINUX=disabled

    • 重启生效，如果不想重启，用命令：

      #setenforce 0
如果无法访问，可试试：
setsetool -P use_samba_home_dirs 1

  4. 设置smb、nmb服务自启动：

    chkconfig smb on　 ← 设置 Fedora Samba 自启动
chkconfig --list smb　 ← 确认 Fedora Samba 启动标签，确认 2-5 为 on 的状态
smb 0:off 1:off 2:on 3:on 4:on 5:on 6:off

chkconfig nmb on　 ← 设置 nmb 自启动
chkconfig --list nmb　 ← 确认 nmb 启动标签，确认 2-5 为 on 的状态
nmb 0:off 1:off 2:on 3:on 4:on 5:on 6:off

  5. 启动smb、nmb服务：

    /etc/rc.d/init.d/smb start　 ← 启动 Fedora Samba 服务
Starting SMB services: [ OK ]
/etc/rc.d/init.d/nmb start　 ← 启动 nmb 服务
Starting NMB services: [ OK ]

  6. 如果想以用户名密码方式登录samba服务器，需先将用户添加至samba用户数据库：

    smbpasswd -a dup　 ← 将系统用户 dup（例）加入到 Fedora Samba 用户数据库
New SMB password:　 ← 输入该用户用于登录 Fedora Samba 的密码
Retype new SMB password:　 ← 再次确认输入该密码
同时修改/etc/samba/smb.conf ：
[global]
security = user//安全级别，user需要用户名和密码，share级别则不要

  7. 如果Windows无法访问Linux共享目录：

    依次打开 系统->管理->SELinux Manegement

在弹出来的窗口中选择左边的“布尔值”选项，然后在右边查看以 samba 开头的选项，选择“allow samba to share any file/directory read/write” 和 “allow samba to share any file/directory read only”。
