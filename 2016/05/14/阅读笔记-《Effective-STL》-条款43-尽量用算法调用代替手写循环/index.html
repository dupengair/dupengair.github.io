#一、算法

  1. 每个算法接受至少一对用来指示将被操作的对象区间的迭代器
  2. 算法内部是一个循环

#二、算法比手写循环优越

  • 效率：算法通常比程序员产生的循环更高效
  • 正确性：写循环时比调用算法更容易产生错误
  • 可维护性：算法通常使代码比相应的显式循环更干净、更直观

#三、对比

  1. 效率

    • 手写循环

      for (list<Widget>::iterator i = lw.begin(); i != lw.end(); ++i) {
    i->redraw();
}
      • 会经过多次循环比较和检查list::end

    • 算法

      // 调用lw.end()求值，只有一次
for_each(lw.begin(), lw.end(), mem_fun_ref(&Widget::redraw));

    • STL的实现者知道begin和end（以及类似的函数，比如size）用得很频繁，所以他们尽可能把它们实现得最高效。几乎肯定会inline它们，并努力编码使得绝大部分编译器都可以通过将计算结果提到循环外（译注：频度优化的一种）来避免重复计算

    • 第一个主要影响因素是库的实现者可以利用他们知道容器的具体实现的优势，用库的使用者无法采用的方式来优化遍历。比如，在deque中的对象通常存储在（内部的）一个或多个固定大小的数组上。基于指针的遍历比基于迭代器的遍历更快，但只有库的实现者可以使用基于指针的遍历，因为只有他们知道内部数组的大小以及如何从一个数组移向下一个

    • 第二个主要的效率因素是，除了最微不足道的STL算法，所有的STL算法使用的计算机科学都比一般的C++程序员能拿得出来的算法复杂——有时候会复杂得多。几乎不可能被打败的sort及其同族算法（比如，stable_sort()，nth_element()等，参见条款31）；适用于有序区间的搜索算法（比如，binary_search，lower_bound等，参见条款34和35）也一样好；就算是很平凡的任务，比如从连续内存容器中除去一些对象，使用erase-remove惯用法都比绝大多数程序员写的循环更高效（参见条款9）

    • 写循环时，比较麻烦的事在于确保所使用的迭代器（a）有效，并且（b）指向你所期望的地方

  2. 正确性与可维护性

    • 示例

      // C API：这个函数的参数是一个能放arraySize个double的数组的指针， 函数向这个数组写入数据。它返回写入double的个数
size_t fillArray(double *pArray, size_t arraySize);
double data[maxNumDoubles];    // 建立本地数组，大小是最大可能的大小
deque<double> d;             // 建立deque，
... 
size_t numDoubles = fillArray(data, maxNumDoubles); // 从API获取数组数据

// 这段代码有一个bug！
for (size_t i = 0; i < numDoubles; ++i) {             // 对于每一个数据，
    d.insert(d.begin(), data[i] + 41);                 // 在d的前端插入data[i]+41
}
      • 这段代码有一个bug，插入的元素于在data中对应的元素是反序的。因为每次的插入点都是d.begin()，最后一个被插入的元素将位于deque的前端

    • 第一次修改：

      deque<double>::iterator insertLocation = d.begin(); // 记下d的起始迭代器

// 在insertLocation插入data[i]+41，然后insertLocation递增；这段代码也有bug！
for (size_t i = 0; i < numDoubles; ++i) { 
    d.insert(insertLocation++, data[i] + 41);  
}
      • 递增了指示插入位置的，避免了循环每次对begin的调用
      • 但它导致了未定义的结果。每次调用deque::insert，都将导致所有指向deque内部的迭代器无效，包括上面的insertLocation。在第一次调用insert后，insertLocation就无效了，后面的循环迭代器可以直接让人发疯

    • 最终正确版：

      deque<double>::iterator insertLocation = d.begin(); 

// 每次调用insert后都重新读取insertLocation以保证迭代器有效，然后递增它
for (size_t i = 0; i < numDoubles; ++i) { 
    insertLocation = d.insert(insertLocation, data[i] + 41); 
    ++insertLocation; 
}

  3. 使用算法

    // 把所有元素从data拷贝到d的前端，每个增加上41
transform(data, data + numDoubles, inserter(d, d.begin()), bind2nd(plus<double>(), 41));
