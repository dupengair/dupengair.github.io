title: 六、区别increment和decrement操作符的前置（prefix）和后置（postfix）形式
date: 2016-05-13 17:02:12
categories: [阅读笔记, 《More Effective C++》]
tag: 《More Effective C++》
toc: true

#一、后置式的int参数

  1. 为了区分前置和后置形式，语言上规定后置式有一个int变量，在后置式被调用时，编译器默默的为int指定一个0值。
  2. 后置式的int变量的唯一目的是区分前置和后置，在定义中并不会用到，为了避免编译器给出参数没有使用的警告，定义时可省略不打算使用的参数名称。

#二、返回值

  1. 更重要的是，操作符的前置和后置式返回不同的类型，前置返回reference，后置返回const。

  2. 如果不返回const 对象：

    UPInt UPInt::operato++ (int)        //返回值应为const
{
    UPInt old = *this;
    ++(*this);
    return old;
}

UPInt i;
i++++;
    • 等价于i.operator++(0).operator++(0);它有2点坏处：
      • 和内置类型的行为不一致。设计class的一条无上宝典就是，一旦有疑虑，看内置类型的行为，内置int类型不允许连续两次使用后置的increment操作符。
      • 即使能两次使用，其行为也不是预期。因为第二个operator++改变的是第一个返回的临时对象，而不是原始对象，原始对象i只被++操作了一次

#三、处理用户定制类型

  1. 应尽可能使用前置increment，因为它天生体质较佳。

  2. 后置式increment和decrement操作符的执行应该以其前置式兄弟为基础。这样就只用维护前置式版本，后置式版本会自动调整为一致的行为。

    class UPInt {
public:
    UPInt& operator++();
    const UPInt operator++(int);

    UPInt& operator--();
    const UPInt operator--(int);

    UPInt& operator += (int);
    …
};

UPInt& UPInt::operator++()
{
    *this += 1;
    return *this;
}

const UPInt UPInt::operator++(int)
{
    UPInt oldValue = *this;
    ++(*this);
    return oldValue;
}

UPInt i;
++i;
i++
--i;
i--;
