title: 三十六、绝不重新定义继承来的non-virtual函数
date: 2016-05-13 17:02:02
categories: [阅读笔记, 《Effective C++》]
tag: 《Effective C++》
toc: true

#一、实例：像polymorphic那样使用non-virtual函数的后果

类定义：
class B {
public:
    void mf();
};

class D : public B { 
public:
    void mf();         // 覆盖基类的接口
};

使用：
D x;

B* pB=&x;
pB->mf();        //调用B::mf()

D* pD=&x;
pD->mf();        //调用D::mf()
  • 由此可见，同一个派生类的对象，由基类指针和派生类指针调用，表现出不一样的行为。这和virtual的效果正好相反。

#二、原因

  • non-virtual函数是静态绑定(到指针类型)的，其决定因素不在对象自身，而在于指向对象的指针或引用的声明形式。这使得Reference和Pointer都会在同一个对象上展现出不一致行径。

#三、引申

  1. public继承意味着is-a的关系，在class内声明一个non-virtual函数会为class建立一个不变性（invariant），凌驾于其特异性（specialiazation）。这就意味着派生类不能凌驾于这个特性，否则就违反了is-a的关系。
  2. 如果是特异性，应该声明为virtual函数

#四、总结
任何情况下都不应该重新定义一个继承而来的non-virtual函数

注：模板中的参数化虚拟性使用的就是这个规则，可以当做是应用特例，任何事都有好坏之分。见《C++ templates》16.4 P289
