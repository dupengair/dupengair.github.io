title: 四十六、需要类型转换时为模板定义非成员函数
date: 2016-05-13 17:02:05
categories: [阅读笔记, 《Effective C++》]
tag: 《Effective C++》
toc: true

还记得条款24，当需要类型转换时，定义non-member函数，对于模板也一样，不过实际情况就复杂的多了。

#一、以template来讨论条款24
我们将条款24的实例模板化：

template<typename T>
class Rational {
public:
    Rational(const T& numerator = 0, const T& denominator = 1);
    const T numerator() const;
    const T demoninator() const;
    …
};

同样non-member函数也定义为模板：
template<typename T>
const Rational<T> operator* (const Rational<T>& rhs, const Rational<T>& rhs)
{……}

相同的调用方式：
Rational<int> oneHalf(1, 2);                // 唯一不同的是将Rational改为template
Rational<int> result = 2 * oneHalf;            // Bomb!
  1. 相同的处理方式，却无法通过编译，这说明template与non-template版本有不一样的地方。
  2. 答案就在于隐式类型转换，《C++ primier》里面写的很清楚，template在实例化过程中不进行隐式类型转换。
  3. 我们原本希望这里的操作数2被实例化的Rational 类构造函数隐式转换后，再实例化这个non-member的函数模板，但编译器不会这么做，而是用2的类型int和oneHalf的类型Rational对这个函数模板进行实参推导。结果就显而易见啦，由于函数模板实参推导时不进行隐式类型转换，所以要求两个参数一致才能实例化。

#二、利用class template进行函数模板的实参推导

  1. 上面的事实告诉我们一个结论：要先存在函数，才能在调用函数的过程中进行隐式类型转换。

  2. class template并不依赖template实参推导，而template class内的friend声明可以声明一个特定函数:

    template<typename T>
class Rational {
public:
    ……
    friend const Rational operator*             //在class template内声明为friend
        (const Rational* lhs, const Rational* rhs);
};

  3. 在一个class template内，template名称可以被用来作为template和其参数的简写表达，所以在Rational内可以只写Rational，上面的定义也可以这样写:

    template<typename T>
class Rational {
public:
    ……
    friend const Rational<T> operator*             
        (const Rational<T>* lhs, const Rational<T>* rhs);
};

  4. class template在实例化时，这个friend函数也被声明出来，这是一个non-member函数，而非函数模板，因此编译器可以在调用的时候进行隐式类型转换。

    • 即为Barton-Nackman方法，见《C++ Template》11.7 P170

#三、non-member函数定义

  1. 上面的代码能通过编译，却无法连接，原因是当这个class template被实例化时，声明于class内的friend函数是一个正常函数而非函数模板，此时这个函数是没有定义的。

  2. 最简单的办法是在声明时直接提供一份定义：

    template<typename T>
class Rational {
public:
    ……
    friend const Rational operator*             //在class template内声明friend并且定义
        (const Rational* lhs, const Rational* rhs);
    {
        return Rational( lhs.numerator() * rhs.numerator(),
             lhs.denominator() * rhs.denominator() );    
    }
};
    • 现在编译连接都没问题了。

#四、关于friend的使用
这里使用了friend声明，但目的却和传统用法不同，不是为了访问class的non-public成分。

  1. 为了让函数的所有参数都可以进行类型转换，我们需要把函数定义为(非模板的)non-member函数（条款24）
  2. 为了让这个函数随它的类型被实例化，需要把它声明在class内部
  3. 能在class内部声明non-member函数的唯一办法就是声明一个friend成员

#五、将inline声明带来的冲击最小化

  1. 可以让这个inline的成员调用一个定义于class外部的辅助函数:

    template<typename T> class Rational;                //向前引用声明    
const Rational<T> doMultiply             
    (const Rational<T>* lhs, const Rational<T>* rhs);    

template<typename T>
class Rational {
public:
    ……
    friend const Rational operator*                 //调用一个外部函数
        (const Rational* lhs, const Rational* rhs);
    {
        return doMultiply(lhs, rhs);    
    }
};

template<typename T>                                //定义外部函数    
const Rational<T> doMultiply             
    (const Rational<T>* lhs, const Rational<T>* rhs);        

{
    return Rational<T>( lhs.numerator() * rhs.numerator(),
        lhs.denominator() * rhs.denominator() );    
}
  1. 定义在外部的doMultiply()函数不支持隐式类型转换，所以不能直接使用，只被类中重载的operator*操作符调用，类型转换的工作完全由这个operator*操作符来完成。
