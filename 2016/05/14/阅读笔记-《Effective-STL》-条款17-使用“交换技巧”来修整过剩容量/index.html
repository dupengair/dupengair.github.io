#一、问题产生

  • 使用erase删除vector元素时，仅仅改变容器size，而不改变其容量，因此内存占用没有变化

#二、shrink to fit方法

  1. 原始的容器

    class Contestant {...};
vector<Contestant> contestants;        //经过长期使用，容量已经很大

  2. shrink to fit —— swap方法

    vector<Contestant>(contestants).swap(contestants);
    • 表达式vector(contestants)通过vector的拷贝构造函数建立一个临时vector，是contestants的一份拷贝
    • vector的拷贝构造函数只分配拷贝的元素需要的内存，所以这个临时vector没有多余的容量
    • 然后让临时vector和contestants交换数据，这时contestants只有临时变量的修整过的容量，而这个临时变量则持有了曾经在contestants中的发胀的容量
    • 这个语句结尾临时vector被销毁，因此释放了以前contestants使用的内存

  3. 同样适用于string

    string s;
...                 // 使s变大，然后删除所有它的字符
string(s).swap(s);    // 在s上进行“收缩到合适”

  4. 目的是使容量和这个实现可以尽量给容器的当前大小一样小

#三、清除容器并减少容量到最小值

vector<Contestant> v;
string s;
...                             // 使用v和s
vector<Contestant>().swap(v);    // 清除v而且最小化它的容量
string().swap(s);                 // 清除s而且最小化它的容量
