三十二、牢记public继承等于is-a

May 14, 2016 | 阅读笔记《Effective C++》 | Hits
文章目录

用C++进行面向对象编程，最重要的规则：public 继承意味着 “is-a”关系。

#一、bird 与 penguin

  • penguin属于bird的一种，那么让penguin类public继承bird类是否合理呢：

    class Bird {
public:
    virtual void fly();
    …
}；
class Penguin : public Bird {        //warning: penguin can not fly
    …
}；

  • Bird类中有fly方法，对于public继承它的penguin类来说，fly方法是不合理的。

    • 改进1：重新定义基类 

      class Bird {                //no fly method
public:        
    …
}；
class FlyBird : public Bird {        //only fly birds can fly    
    virtual void fly();
    …
}；
class Penguin : public Bird {        //ok
    …
};

    • 改进2：派生类中重新定义fly方法

      class Penguin : public Bird {        
    virtual void fly() { error("Penguin should not fly"); }
    …
};

  • 方法1在编译器就杜绝了问题的发生，方法2在运行期才发现问题，违背条款18，好的接口应该防止无效的代码通过编译，所以应该采用方法一。

#二、不要让生活经验干扰oop

老师告诉我们，正方形是长方形的一种，那么正方形应该public继承长方形类吗？

class Rectangle {
private:
    int Height;
    int Weight;
public:
    …
    virtual bool setHeight(int NewHeight);
};
class Square : public Rectangle {
    …
};

bool Rectangle::setHeight(int NewHeight)
{
    Height = NewHeight;
}
  • 很明显，如果Square类调用了继承自Rectangule类的setHeight()方法，那就违反了正方形的定义，高和宽可能不再相等了。

#三、关于public继承

  1. public继承意味着“is-a”，适用于base class的每一件事都一定适用于derived class，因为每一个derived class对象也一定是一个base class对象。
  2. 只对public继承才成立，private继承完全是另外一层含义。
  3. DIP原则（依赖倒置，见设计模式）—— 基类适用的地方，子类一定适用
