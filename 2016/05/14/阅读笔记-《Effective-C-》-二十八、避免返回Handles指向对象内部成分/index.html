title: 二十八、避免返回Handles指向对象内部成分
date: 2016-05-13 17:02:03
categories: [阅读笔记, 《Effective C++》]
tag: 《Effective C++》
toc: true

#一、Handles

  • Reference、指针和迭代器就是Handles，返回一个“代表对象内部数据”的handle，危害不小。

#二、封装性的讨论

如：    
class Point {                    //坐标class
public:
    Point(int x, int y);
    ...
    void setX(int newVal);
    void setY(int newVal);
    ...
};
struct RectData {                //矩形资源管理class，见条款3
    Point ulhc;
    Point lrhc;
};
class Rectangle {                //矩形class        
private:
    std::tr1::shared_prt<RectData> pData;    
    ...
public:
    Point& upperLeft() const { return pData->ulhc; }
    Point& lowerRight() const { return pData->lrhc; }
    ...
};

  1. 一个矩形类包含它的资源管理类、描述数据的坐标类。矩形类的方法获取左上和右下两个点的坐标值，不做改动，所以用const保护，但这里返回的是对内部对象的reference，所以事与愿违：

    Point p1(0, 0);
Point p2(10, 10);
const Rectangle rec(p1, p2);        //定义一个矩形对象，从(0, 0)到(10, 10)

rec.upperLeft().setX(50);        //通过返回的reference，修改了矩形对象的内部数据

  2. 所以，这里看出：

    • 成员变量的封装性最多只等于“返回其reference”的函数的访问级别。

    • 如果const函数传出一个reference，后者所指数据与对象自身有关联，而又被存于对象之外（不受对象直接的成员变量的访问控制），那么这个函数可以修改这个数据。

      改进：
class Rectangle {                //矩形class    
    ...
public:
    const Point& upperLeft() const { return pData->ulhc; }        //使返回值
    const Point& lowerRight() const { return pData->lrhc; }        //也成为常量
    ...
};

#三、dangling Handlers

  1. 即handler所属对象不存在，这种不存在的对象最常见来源就是函数返回值。

    如：    
const Rectangle boundingBox();        //某个使用矩形的GUI对象，返回一个临时对象
const Point* pUpperLeft = &(boundingBox().upperLeft());
    • boundingBox返回一个临时的Rectangle对象，当语句执行完毕后临时对象被析构，pUpperLeft成一个dangling Handlers。

#四、总结
避免返回handlers（reference、指针、迭代器）指向对象内部，可增加封装性，约束const成员函数的行为、降低dangling Handlers的可能性。
