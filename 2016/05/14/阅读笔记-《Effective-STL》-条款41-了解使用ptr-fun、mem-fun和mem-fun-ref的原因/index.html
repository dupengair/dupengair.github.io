和前一章一样，这几个函数已经过时

#一、函数调用

  1. C++函数调用的3种方式

    f(x);    // 语法#1：当f是一个非成员函数
x.f();    // 语法#2：当f是一个成员函数，而且x是一个对象或一个对象的引用
p->f();    // 语法#3：当f是一个成员函数，而且p是一个对象的指针

  2. 在for_each中调用

    • 调用函数

      void test(Widget& w); 就标记为“failed”
vector<Widget> vw; // vw容纳Widget
for_each(vw.begin(), vw.end(), test); // 调用#1

    • 调用容器对象的方法

      class Widget {
    public:
    void test(); 
    ...
};

for_each(vw.begin(), vw.end(), &Widget::test); // 调用#2（编译不通过）

#三、for_each

  1. 实现只有一种

    template<typename InputIterator, typename Function>
Function for_each(InputIterator begin, InputIterator end, Function f)
{
    while (begin != end) f(*begin++);
}
    • 实现是按照方法1进行调用

  2. 因此mem_fun和mem_fun_ref的目的是让成员函数（通常必须使用方法#2或者#3来调用的）使用句法1调用

    // 用于不带参数的non-const成员函数的mem_fun声明，C是类，R是被指向的成员函数的返回类型
template<typename R, typename C> 
mem_fun_t<R,C> 
mem_fun(R(C::*pmf)());
    • mem_fun带有一个到成员函数的指针，pmf，并返回一个mem_fun_t类型的对象。这是一个仿函数类，容纳成员函数指针并提供一个operator()，它调用指向在传给operator()的对象上的成员函数

  3. 正确的调用方式

    list<Widget*> lpw; 
...
for_each(lpw.begin(), lpw.end(), mem_fun(&Widget::test));
    • for_each接受一个mem_fun_t类型的对象，持有一个Widget::test的指针。对于在lpw里的每个Widget*指针，for_each使用语法#1调用mem_func_t，而那个对象立刻在Widget*指针上使用句法#3调用Widget::test

  4. 用法

    • mem_fun适配语法#3到语法1
    • mem_fun_ref函数适配语法#2到语法#1

  5. 关于typedef

    for_each(vw.begin(), vw.end(), test);             // 调用#1
for_each(vw.begin(), vw.end(), ptr_fun(test));    // 增加typedef，也没问题
    • for_each没有使用ptr_fun增加的typedef，所以当把test传给for_each时不必使用ptr_fun，另一方面，增加的typedef也不会造成任何问题

#四、这三个函数的使用

  1. 考虑每传递一个函数给STL组件时都使用ptr_fun，不会有运行期成本，可以编译失败时再加上，因为只是增加typedef
  2. 只要传一个成员函数给STL组件，就必须使用mem_fun和mem_fun_ref，因为其用于成员函数的适配到在STL
