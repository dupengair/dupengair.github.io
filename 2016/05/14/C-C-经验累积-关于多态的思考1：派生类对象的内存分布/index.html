title: 关于多态的思考1：派生类对象的内存分布
date: 2016-05-13 17:01:51
categories: C&C++经验累积
tag: C&C++经验累积
toc: true

今天通过两个实例，思考了一下多态的实现过程，总结如下:

  • 实例1：

    Class Base
{
……
operation();
……
}

Class Derived_A
{
……
operation();
……
}

func_op(Base &base)
{
base.operation();
}

main()
{

Derived_A A;
Base *bp = &A；

func_op(A);//调用基类Base中定义的operation();
A.operation();//调用派生类A中定义的operation();
bp->operation();//调用基类Base中定义的operation();

}

  • 实例2：

    Class Base
{
……
virtual operation();//将基类的方法定义为虚函数，实现多态
……
}

Class Derived_A
{
……
operation();
……
}

func_op(Base &base)
{
base.operation();
}

main()
{

Derived_A A;
Base *bp = &A；

func_op(A);//调用派生类A中定义的operation()，和上面不同，实现了多态;
A.operation();//调用派生类A中定义的operation();
bp->operation();//调用派生类A中定义的operation()，和上面不同，体现了多态实现的本质，派生类对象的内存布局;

}

  • 没有多态的情况下，引用派生类对象的基类方法，调用的是基类的方法，有多态的情况下，调用的是派生类的方法。

  • 而用基类指针指向派生类对象，在没有多态的情况下，调用的是基类的方法，有多态的情况下，调用的是派生类的方法。

  • 在没有多态的情况下，派生类重载了基类的getvehicle()方法，在派生类对象的内存中，同时存在基类和派生类的getvehicle()方法。所以，我们以基类类型来引用派生类对象，或是通过基类指针指向派生类对象，都能访问到派生类里的基类方法。

  • 而在多态的情况下，派生类实现的getvehicle()方法覆盖了基类的方法，在派生类对象的内存中，只存在派生类的getvehicle()方法。所以，我们以基类类型来引用派生类对象，或是通过基类指针指向派生类对象，只能访问到派生类里自己实现的getvehicle()方法。

  • 这么做的原因是，C++是一个强类型语言，如果在派生类对象的内存中，同时存在基类和派生类的getvehicle()方法，在实际的应用场合中，如果以基类类型来访问派生类对象，比如以基类类型来引用派生类对象，或是通过基类指针指向派生类对象，编译器会优先匹配基类的方法。

  • 如果没有多态，因为可以在派生类对象中找到基类的getvehicle()方法，所以以基类类型来引用派生类对象，或是通过基类指针指向派生类对象，就会匹配上派生类对象的内存中的基类方法，这相当于派生类将基类的getvehicle()方法重载。

  • 反之如果使用多态，派生类对象实现的getvehicle()方法覆盖了基类，派生类对象的内存中只有它自己实现的getvehicle()方法，那么编译器在类型匹配的时候，找不到基类的getvehicle()方法，就只能匹配派生类中的getvehicle()方法。因此，以基类类型来引用派生类对象，或是通过基类指针指向派生类对象，能且只能访问到派生类中实现的getvehicle()方法，因为基类实现的方法已经被派生类中的实现覆盖了。

  • 所以，多态的实现方式，其实质是派生类对基类(虚)函数的实现是重载（非多态，派生类对象中存在基类方法）还是覆盖（多态，派生类对象中只有派生类方法）。使用多态时，当通过基类类型去访问派生类对象，就能调用派生类定义的方法了。

  • 另外多说一句，多态的表现形式，即是通过基类指针访问派生类对象的方法，根据派生类对象的不同，同一个方法表现出来的行为不同：

    Class Base
{
……
virtual operation();
……
}

Class Derived_A
{
……
operation();
……
}

Class Derived_B
{
……
operation();
……
}

func_op(Base &base)
{
base.operation();
}

main()
{
Derived_A A;
Derived_B B;

func_op(A);//调用派生类A中定义的operation();
func_op(B);//调用派生类B中定义的operation();
}
