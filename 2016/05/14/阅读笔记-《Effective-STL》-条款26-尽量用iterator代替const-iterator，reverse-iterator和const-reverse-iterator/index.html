title: 条款26 尽量用iterator代替const_iterator，reverse_iterator和const_reverse_iterator
date: 2016-05-13 17:02:06

#一、标准迭代器

  1. 迭代器类型

    iterator
const_iterator
reverse_iterator
const_reverse_iterator

iterator  -> const_iterator   <-  const_reverse_iterator
         <-> reverse_iterator  ->

  1. 标准容器通常只接受iterator类型的参数

    iterator insert(iterator position, const T& x);
iterator erase(iterator position);
iterator erase(iterator rangeBegin, iterator rangeEnd);

  2. 尽量使用iterator取代const或者reverse类型的迭代器

    • insert和erase的一些版本要求iterator。如果你需要调用这些函数，就必须产生iterator，而不能用const或`everse iterators
    • 不可能把const_iterator隐式转换成iterator
    • 从reverse_iterator转换而来的iterator在转换之后可能需要相应的调整

#二、对于比较

  1. 混用iterator和const_iterator

    deque<int>::iterator        i;
deque<int>::const_iterator  ci;

if (i == ci)         // 比较iterator和const_iterator
    • 正常情况下，iterator应该在比较之前隐式的转换成const_iterator，真正的比较应该在两个const_iterator之间进行

    • 对于某些STL实现，无法通过编译，因为这些实现将operator==作为const_iterator的成员函数，此时左边不是const_iterator类型(见E条款24)，因此交换等式两边的内容就行：

      if (ci == i)     // 当上面比较无法通过编译时的解决方法

  2. 如果迭代器的类型不同，就有可能有问题

    if (i - ci >= 3) ... // 如果i与ci之间至少有三个元素...

    • 此时需要强制类型转换

      if (static_cast<ConstIter>(i) - ci >= 3) ... // 解决方法
