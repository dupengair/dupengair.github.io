title: system与popen执行命令的区别
date: 2016-05-13 17:01:54
categories: Linux系统编程
tag: Linux系统编程
toc: true

  • 今天遇到一个奇怪的问题，程序中用函数调用一个shell脚本，内容是输出字符到stdin
  • 开始用system调用，发现shell脚本打印的字符串直接输出到stdin，

  • 后来改用popen来操作，shell脚本输出到stdin的字符被重定向到文件流指针所指向的管道上，并没有在程序的stdin上输出。

  • 测试程序如下：

    shell:
#! /bin.sh
echo "lalalal"

程序：
#ifdef TEST_SYS
#include <iostream>
#include <string>
#include <stdio.h>
#include <stdlib.h>
#include <sstream>

using namespace std;

void call_script(string command)
{
    system(command.c_str());
}

string ReadOutputCommand (string command)
{
    #ifdef _WIN32
    #define popen _popen
    #define pclose _pclose
    #endif

    #define MAX_RES_BUF 10000

    FILE *fp;
    int status;
    char buf[MAX_RES_BUF];

    fp = popen(command.c_str(), "r");
    if (fp == NULL)
    {
        return "Cannot execute command";
    }

    stringstream response;
    while (fgets(buf, MAX_RES_BUF, fp) != NULL)
    {
        response << buf;
    }

    if(fp)
    {
        status = pclose(fp);
    }

    return response.str();
}

string call_script(void);

int main(void)
{
    string str=ReadOutputCommand("./test_echo.sh");
    cout << "popen:" << str << endl;

    call_script("./test_echo.sh");

        return 0;
    }
    #endif


    运行结果：
    #./test
    popen:lalalal

    lalalal




+ 在写Edge OS的CGI时，函数里用system调用nandwrite设置安霸设备的mac地址，完成后这个函数向标准输出返回一个标准格式的字符串表示操作成功，被web网页接受判断是否成功。一开始网页总是判断失败，但操作都是成功的。后来改用popen，网页判断操作成功，分析是因为system调用的nandwrite命令往stdin输出了字符串，影响了网页判断，改用popen后nandwrite的输出被重定向到返回的文件流中，不再输出到stdin，因此网页操作成功。
