title: 条款21 永远让比较函数对相等的值返回false
date: 2016-05-13 17:02:06
categories: [阅读笔记, 《Effective STL》]
tag: 《Effective STL》
toc: true

#一、问题

  1. 用于容器

    set<int, less_equal<int> > s; // s以“<=”排序
s.insert(10);         // 连续两次插入10        
s.insert(10);

对于容器比较，想要采取降序排列：
struct StringPtrGreater: public binary_function<const string*, const string*, bool> {
    bool operator()(const string *ps1, const string *ps2) const
    {
        return !(*ps1 < *ps2); // 只是相反了旧的测试；
    } 
};
    • 如果遇到相等值，会认为true，这对容器来说是一个无效的判别式
    • 对于比较函数实际比较的是>=

  2. 比较函数

    • 比较函数的返回值表明的是在此函数定义的排序方式下，一个值是否大于另一个，相等的值绝不该一个大于另一个

    • 对于multi类的关联容器，equal_range不是指示出相等的值的范围，而是等价的值的范围，因此如果比较函数不满足下面表达式，将不会被认为相同，也就无法被找到：

      !(x < y) && !(y < x)

  3. 因此除非比较函数总是为相等的值返回false，否则将会打破所有的标准关联型容器，不管它们是否允许存储复本

#二、正确的做法

struct StringPtrGreater: public binary_function<const string*, const string*, bool> {
    bool operator()(const string *ps1, const string *ps2) const
    {
        return *ps2 < *ps1; // 返回*ps2是否大于*ps1（也就是交换操作数的顺序）
    } 
};
