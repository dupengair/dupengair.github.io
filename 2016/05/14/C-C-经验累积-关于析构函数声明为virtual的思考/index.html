title: 关于析构函数声明为virtual的思考
date: 2016-05-13 17:01:51
categories: C&C++经验累积
tag: C&C++经验累积
toc: true

  • 情况一：

    class Base {
……
virtual ~Base(){
cout << "destuct base" << endl;
}
};

class Derived : public Base {
……
~Derived(){
cout << "destuct derived" << endl;
}
};

main(){
Base *pBase = new Derived()；
delete pBase;//pBase为基类指针，指向一个派生类对象。
}

    • 结果：

      destuct base
destuct derived

  • 情况二：

    class Base {
……
~Base(){
cout << "destuct base" << endl;
}
};

class Derived : public Base {
……
virtual ~Derived(){
cout << "destuct derived" << endl;
}
};

main(){
Base *pBase = new Derived()；
delete pBase;//pBase为基类指针，指向一个派生类对象。
}

    • 结果：

      destuct base

说明了两个问题：

  1. 基类析构函数的virtual特性，能自动传给子类

  2. 要通过执行基类指针的析构函数，来析构其所指向的派生类对象，必须采用多态的方法，即将基类析构函数声明为virtual，
    如：

    class Base {……};
class Derived : public Base {……};

main(){
Base *pBase = new Derived()；
delete pBase;//pBase为基类指针，指向一个派生类对象。
}
    • 如果基类的析构函数声明为virtual，delete的时候才能调用派生类的析构函数，否则只会调用基类自己的析构函数，因为析构方法不是多态的。即使把派生类的析构方法声明称virtual也一样。

结论：

如果通过多态的方法访问派生类（基本都这样），基类析构函数，一定要声明为虚函数。
