title: 条款9 在删除元素时仔细选择
date: 2016-05-13 17:02:09
categories: [阅读笔记, 《Effective STL》]
tag: 《Effective STL》

#一、删除为某值的元素
Container c;

  1. 对连续内存容器(vector、deque或string)，最好的方法是erase-remove惯用法

    c.erase(remove(c.begin(), c.end(), 1963), c.end());

  2. 对list，成员函数remove更高效

    c.remove(1963);

  3. 对标准关联容器（set、multiset、map或multimap），调用erase

    c.erase(1963);
    • 没有remove成员函数
    • 使用remove算法可能覆盖容器值，潜在地破坏容器

#二、通过判断式删除元素
bool badValue(int x); // 返回x是否是“bad”

  1. 对连续内存容器(vector、deque或string)，最好的方法是erase-remove_if惯用法

    c.erase(remove_if(c.begin(), c.end(), badValue), c.end());

  2. 对list，成员函数removet_if

    c.remove_if(badValue);

  3. 对标准关联容器（set、multiset、map或multimap），必须写一个循环来删除元素

    • 版本1：使迭代器失效的错误做法

      for (AssocContainer<int>::iterator i = c.begin(); i!= c.end(); ++i) { 
    if (badValue(*i)) 
        c.erase(i);     // Bomb! 迭代器失效！
}
      • 迭代器失效导致后面的操作结果未知

    • 版本２：使用后置递增，先得到下一个迭代器

      for (AssocContainer<int>::iterator i = c.begin(); i!= c.end();) { 
    if (badValue(*i)) 
        c.erase(i++); 
    else
        ++i;    
}

#三、在循环中的操作

  1. 对标准关联容器（set、multiset、map或multimap），直接在循环中增加操作

    for (AssocContainer<int>::iterator i = c.begin(); i!= c.end();) { 
        if (badValue(*i)) {
            logFile << "Erasing " << *i; // 写日志文件
            c.erase(i++); 
        } else
            ++i;    
    }

  2. 对连续内存容器(vector、deque或string)，使用循环，由于失效问题，要获取erase返回值

    for (SeqContainer<int>::iterator i = c.begin(); i != c.end();){
    if (badValue(*i)){
        logFile << "Erasing " << *i;
        i = c.erase(i);        // 通过把erase的返回值赋给i来保持i有效
    } else
        ++i;
}
    • 由于标准序列容器的erase会使被删元素之后的所有迭代器失效，所以要取得返回值进行操作

  3. 对list，两种办法皆可

#四、总结

  1. 去除一个容器中有特定值的所有对象：
    • 如果容器是vector、string或deque，使用erase-remove惯用法。
    • 如果容器是list，使用list::remove。
    • 如果容器是标准关联容器，使用它的erase成员函数。
  2. 去除一个容器中满足一个特定判定式的所有对象：
    • 如果容器是vector、string或deque，使用erase-remove_if惯用法。
    • 如果容器是list，使用list::remove_if。
    • 如果容器是标准关联容器，使用remove_copy_if和swap，或写一个循环来遍历容器元素，当你把迭代器传给erase时记得后置递增它。
  3. 在循环内做某些事情（除了删除对象之外）：
    • 如果容器是标准序列容器，写一个循环来遍历容器元素，每当调用erase时记得都用它的返回值更新你
      的迭代器。
    • 如果容器是标准关联容器，写一个循环来遍历容器元素，当你把迭代器传给erase时记得后置递增它
