title: 条款14 使用reserve来避免不必要的重新分配
date: 2016-05-13 17:02:05

#一、容器的自动增长

  1. 只要不超过它们的最大大小(通过max_size()获取)，就可以自动增长
  2. 对于vector和string，以realloc等价的思想来增长：
    • 分配新的内存块，比容器目前容量大几倍。在大部分实现中，vector和string的容量每次以2为因数增长。也就是说，当容器必须扩展时，它们的容量每次翻倍
    • 把所有元素从容器的旧内存拷贝到它的新内存
    • 销毁旧内存中的对象
    • 回收旧内存
  3. 增长的代价很昂贵
    • 性能开销
    • 所有迭代器、指针和引用都会失效

#二、reserve成员函数

  1. reserve成员函数允许最小化必须进行的重新分配的次数，因而可以避免真分配的开销和迭代器/指针/引用失效

  2. 只有vector和string提供的成员函数：

    • size()：容器当前元素数量，而非容量
    • capacity()：容器容量，即总共可以容纳元素数量。如果size和capacity返回同样的值，容器中就没有剩余空间了，而下一次插入（通过insert或push_back等）会引发上面的重新分配。
    • resize(Container::size_type n)：修改当期元素数量。调用resize之后，size将会返回n。如果n小于当前大小，容器尾部的元素会被销毁。如果n大于当前大小，新默认构造的元素会添加到容器尾部。如果n大于当前容量，在元素加入之前会发生重新分配。
    • reserve(Container::size_type n)：修改容器容量。提供的n不能小于当前大小。这一般强迫进行一次重新分配，因为容量需要增加。（如果n小于当前容量，vector忽略它，这个调用什么都不做，string可能把它的容量减少为size()和n中大的数，但string的大小没有改变。)

  3. 只要有元素需要插入而且容器的容量不足时就会发生重新分配，以及所带来的迭代器失效。所以，避免重新分配的关键是使用reserve尽快把容器的容量设置为足够大，最好在容器被构造之后立刻进行

    • 避免重分配：

      vector<int> v;
v.reserve(1000);
for (int i = 1; i <= 1000; ++i) v.push_back(i);

    • 避免迭代器失效：

      string s;
...
if (s.size() < s.capacity()) {
    s.push_back('x');
}

#三、应用
通常有两情况使用reserve来避免不必要的重新分配:

  1. 当确切或者大约知道容器元素可能的最大数量，只需要提前reserve适当数量的空间(就像上面的vector代码)
  2. 保留你可能需要的最大的空间，一旦添加完全部数据，修整掉任何多余的容量
