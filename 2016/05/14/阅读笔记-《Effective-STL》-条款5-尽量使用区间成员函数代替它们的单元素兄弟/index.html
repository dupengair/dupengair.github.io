条款5 尽量使用区间成员函数代替它们的单元素兄弟

May 14, 2016 | 阅读笔记《Effective_STL》 | Hits
文章目录

  2. 2. 二、区间成员函数

  5. 5. 五、区间函数
  6. 6. 六、总结

一、问题

  1. 问题

    • 给定两个vector，v1和v2，使v1的内容和v2的后半部分一样的最简单方式：

      v1.assign(v2.begin() + v2.size() / 2, v2.end());

  2. assign方法对于所有标准序列容器（vector，string，deque和list）都有效

二、区间成员函数

  • 区间成员函数是一个像STL算法的成员函数，使用两个迭代器参数来指定元素的一个区间来进行某个操作

三、尽量避免手写显式循环

  1. 效率低下

  2. 使用算法代替

    v1.clear();
copy(v2.begin() + v2.size() / 2, v2.end(), back_inserter(v1));
    • 涉及到的工作也比区间成员函数更多
    • copy算法中隐含了一个循环
    • 语义晦涩，这里是要插入元素

  3. 使用insert的区间版本代替：

    v1.clear();
v1.insert(v1.end(), v2.begin() + v2.size() / 2, v2.end());
    • 语义比copy明确

四、几乎所有目标区间被插入迭代器指定的copy的使用都可以用调用的区间成员函数的来代替

  1. 可以输入更少的代码，避免错误
  2. 代码更清晰，语义更明确
  3. 对于标准序列容器，单元素成员函数比区间成员函数需要
    + 更多地内存分配、移动
+ 更频繁地拷贝对象
+ 造成多余操作
  4. 效率
    • 多次函数调用(insert)
    • 内存移动与拷贝(调整元素位置)

五、区间函数

  1. 区间构造

    • 所有标准容器都提供这种形式的构造函数：

      container::container(InputIterator begin, // 区间的起点
InputIterator end);                         // 区间的终点

  2. 区间插入

    • 所有标准序列容器都提供这种形式的insert：

      void container::insert(iterator position, // 区间插入的位置
InputIterator begin,                       // 插入区间的起点
InputIterator end);                       // 插入区间的终点

    • 某些形式的单元素插入：

      • push_front和push_back都把单元素插入容器
      • 一个循环调用push_front或push_back，或如果一个算法——比如copy——的参数是front_inserter或者back_inserter，就应该将insert的区间形式应该作为优先策略的地方

  3. 区间删除

    • 每个标准容器都提供了一个区间形式的erase

    • 但是序列和关联容器的返回类型不同，标准认为关联容器版本返回一个迭代器（被删除的那个元素的下一个）会招致一个无法接受的性能下降

      • 序列容器提供：

        iterator container::erase(iterator begin, iterator end);

      • 关联容器提供这个：

        void container::erase(iterator begin, iterator end);

  4. 区间赋值

    • 所有标准列容器都提供了区间形式的assign：

      void container::assign(InputIterator begin, InputIterator end);

六、总结

  • 区间成员函数更容易写，它们更清楚地表达你的意图，而且它们提供了更高的性能。那是很难打败的三驾马车
