#一、STL容器

  1. STL框架是开放-封闭原则的极佳范例——对扩展开放，对修改封闭
  2. 使容器符合STL规范的三种方法：
    • 侵入式：直接提供STL所需接口
    • 非侵入式：提供特殊迭代器作为算法的接口，实现遍历所有元素的能力
    • 包装：提供wrapper来包装数据，呈现与STL类似的接口

#二、String作为STL容器

  1. String可看作字符构成的序列，提供了STL的接口
  2. 可以对单个字符采用STL算法

#三、C-Style Array作为STL容器

  1. 不存在任何成员函数，只能采取非侵入式或者包装
  2. 对于非侵入式作法，只需要可以通过STL迭代器接口遍历array所有元素的对象，即pointer
  3. pointer可以当做随机访问迭代器，体现了STL抽象的泛化概念：任何东西只要行为类似迭代器，就是一种迭代器
  4. C++11前必须传递raw pointer给算法，因为没有全局begin()和end()，注意确保算法中区间尾端是最末元素的下一位置

#四、实现reference语义

  1. STL容器提供的都是value语义，而非reference语义，构建了元素的内部拷贝，任何操作返回的也是这些拷贝
  2. 要实现reference语义，需要使用smart pointer

  3. 可以使用std::reference_wrapper<>来保证，只要容器存在，被指向的元素一定存在，优点是不需要pointer语法，但需要以第一元素之间调用成员函数时，get()是必须的

    class MyClass {
public:
    getName();
...
};
std::vector<std::reference_wrapper<MyClass>> coll;
for(const auto& MyClass : Iterm)
    std::cout << Iterm.get().getName() << std::endl;

#五、各种容器的使用

  1. 默认选vector，简单高效、允许随机访问，内存整块
  2. 头尾安插移除选deque，内存多块，容量更大，能自动缩减大小
  3. 中段安插删除选list，移动元素常量时间，不支持随机访问
  4. 以节点为基础的容器，list保证不失效，vector和deque在大小改变时会失效
  5. 对异常要求“操作不成功无任何作用”，用list，元素比较过程中可能抛出异常时不要调用merge remove remove_if unique
  6. 需要查找准则时，不关心顺序用unordered set/multiset，关心顺序用set/multiset
  7. 处理关系型数值或者关联数组时，不关心顺序用unordered map/multimap，关心顺序用map/multimap
  8. 处理字典时，不关心顺序用unordered multimap，关心顺序用multimap
  9. 需要多个容器共享元素，实现不同排序准则时，可以通过reference语义实现
  10. 关联式容器可以自动排序，但不意味着效率高，因为每安插一个元素就要排序一次，效率不如序列式容器的做法：先安插所有元素，然后进行一次完全排序
