#一、Forward List

  • 采用singlely linked list管理元素，相比list内存占用少，略快速

    namespace std {
    template <typename T, typename Allocator = allocator<T>>
    class forward_list;
}

#二、能力

  1. 与list不同：
    • 只提供前向迭代器，不支持双向或反向迭代器，所以不提供如reverse_iterator、rbegin()、rend()、crbegin()、crend()之类的成员函数
    • 不提供size()操作，需要知道大小时，要换成list
    • 没有指向最末元素的anchor，所以不提供处理最末元素的方法，如back()、push_back()/pop_back()
    • 提供特殊版本的安插和删除元素操作。成员函数都以_after的形式提供，如insert_after()，注意正常的insert操作都是在pos之前插入
    • 提供before_begin()和rbefore_begin()，会产生出第一个元素之前的一个虚拟位置，用来配合_after的算法使用

#三、操作

  1. 不提供size()操作，因为不存储元素数量，无法在常量时间内计算得到。需要通过destance方法计算得到，具有线性复杂度

    std::forward_list<int> l = {1， 2， 3};
std::distance(l.begin(), l.end());

  2. 只提供front()来直接访问元素

  3. 所有迭代器都是Forward迭代器，不支持reverse迭代器，因此无法调用需要双向或者随机迭代器的算法，特别是排序，提供自身的成员函数sort()进行排序
  4. 提供before_begin()、cbefore_begin()的目的是为了能改动第一元素，除了复制和赋值，对其有效的操作符只有++、==和!=
  5. 提供成员函数的remove()算法，比STL的remove算法更快，因为只操作pointer
  6. 所有的安插、替换与擦除操作，需要获取元素pos，在该pos上操作时，由于无法从该pos返回到前一元素（不提供反向list的pointer）进而改变前一pointer的指向，所以只能改变当前pos元素的指向，也就只能在当前pos后进行操作，所以这些操作都以_after为后缀。

  7. 对于_after函数的操作，可以结合before_begin()，但不能传入end()，否则会导致不明确行为，因为end()不能代表最末元素的位置

    l.insert_after(l.before_begin(), {4, 5, 6});        // 4 5 6 1 2 3
l.insert_after(l.end(), {4, 5, 6});            // Error!

  8. 对于别的容器安插、替换与擦除是在指定位置之前，对于Forward list的_after操作是在pos之后，如果要在之前操作，就要保留上一元素的位置，对于C++11可以使用提供的next操作获得下一元素

    // C++98
auto posBefore = l.before_begin();
for (auto pos=l.begin(); pos!=l.end(); ++pos, ++posBefore)
    if (*pos == 2)
        break;
l.insert_after(posBefore, 100);

// C++11
auto posBefore = l.before_begin();
for (; next(posBefore)!=l.end(); ++posBefore)
    if (*next(posBefore) == 2)
        break;
l.insert_after(posBefore, 100);

  9. 可以在常量时间移除、安插元素，元素在容器间的移动效率很高。splice_after()取代了splice()，因为传入的是起作用的前一元素pos
    10、splice_after操作的来源和目的可以相同，可以在同一个容器内搬运元素，但绝不能传入end()
