#一，C++（编译器）会为类默认生成的函数

  1. 构造函数
  2. 析构函数
  3. 拷贝构造函数
  4. 重载赋值运算符=

#二、关于析构函数

  • 除非基类声明有virtual的析构函数，编译器自动生成的析构函数为non-virtual

#三、无法生成默认重载赋值运算符的情况

  1. 类含有const或者reference成员，因为这些成员不允许改动，所谓编译无法成bit-to-bit copy行为的默认重载赋值运算符=

  2. 如果基类的重载赋值运算符=为private，那么编译器无法为派生类生成默认重载赋值运算符=，因为无法处理基类重载赋值运算符的操作

    如：
class Name{
    public:
        Name(string &name, const int &value) {};
        ……
    private:
        string &name;        //引用类型成员，无法生成默认重载赋值运算符=
        const int value;    //const类型成员，无法生成默认重载赋值运算符=
    }
