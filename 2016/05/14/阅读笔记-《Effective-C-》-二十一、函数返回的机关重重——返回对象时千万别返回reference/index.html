title: 二十一、函数返回的机关重重——返回对象时千万别返回reference
date: 2016-05-13 17:02:02
categories: [阅读笔记, 《Effective C++》]
tag: 《Effective C++》
toc: true

  • 函数返回对象时，千万别返回reference，谨记！(除了赋值操作符，实现连续赋值)
  • 函数创建新对象的途径有二：在stack空间或者heap空间

#一、返回stack上的local对象——Bomb!

  1. 地球人都知道，不能返回一个stack上分配的local对象。定义一个local变量，即在stack空间创造对象，返回这样对象的reference，后果将是灾难性的.

    如：
const Rational& operator* (const Rational& lhs, const Rational& rhs)
{
    Rational result( lhs.n * rhs.n, lhs.d * rhs.d ); 
    return result;                        //Bomb!
}

//local对象在函数退出前就销毁了！

#二、返回heap上的local对象——Bomb!

  1. 这个颇具迷惑性，主要问题是谁该对new出来的对象负责delete，看看具体应用就知道了。

    如：
const Rational& operator* (const Rational& lhs, const Rational& rhs)
{
    Rational* result = new Rational( lhs.n * rhs.n, lhs.d * rhs.d ); 
    return *result;                        
}

  2. 到这里还没什么问题，但看看如下的应用：

    Rational w, x, y, z;
w = x * y * z;            //Bomb!
    • 这里同一条语句内调用了两次operator*，因而两次使用new，也就需要两次delete。但这里没有合理的办法来调用delete，第二次调用operator*返回的时候reference的指向已经发生变化，造成资源泄漏。

#三、返回heap上的static local对象——Bomb!

  1. 如果这里可以再改进一下的话，可能会考虑返回heap上的static local对象，这样就不会产生多次构造，从而避免多次分配资源。

    如：
const Rational& operator* (const Rational& lhs, const Rational& rhs)
{
    static Rational result; 
    result = ……;      //用lhs* rhs的结果来初始化这个static对象 
    return result;                        
}

  2. 除了多线程安全性意外，还有重大的bug在应用中体现出来。

    如：
bool operator==(const Rational& lhs, const Rational& rhs);    //重载operator==
Rational a, b, c, d;

if( (a * b) == (c * d))        //Bomb!
{
    ……        // ab与cd的乘积相等时
}
else
{
    ……        // ab与cd的乘积不等时
}

    • 如上的函数调用可以转化为如下的形式：

      if( operator== ( operator*(a, b), operator*(c, d) ) )

    • 可以看到，这里两个operator*的调用完毕以后才会去调用operator==，而由于operator*内部返回的是一个local static对象，无论这两个operator*的调用孰先孰后，最终的返回的结果，都是operator*的实现内部那个local static对象的最终值，因此，这里的==永远成立，operator==返回始终为真。

#四、返回新对象的正确做法

  1. 返回一个新对象永远是正确的做法，即使要承受返回值的构造成本和析构成本。
  2. 由于编译器的优化，某些情况下返回值的构造成本和析构成本会被消除。（返回值优化）

  3. 绝对不要返回pointer或者reference指向一个local stack对象，或返回reference指向一个heap-allocated对象，或者返回一个local static对象而有可能同时需要多个这样的对象。

    正确做法：
inline const Rational operator* (const Rational& rhs, const Rational& rhs)
{
    return Rational( lhs.numerator() * rhs.numerator(),
             lhs.denominator() * rhs.denominator() );
}
