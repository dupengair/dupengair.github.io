八、了解各种不同意义的new和delete

一、new operator

sting *ps = new string("Memory Management");
  1. 这是个语言内建的操作符
  2. 实施两个动作：
    • 分配足够的内存用来放置对象
    • 调用一个constructor初始化这个对象
  3. 这个操作符不能重载，只能改变上面的动作1)，即重载operator new。
  4. 如果想要一个heap-based object，一定要使用new operator，不能直接调用constructor。

二、operator new

void* operator new(size_t size);
  1. new operator 调用 operator new 分配内存。
  2. 可以重载operator new 时，加上额外参数，但第一参数的类型必须为 size_t

  3. operator new 只负责内存分配，取得operator new 返回的内存并将其转换为一个对象是new operator的责任

    string *ps = new string("Memory Management");
等价于：

void* memory = operator new(sizeof(string));        //取得原始内存
call string::string("Memory Management") on *memory    //初始化内存中的对象
string *ps = static_cast<string *>(memory);            //返回内存地址上的对象

三、Placement new

Widget *pw = new (buffer) Widget(sizeof(Widget));
  1. 当程序运行到shared memory 或 memory-mapped I/O，这类函数可能是由于的，因为对象必须作用于特定的地址，或者是特殊函数分配出的地址上

  2. 因为调用者知道对象应放在哪里，所以placement new唯一需要做到的就是讲它获得的指针再返回。它的函数原型如下：

    void* operator new(size_t, void *location)
{
    return location;
}

四、运用

  1. 如果想要对象产生于heap，请使用new operator。
  2. 如果只打算分配内存，请使用operator new
  3. 如果打算在heap object产生时自己决定内存分配方式，请编写一个自己的operator new。
  4. 如果打算在自己分配的内存中构造对象，请使用placement new

五、删除（deletion）与内存释放（deallocation）

string *ps;
……
delete ps;
的执行过程如下：

ps->~string();
operator delete(ps);
  1. 对于delete operator，它必须析构地址上的对象，并且释放该对象所占内存。

  2. 内存释放函数由operator delete执行，声明如下：

    void operator delete(void *memory)

  3. 如果只打算处理未初始化的原始内存，应该完全回避new/delete operator，调用operator new取得内存并以operator delete归还系统，这种行为相关于在C中调用malloc和free。

  4. 如果使用placement new 在某内存块中产生对象，应该避免对该内存使用delete operator。因为delete operator会调用operator delete来释放内存，但是该内存内含的对象最初并非由operator new分配，placement new只是返回它所接收的指针而已。所以要抵消placement new 对该对象调用constructor的应该，应该直接调用该对象的destructor。

六、数组

  1. 内存不再以operator new分配，而是operator new []函数，也叫array new。
  2. 针对数组中的每个对象调用一个constructor。
