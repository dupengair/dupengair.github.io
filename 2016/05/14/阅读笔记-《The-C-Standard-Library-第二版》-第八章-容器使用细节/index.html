#细则：

  1. 检查容器是否为空时，应使用empty，比size更有效率
  2. max_size的结果可能取决于容器的内存模型，由于vector通常使用单一内存区段，所以这个值往往小于其它值
  3. unordered容器的相等，除元素数量和内容，还必须包括次序，其==操作在最坏情况下可能是二次复杂度
  4. unordered容器不提供大小比较<>操作
  5. 当不需要被赋值的对象时，swap比copy assignment更有效率
  6. array在swap时要交换的不止pointer，所以是线性复杂度，其它容器swap时只交换pointer，所以是常量复杂度
  7. 只有Array、Vector、Deque、String支持随机访问迭代器

#关于元素插入：

  1. insert(value)、insert(init-list)、insert(begin, end)：指定value的插入只支持关联和unordered容器，因为可以自动排序
  2. insert(pos, value)：指定pos插入支持所有容器，vector和deque可能会导致iterator和reference失效，对关联和unordered容器pos只是一个提示
  3. insert(pos, init-llist)、insert(pos, begin, end)：指定位置插入一段数据支持vector、deque、list、string，因为插入一段数据对关联容器和unordered每个元素都要一次排序，效率低下
  4. push_front：支持deque和（forward）list
  5. push_back：支持vector、deque、list、string

#关于元素移除：

  1. erase(value)、erase(begin, end)：同insert，指定value只支持关联和unordered容器
  2. erase(pos)：同insert，指定pos支持所有容器
  3. erase(begin, end)：支持所有容器
  4. pop_front：同insert，支持deque和（forward）list
  5. push_back：同insert，支持vector、deque、list、string
  6. clear：调用元素的析构函数，移除所有元素，清空容器，支持所有容器

#关于resize：

  1. 支持vector、deque、list、string
