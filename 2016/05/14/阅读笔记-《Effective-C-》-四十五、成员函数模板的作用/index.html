title: 四十五、成员函数模板的作用
date: 2016-05-13 17:02:04
categories: [阅读笔记, 《Effective C++》]
tag: 《Effective C++》
toc: true

智能指针（smart pointers）是指行为像指针的对象，可以提供指针没有的机能。STL容器的迭代器几乎总是智能指针。我们以智能指针为例，讨论成员函数模板的神奇作用。

#一、隐式转换（implicit conversions）

  1. 智能指针的一大特点是对隐式转换的支持，如：

    class Top { …… };
class Middle : public Top { …… };
class Bottom : public Middle { …… };

Top* pt1 = new Middle;            //将Middle*隐式转换为Top*
Top* pt2 = new Bottom;            //将Bottom*隐式转换为Top*    
const Top* pct2 = pt1;            //将Top*隐式转换为const Top*

#二、在智能指针的定义中支持隐式转换
定义一个智能指针的类模板：

template<typename T>
class SmartPtr {
public:
    explicit SmartPtr(T* realPtr);                    //以内置原始指针完成初始化
    ……
};

我们希望能够这样：

SmartPtr<Top> pt1 = SmartPtr<Middle>(new Middle);    //将SmartPtr<Middle>转换为SmartPtr<Top>
SmartPtr<Top> pt2 = SmartPtr<Bottom>(new Bottom);    //将SmartPtr<Bottom>转换为SmartPtr<Top>
const SmartPtr<Top>pct2 = pt1;                        //将SmartPtr<Top>转换为SmartPtr<const Top>
  • 但是，同一个template的不同实例化之间并不存在什么关系，所以对编译器来说SmartPtr<Middle>和SmartPtr<Top>是完全不同的class，如果要实现这样的转换，需要代码级的支持。

#三、构造函数

  1. 要支持向上转型，需要构造函数能接受一个类的派生类，这与模板的实例化产生矛盾。因为实例化的时候生成一个既定的类，此时它不可能知道自己的派生类类型，只有当它被继承时才可能产生派生类类型。
  2. 要解决这个问题，就需要构造一个与模板类型不同的构造函数，接受来自不同类型的参数，称之为泛化（generalized）拷贝构造函数。

  3. 所以我们最终不是写一个构造函数，而是要写一个构造函数模板，作用是为class生成合适类型的构造函数，以支持隐式类型转换：

    template<typename T>
class SmartPtr {
public:
    template<typename U>                    //成员函数模板
    SmartPtr(const SmartPtr<U>& other);        //通过作为参数的SmartPtr对象来构造
    ……
};

  4. 这个构造函数未被声明为explicit，是因为原始指针之间的类型转换就是隐式转换，如derived class pointer转为base class pointer，所以这里应符合其行为。

#四、类型匹配

  1. 通过上面泛化（generalized）拷贝构造函数的定义，已经可以支持隐式类型转换，但得到似乎比想要的多。由于对转换未加限定，上述的定义使得任意类型的SmartPtr都可以转换为当前类型，比如SmartPtr转换为SmartPtr，这不是我们希望的，所以必须对这一成员函数模板（member function template）所创建的函数成员进行筛选。

  2. 智能指针支持get()方法获取其所保持的原始指针，这就要在类中添加保持原始指针的成员，并且在构造函数中进行初始化，可以通过这样的初始化来约束构造函数带来的转换行为：

    template<typename T>
class SmartPtr {
public:
    template<typename U>                
    SmartPtr(const SmartPtr<U>& other)             
        : heldPtr(other.get()) { … }        //用原始指针来初始化，要求能够隐式类型转换
    T* get() const { return heldPtr; }
    ……
private:
    T* heldPtr;                                //保存原始指针
};

  3. 这里对heldPtr的初始化使用了作为拷贝构造函数参数的SmartPtr对象的原始指针，即存在U*->T*的隐式类型转换，只有在原始指针可以转换的情况下，类型匹配才能成功，从而限制了无法转换的类型。

#五、赋值操作
tr1::shared_ptr的定义如下：

template<typename T>
class SmartPtr {
public:
    template<typename Y>                    //构造任何兼容内置指针类型
        explicit shared_ptr(Y* p);
    template<typename Y>                    //泛化（generalized）拷贝构造函数    
        shared_ptr(shared_ptr<Y> const& r);
    template<typename Y>                    //weak_ptr类型
        explicit shared_ptr(weak_ptr<T> const& r);    
    template<typename Y>                    //auto_ptr类型
        explicit shared_ptr(auto_ptr<Y>& r);        
    template<typename Y>                    //赋值给任何兼容的shared_ptr类型    
        shared_ptr& operator=(shared_ptr<Y> const& r);
    template<typename Y>                    //赋值给auto_ptr类型    
        shared_ptr& operator=(auto_ptr<Y>& r);
    ……
};
  1. 泛化拷贝构造函数没有explicit，这是为了从一个shared_ptr类型到另一个shared_ptr类型的隐式类型转换。
  2. 其它构造函数都有explicit，意味着不能从其它类型隐式转换到shared_ptr类型。
  3. auto_ptr类型的参数没有const，是因为对一个auto_ptr的复制操作是破坏性的。

#六、合成成员函数

  1. 当我们没有自行定义时，编译器会为我们合成4个成员函数
  2. 当SmartPtr模板中的T和Y相同时，泛化拷贝构造函数被实例化为正常的拷贝构造函数。但不相同时总是会生成一个non-template的拷贝构造函数。

  3. 上事实说明，要同时声明泛化拷贝构造函数和正常的拷贝构造函数。赋值（assignment）操作符也一样。

    template<typename T>
class SmartPtr {
public:
    shared_ptr(shared_ptr const& r);                //copy构造函数

    template<typename Y>                            //泛化（generalized）拷贝构造函数    
    shared_ptr(shared_ptr<Y> const& r);

    shared_ptr& operator=(shared_ptr<Y> const& r);    //普通的赋值操作符

    template<typename Y>                            //泛化的赋值操作符    
    shared_ptr& operator=(shared_ptr<Y> const& r);    
    ……
};

#七、总结

  1. 使用成员函数模板生成兼容素有（可兼容）类型的函数。
  2. 要同时声明泛化和正常的拷贝构造函数、赋值（assignment）操作符。
