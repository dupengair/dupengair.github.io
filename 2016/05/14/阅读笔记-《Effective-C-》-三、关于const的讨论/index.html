#一、什么是const？

  1. 借助编译器实现对语义约束的指定，编译器的强制实施确保指定的约束不被违反。

#二、const与指针

  1. const出现在*左边，被指物为常量，const出现在*右边，指针为常量

    int a = 10;
const int * p =    &a;   //p指向的a为常量，不能修改被指物的值    
*p = 5;        //error!
p = &b;        //ok

int * const p =    &a;   //p指针自身为常量，不能修改指针的指向    
*p = 5;        //ok
p = &b;        //error!

  2. const 与迭代器

    STL迭代器以指针为根据，类似于 T*

    • 声明迭代器为const等价于声明一个为const的指针（T* const），不能指向不同的被指物，但指向物的内容可以修改。

      std::vector<int>vec;
const std::vector<int>::interator iter = vec.begin();     //等价于 T* const
++iter;        //error!
*iter = 10;    //ok

    • 要想声明一个指向常量（const T*）类型的迭代器，要用

      const_iterator
std::vector<int>::const_iterator iter = vec.begin();
++iter;        //ok
*iter = 10;    //error!

#三、将函数返回值声明为const，如果不需要做左值的话

  1. 这样会防止意外，保证安全性和高效性，如：

    class Rational {};
const Rational operator* (const Rational &lhs, const Rational &rhs);
Rational a,b,c;
if(a * b = c)    //error! 可以避免这样的手误
    • 如果a、b都是内置类型，这样的操作直接不合法，但为自定义类型时，需要避免这样与内置类型不兼容的做法，“良好的自定义类型”。
    • 做参数的情况下，除非要改动参数或者Local对象，否则声明为const
