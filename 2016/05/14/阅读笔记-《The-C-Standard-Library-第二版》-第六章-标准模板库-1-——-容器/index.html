#STL组件

  1. STL组件包括：容器、迭代器、函数对象、算法
  2. 容器表示某类对象的集合，迭代器为所有容器提供了一组很小的共通接口，用来在一个容器内遍历元素。算法使用所有容器和迭代器提供的统一接口来处理容器内的元素。
  3. STL的基本观念是将数据和操作分离。数据由容器类加以管理，操作由算法进行定义。迭代器使得任何算法可以和任何容器交互运作。
  4. STL采用泛型的思想，容器和算法可以被泛化为可适用于任何type和class

#容器

  1. 分类

    • 序列式容器: 有序集合，位置取决于插入时机和地点，与元素值无关。STL提供5个预定义的sequence容器：array、vector、deque、list、forward_list
    • 关联式容器: 已排序集合，位置取决于元素值，和插入顺序无关。STL提供4个预定义的associative容器：set、multiset、map、multimap
    • 无序容器: 无序集合，元素值或安插顺序不影响位置，元素位置可能在生命周期中改变。STL提供4个预定义的unordered容器：unordered_set、unordered_multiset、unordered_map、unordered_multimap

  2. 实现

    • sequence容器——通常被实现为array或linked list
    • associative容器——通常被实现为binary tree
    • unordered容器——通常被实现为hash table

#一、序列式容器

  1. Vector

    • 允许随即访问，在尾部添加或移除元素很迅速。（当需要重新分配内存，将现在元素拷贝到新位置时会很慢，这是折旧成本）
    • 所有的序列式容器都可以通过push_back()在尾部附加新元素，因为这永远是可能的，而且效率相当高
    • 通过size()返回容器元素的个数，通过下标操作符[]访问vector的元素

  2. Deque

    • double ended queue，可以向两端发展，在头尾安插元素都很迅速
    • push_front()在头部，push_back()在尾部
    • Vector不提供push_front()是因为时效差，需要移动所有元素。一般而言，STL只提供时效性好的操作，其复杂度为常数或者对数

  3. Array

    • 大小固定，在建立时指定，不能改变元素个数，只能改变值
    • 元素个数是array类型的一部分，类型不同不能比较，如array和array

  4. List

    • 有双向链表实现，每个元素指向前导和后继元素
    • 不提供随机访问，访问时间和元素数量成正比，任何位置上安插和删除元素都很迅速
    • 使用range based for时，如果希望在不需要改变值时不产生copy，应该使用指向const reference：
      for(const auto& elem : coll)
    • front()返回第一个元素，pop_front()删除第一个元素，但不会返回

  5. Forward List

    • 单一链表，为了节省内存，只指向下一个元素
    • 不提供push_back或者size()
    • 无法查找某个元素然后删除，或者在它前面安插元素，必需位于前一个元素的位置
    • 一般定义后使用resize()插入元素，代价昂贵

#二、关联式容器

  1. 元素自动排序，默认采用<，可以提供比较函数。排序准则也用来测试等同性，如果两个元素的key/value互不小于对方则相等
  2. 由二叉树实现，每个元素有一个父节点两个子节点，左字数的所有元素都比自己小，右子数比自己大
  3. 能很快找到某个特定value的元素，具备对数复杂度，但不能直接修改元素的value，因为会破坏自动排序
  4. STL的关联容器
    • Set：元素根据value自动排序，每个元素只能出现一次，不能重复
    • Multiset：可以重复
    • Map：键值对，不能重复，key是排序准则
    • Multimap：元素可以重复，可被当做字典使用
  5. 对于key相同的元素，C++11保证，新插入的元素在multiset和multimap的有效元素末尾

#三、无序容器

  1. 元素无明确次序，只关心存在性，访问速度可能快过关联式容器
  2. 通常以hashtable实现，函数函数目标是让元素位置有利于快速访问
  3. hash函数可能让多个元素位于同一位置，以linked list存放一个位置上的多个元素
  4. 容器同关联式
  5. 可以通过template实参指明hash函数和等效准则，以寻找给定值并判断是否发生重复，默认为==
  6. C++standard保证，删除元素不会引发rehashing，但删除之后的一次安插可能引发

#四、关联式数组

  1. map或unordered map的元素都是key/value组合，每个元素带着唯一的key，可以被视作associative array，其索引非整数
  2. 可以使用下标操作符[]访问元素，当key不存在时会添加新元素
  3. 从C++11开始可以使用at()访问元素，当key不存在时会抛出out_of_range异常
  4. 对于unordered map来说操作元素为常量复杂度，对map来说为对数复杂度，所以除非需要排序，首选unordered

#五、其它容器

  1. String：类似Vector，元素都是字符
  2. C-style array：不是STL容器，不提供成员函数，但可以应用算法。Vector和array提供了所有特性，并具备更安全的接口。

#六、容器适配器

  1. 实现特殊功能的容器
    • stack：LIFO，后进先出
    • queue：FIFO，先进先出
    • priority queue：基于优先权，当优先权相等时顺序无明确定义
