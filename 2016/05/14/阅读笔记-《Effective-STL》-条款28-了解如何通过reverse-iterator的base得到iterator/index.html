title: 条款28 了解如何通过reverse_iterator的base得到iterator
date: 2016-05-13 17:02:07
categories: [阅读笔记, 《Effective STL》]
tag: 《Effective STL》
toc: true

#一、示例

vector<int> v;
v.reserve(5); 
for(int i = 0；i < 5; ++ i) {         // 向vector插入1到5
    v.push_back(i);
}
vector<int>::reverse_iterator ri = find(v.rbegin(), v.rend(), 3); // 使ri指向3
vector<int>::iterator i(ri.base());    // 使i和ri的base一样

* 上面代码执行后，结果如图：

         rend()        ri       rbegin()
              |         |       |
                1    2    3    4    5
                |           |     |
                begin()     i     end()

#二、插入元素

  1. 要实现在一个reverse_iterator的ri指出的位置上插入新元素，在ri.base()指向的位置插入就行了。
  2. 对于insert操作而言，ri和ri.base()是等价的，而且ri.base()真的是ri对应的iterator

#三、删除

  1. 不能直接使用i，因为i与ri不是指向同一个元素，要删除的是i的前一个元素

  2. 要实现在一个reverse_iterator的ri指出的位置上删除元素，就应该删除ri.base()的前一个元素。对于删除操作而言，ri和ri.base()并不价，而且ri.base()不是ri对应的iterator

    v.erase(--ri.base()); // 尝试删除ri.base()前面的元素 对于vector，一般来说编译不通过

  3. 但对于大多数vector和string的实现，无法通过编译，因为iterator（和const_iterator）会采用内建的指针来实现，所以ri.base()的结果是一个指针，C和C++都规定了不能直接修改函数返回的指针

  4. 所以应该尽量避免修改base的返回值，只需要先增加reverse_iterator的值，然后再调用base

    v.erase((++ri).base()); // 删除ri指向的元素；
