title: 关于多态的思考2：虚函数的重载（overload）与改写（override）
date: 2016-05-13 17:01:51
categories: C&C++经验累积
tag: C&C++经验累积
toc: true

今天通过两个实例测试对基类虚函数重载的多态调用方式：

  • 实例一，重载（overload）基类的虚函数：

    class product
{
public:
    product(char * name);
    virtual ~product();
    virtual void display3(int ID);    
};

class product_A : public product
{
public :
    product_A(char * name = "no name A");
    ~product_A();
    virtual void display3();   //重载overload基类的display3()虚函数 
};

int main()
{   
   product *p=new product_A("pdt1");
   p->display3(321);           //调用基类的display3(int ID)
   p->display3();              //error! 基类无void display3()函数定义
   p->product_A::display3();     //error! 基类无void display3()函数定义

   product_A aa("pdt2");
   aa.display3(321);           //error! 派生类无display3(int ID)函数定义
   aa.display3();              //调用派生类的display3()
   aa.product_A::display3();     //调用派生类的display3()

   return 0;
}

  • 实例二，改写（override）基类的虚函数：

    class product
{
public:
    product(char * name);
    virtual ~product();
    virtual void display4();       
};

class product_A : public product
{
public :
    product_A(char * name = "no name A");
    ~product_A();
    virtual void display4();   //改写override基类的display4()虚函数 
};

int main()
{   
   product *p=new product_A("pdt1");
   p->display4();             //调用派生类的display4()
   p->product::display4();     //调用基类的display4()

   return 0;
}
  • 从上面两个实例看出，多态的实现，要求基类和派生类实现同一个虚函数，即派生类改写（override）基类的虚函数，此时的派生类对象内存中同时存在基类和派生类的虚函数，基类指针可同时访问这两个虚函数，通过作用域标识符来区分，如实例二：基类指针指向派生类对象时，默认调用派生类的虚函数，除非显示指明基类作用域。
  • 而当派生类重载（overload）基类的虚函数时，基类指针指向派生类对象，却无法实现多态（实例一），基类指针只能访问派生类对象内存中的基类虚函数，而派生类中重载的虚函数被认为只能被派生类对象访问，两个是不一样的函数，由此可见无法实现多态。

结论：

多态的实现，需派生类改写（override）基类的虚函数，这样基类指针才能同时访问基类和派生类的虚函数，从而实现多态。而如果派生类重载（overload）基类的虚函数，基类指针只能访问派生类对象内存中的基类虚函数，也就是说无法根据指向的派生类对象不同而动态绑定不同对象的虚函数，无法实现多态。
