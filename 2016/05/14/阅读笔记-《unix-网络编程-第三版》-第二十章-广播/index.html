#一、概述

  1. 寻址方式

    类型  IPv4  IPv6  TCP UDP  标识接口数  递送到接口数
单播    Y     Y    Y   Y       1个     1个     
任播          Y        Y      一组     一组中的一个
多播   可选    Y        Y      一组     全组 
广播    Y              Y       全体       全体
    • TCP只支持单播，UDP和原始套接字支持别的
    • 任播从一组主机(通常提供相同服务)中选一个(通常按照某种测度离源主机最近)
    • IPv4可选多播，支持广播，IPv6必需支持多播，不支持广播
    • 广播的用途：
      • 资源发现，在本地子网定位一个服务器主机
      • 减少网络环境中的分组流通
    • 广播的应用：
      • ARP：使用链路层广播，通过IP找MAC
      • DHCP：客户端广播自己的请求(255.255.255.255)
      • NTP：服务器主机广播当前时间，避免客户端自己的轮询
      • 路由守护进程routed：广播路由表，使附近的路由器或者主机可以接收通告，以更新自己的路由表

#二、广播地址

  1. 记法{子网ID， 主机ID}
    • 子网ID表示子网掩码覆盖的连续位
  2. 子网定向广播地址
    • {子网ID，-1}，如192.168.1.100/24对应为192.168.1.255
    • 最好不要依赖转发子网定向广播进行通信，路由器设置程序允许转发此广播包可能造成放大攻击
  3. 受限广播地址
    • {-1, -1}，255.255.255.255
    • 路由器从不转发
    • 主机发送时将该地址转换成定向子网广播地址
    • 如果发送端有多个接口，可能只在第一个接口上发，如果要每个接口都发送，应该首先获取每个接口的配置，然后对接口所在的子网执行sendto调用，发送定向广播

#三、广播与多播

  1. 以太网帧类型字段，IPv4为0x0800，IPv6为0x86dd
  2. 主机发送时，将广播地址映射成48位全1的以太网地址: ff:ff:ff:ff:ff:Ff
  3. 主机收到广播但UDP未绑定对应端口时，不能发送ICMP端口不可达消息，以避免广播风暴
  4. 广播分组到达子网上所有主机，包括发送主机自身(一般来说，发送主机不用绑定目的端口，不然数据会返回到应用层)
  5. 不关心广播的所有主机沿协议栈相识完整处理UDP广播包，直到被UDP层丢弃
  6. 子网上所有的非IP主机在链路层收到完整的帧，然后丢弃

#四、编码

  1. 必需通过设置套接字SO_BROADCAST属性，显式告诉内核发送广播报
  2. 源自Berkeley的实现不允许对广播报进行分片，超过MTU的数据报发送调用会返回EMSGSIZE错误，但Linux不具有此限制
  3. 为便于移植，发广播报之前使用ioctl的SIOCGIFMTU命令设置MTU大小，如果是局域网可以限制在1472

#五、竞争状态

  1. 当多个进程访问共享数据，其结果依赖于程序的执行顺序时，产生竞争状态
  2. 当信号产生时处于被阻塞期间，内核会记录信号但不递交，直到信号被解除阻塞

  3. 如果要用alarm的方式，用信号通知程序，正确的姿势是用pselect以保证信号阻塞与递交间的原子操作，如NTP程序

    const int on = 1;
setsockopt(sockfd, SOL_SOCKET, SO_BROADCAST, &on, sizof(on));

sigset_t newmask, oldmask, zeromask;

sigemptyset(&zeromask);
sigemptyset(&newmask);
sigaddset(&newmask, SIGALRM);

signal(&newmask, alarm_func);

// 阻塞SIGALRM并等待接收
sigprocmask(SIG_BLOCK, &newmask, NULL);
alarm(5)

// 调用pselect时zeromask代替现有屏蔽字(newmask),直到返回
fd_set rset;
FD_ZERO(rset);
FD_SET(sockfd, &rset);
int n = pselect(sockfd+1, &rset, NULL, NULL, NULL, &zeromask);
if(n < 0) { 
    if(error == EINTR) {
        break;
    }
    ...
} else if(n != 1) {
    printf("pselect error! We only have 1 fd");
} else {
    // it is safe to recvfrom without block
    char buf[1024];
    struct sockaddr* replyaddr;
    int len;
    int m = recvfrom(sockfd, buf, sizeof(buf), replyaddr, &len);
}

* 当pselect被调用时，先以空集zeromask替代进程的信号掩码(不阻塞任何信号，可以捕获SIGALRM)，再检查描述符并可能进入睡眠。pselect返回时，进程的信号掩码又被重置为调用pselect之前的值(阻塞SIGALRM)，从而避免了SIGALRM信号的丢失
