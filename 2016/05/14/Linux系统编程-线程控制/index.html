Linux 3.1 实现 NPTL Native POSIX Thread Library

#一、线程属性

detached    分离
guardsize    栈尾警戒区
stackaddr    最低栈地址（注意生长方向，可能是Max或者Min）
stacksize    栈大小

  1. 分离

    • 启动后：pthread_detached
    • 启动前：pthread_attr_setdetachstate/pthread_attr_getdetachstate

  2. 栈

    • 地址：

      pthread_attr_setstack/pthread_attr_getstack

    • 大小：

      pthread_attr_setstacksize/pthread_attr_getstacksize

    • 减小：线程多，VM不够用

    • 增大：变量多，函数栈深
    • 最低地址 ！= 起始地址 -> 生长方向

  3. 警戒区

    pthread_attr_setguardsize/pthread_attr_getguardsize
    • 栈尾之后避免溢出的扩展内容，默认page_size(sysconf(__SC_PAGESIZE))
    • 手动设置后会使guardsize失效
    • 线程栈帧到达警戒区后系统发送信号到进程

  4. 并发与内核线程模型

    • 1对1：真实并发，切换开销大
    • 1对多：开销小，易阻塞
    • 多对多：并发度

#二、同步

  1. mutex
    共享：

    pthread_mutexattr_setpshared/pthread_muteattr_getpshared
线程间：PTHREAD_PROCESS_PRIVATE
进程间：PTHREAD_PROCESS_SHARED

    类型：

    pthread_mutexattr_settype/pthread_muteattr_gettype
Native：重复锁导致死锁
ErrorCheck：
Recursive：可多次锁
Default：打酱油君

    注意：使用同步条件时mutex不能为Recursive，可能因为pthread_cond_wait返回时会加锁

  2. rwlock 和 condition只支持共享属性的设置

  3. 死锁
    自锁和互锁
    检测：timed_lock

#三、私有数据

  1. errno

  2. usage

    #define TDATA_SIZE 1024
static pthread_key_t key;
static pthread_once_t init_done = PTHREAD_ONCE_INIT;
pthread_mutex_t data_mutex = PTHREAD_MUTEX_INITIALIZER;

static void thread_init(void) {
    pthread_key_create(&key, NULL);
}

char* get_thread_data() {
    char* t_data;

    pthread_once(&init_done, thread_init);       //获取key，如果还没有就创建，同时避免竞争
    pthread_mutex_lock(data_mutex);

    t_data = (char *)pthread_getspecific(key);    // 通过key取得数据
    if(!t_data) {                    // get不到时说明还未绑定
        t_data = malloc(TDATA_SIZE);
        if(!t_data) {
            pthread_mutex_unlock(data_mutex);
            return NULL;
        }
        pthread_setspecific(key, t_data);    // 此时要进行绑定
    }

    ...                        // 操作私有数据

    pthread_mutex_unlock(data_mutex);
    return t_data;
}

#四、线程取消

  1. 默认可取消

    PTHREAD_CANCEL_ENABLE
PTHREAD_CANCEL_DISABLE
pthread_setcancelstat
pthread_testcancel    设置取消点

  2. 取消类型

    PTHREAD_CANCEL_DEFERRED        取消点
PTHREAD_CANCEL_ASYNCHRONOUS    任意时间
pthread_setcanceltype

#六、在线程中创建进程

  1. 子进程继承父进程环境，包括锁的状态
  2. 调用exec函数族可以清除旧地址空间

  3. 否则需要调用pthread_atfork清除锁状态

    int pthread_atfork(void (*prepare)(void), void(*parent)(void), void(*child)(void));

prepare：fork创建子进程前调用，获取父进程的锁，创建子进程后，父子进程都获得了锁（子进程是副本）
parent：fork返回前在父进程中解锁
child：fork返回前在子进程中解锁
