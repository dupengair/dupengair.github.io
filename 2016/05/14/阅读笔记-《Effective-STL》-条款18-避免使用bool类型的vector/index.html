#一、vector<bool>的两个问题

  1. 它不是一个STL容器
  2. 它并不容纳bool

#二、作为STL容器

  1. 一个东西要成为STL容器就必须满足所有在C++标准23.1节中列出的容器必要条件

  2. 其中有一条：如果c是一个T类型对象的容器，且c支持operator[]，
    那么以下代码必须能够编译：

    T *p = &c[0];         // 无论operator[]返回什么，都可以用这个地址初始化一个T*
    • 换句话说，如果使用operator[]来得到Container中的一个T对象，可以通过取它的地址而获得指向那个对象的指针

  3. 而对于vector<bool>，下面代码编译不过

    vector<bool> v;
bool *pb = &v[0];    // 用vector<bool>::operator[]返回的东西的地址初始化一个bool*
    • 因为vector<bool>是一个伪容器，并不保存真正的bool
    • 每个元素只有1bit，在内部，vector<bool>使用了与位域（bitfield）等价的思想来表示它假装容纳的bool

  4. 与位域（bitfield）的区别

    • 可以创建指向真的bool的指针，但却禁止有指向单个比特的指针或引用
    • 因此，vector<bool>::operator[]返回一个对象，其行为类似于比特的引用，也称为代理对象

  5. 本质来看，vector<bool>看起来像这样：

    template <typename Allocator>
vector<bool, Allocator> {
    public:
        class reference {...};                 // 用于产生引用独立比特的代理类
        reference operator[](size_type n);    //operator[]返回一个代理
    ...
}

    • 因此如下代码无法编译通过

      vector<bool> v;
bool *pb = &v[0]; // 错误！右边的表达式是vector<bool>::reference*类型，不是bool*

#三、标准库提供的两个替代品

  1. deque<bool>

    • deque提供了几乎所有vector所提供的操作，除了reserve和capacity
    • deque<bool>是一个STL容器，它保存真正的bool值
    • 当然，deque内部内存不是连续的。所以不能传递deque<bool>中的数据给一个希望得到bool数组的C API，但也不能让vector<bool>做这一点，因为没有可移植的取得vector<bool>中数据的方法。

  2. bitset

    • 不是一个STL容器，但它是C++标准库的一部分
    • 与STL容器不同，它的大小（元素数量）在编译期固定，因此不支持插入和删除元素
    • 此外，因为不是一个STL容器，它也不支持iterator
    • 类似vector<bool>，它使用一个压缩的表示法，使得包含的每个值只占用1bit
    • 并且提供vector<bool>特有的flip成员函数，还有一系列其他操作位集（collection of bits）所特有的成员函数
    • 如果不在乎没有迭代器和动态改变大小，也许bitset正合适
