Pair&Tuple

1、make_pair和pair构造函数在使用上有设么差异？（make_pair可忽略类型）
2、如何从tuple里提取元素？可以遍历元素吗？（不能，std::tie或者std::ref+make_tuple）
3、tuple构造上有什么注意事项？请修改错误：（不支持隐式类型转换，用make_tuple或直接构造）
    std::tuple<int, float> t = {1, 1.2};        （t = make_tuple(1, 1.2);）

smart pointer

4、shared_ptr构造上有什么注意事项？请修改错误： （不支持隐式类型转换，直接构造）
    shared_ptr<string> pStr = new string("hello");        （pStr(new string("hello"))）
5、make_shared与shared_ptr在智能指针构造上有什么区别？（前者更快，更安全，一次分配，直接生成shared_ptr对象；后者二次分配，原始对象和shared_ptr对象）
6、下面对shared_ptr的用法有什么错误，如何改正？    （不能对default构造的shared pointer使用assignment操作，只能通过reset方法）
    shared_ptr<string> pStr;
    pStr = new string("hello");        （pStr.reset(new string("hello"));）
7、为什么要使用weak_ptr？    （避免循环引用）
8、shared_from_this的作用？使用上有什么注意事项？（在类的内部传递自身的shared_ptr，要通过shared_ptr对象来生成）
9、下面对unique_ptr的使用有什么错误？         （做函数参数时，需要std::move显示转换为右值，否则因为不支持copy会有编译错误）
    void add(std::unique_ptr<int> up);
    std::unique_ptr<int> unq = new int(1);
    add(unq);                 （add(std::move(up));）
10、auto_ptr有什么缺陷？为何会被C++标准取消？    （copy和assignment操作被设计为move语义，与C++11的move语义需要显式调用std::move矛盾，容易引起误用）
11、c和c++中的NULL有什么区别？（(void*)0，(int)0，c++中不存在*到其它类型间的隐式转换，所以必须为整型才能赋值给指针）
12、C++98中的NULL定义有什么缺陷？（null pointer可能被解释为一个整型）
        f(0);        // call f(int)
        f(NULL);    // call f(int)
        f(nullptr);    // call f(void*)

STL 容器

13、STL组件包含哪些内容？    （容器、迭代器、函数对象、算法）
14、STL处理数据和操作的基本观念是什么，容器、算法和迭代器的作用是什么？（数据和操作分离，容器管理数据，算法定义操作，迭代器使算法和容器相互操作）
15、STL容器有几类？请举例说明？实现上有什么差异？（序列式容器——有序集合，位置取决于插入时机和地点，与值无关，array vector deque lish，用array或list实现，关联式容器——已排序集合，位置取决于值，和插入顺序无关，set map，binary tree实现，无序容器——无序集合，元素值或安插顺序不影响位置，位置可能在生命周期中改变，unordered，hash table实现）
16、简单说明vector、deque、array、list区别？（vector——尾部插入快，访问快，提供随机访问，deque——头尾快，访问稍慢，array——大小固定，是类型的一部分，list——插入快，不提供随机访问）
17、STL对容器元素有什么要求或者必要条件，对每一类容器有什么要求？ （元素必需可复制或者moveable，assignment操作符必需实现赋值或者move语义，必需可析构，序列式容器——具有default构造函数以便创建非空容器，关联式容器——适用于排序准则，无序容器——适用于hash和等同测试）
18、STL容器是value语义还是ref语义？为什么？如何管理原始对象？（value语义，安插原始是基于copy，行为简单不易出错，使用(smart) pointer）
19、STL是否能扩展新的容器、迭代器或算法？其遵循什么准则？如何从STL派生？（能，两个凡是——凡是行为像容器（迭代器）的东西，就是个容器（）迭代器）， STL类不带virtual函数所以不提供多态行为，不能通过继承给容器添加新的行为，正确的方式是应该定义一个新class，内部使用STLclass，或者以private方式继承STLcalss）

迭代器

20、STL迭代器的基本概念是什么？对迭代器的理解？    （以对象来表现容器位置，类型不同的容器提供相同的接口实现不同的操作以达到泛型的目的）
21、begin和end形成半开闭区间好处？（1) 循环结束简单，到end结束 2) 不必对空区间采取特殊处理，begin等于end）
22、循环时++pos和pos++哪个效率更高？ （++pos，pos++会返回临时对象）
23、对非序列式容器，是否可采用push_front()和push_back()？why？（不能，因为元素位置由容器本身排序规则决定）
24、迭代器分哪几类？操作上有什么差别？分别适用于哪些容器？（单向/双向——Forward、unordered/list、associative，input/output——stream，随机访问——算术运算，vector deque array string）
25、下面两种操作方式，哪种更好？why？    （第一种，因为后者只适用于随机访问迭代器）
        for(auto pos = coll.begin(); pos != coll.end(); ++pos)       
        for(auto pos = coll.begin(); pos < coll.end(); ++pos)
26、对下面range based for，有什么改进？   
        vector<std::string> coll;
        for(auto i: coll)
        为了避免对每个元素调用拷贝构造和析构函数，通常应声明当前元素为const reference:
        for(const auto& i: coll)

算法

27、算法是迭代器的成员函数吗？（不是，是全局函数）
28、算法的区间有效指什么？（从起点出发逐一前进能到达终点，首尾属于同一容器，前后放置正确）
29、更易型算法是什么？是否能在所有容器上使用？ （更改容器内部元素值或顺序的如remove，不能在关联式容器和无序容器上，会破坏容器自身对元素次序的维护，其迭代器都被声明为指向常量的value或者key，只能调用其成员函数）
30、lambda函数有什么好处和缺点？？其作为容器排序准则有设么注意事项？下面程序有什么问题？（实现和调用在一起方便阅读，编译器优化更好，无内部状态，容器排序准则要求具有默认构造函数，在建立容器时调用，如果没有就需要手动传入）
        std::set<Person, decltype(cmp)> coll();    （std::set<Person, decltype(cmp)> coll(cmp);）
31、函数对象有什么优点？（带状态，具有类型可以作为template参数，更好的执行期优化比普通函数更快）
32、对序列式、关联式、无序容器的元素分别有什么要求？（有default构造函数以创建非空容器，有排序准则，有hash函数和等同测试）
