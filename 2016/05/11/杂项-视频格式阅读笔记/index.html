视频格式阅读笔记

  1. 分辨率

    QVGA：320 x 240
    VGA： 640 x 480
    NTSC：720 x 480
    PAL： 768 x 576
    SVGA：800 x 600
    XGA： 1024 x 768
    HD720： 1280 x 720
    HD1080：1920 x 1080

  1. H.264/AVC

    H.264/AVC 是ITU-T VCEG（国际电联 视频编码专家组） 和ISO/IEC MPEG （国际标准化组织 运动图像专家组）组建的联合视频组（JVT）共同制定的共同开发的视频处标准，作为国际准14496-10（MPEG-4 第10部分）高级视频编码（AVC）。

    H.264/AVC的设计方案包含两个层次：

    视频编码层（VCL，Video Coding Layer）：主要致力于有效地表示视频内容
网络抽象层（NAL，Network Abstraction Layer）：格式化VCL视频表示，提供头部信息，适合多种传输和存储媒体。

    位率：8~12b/pix
    采样方式: 4:2:0 4:2:2 4:4:4
    支持rgb格式的压缩，避免了色度空间转换的舍入误差。
    编码解码都是以宏块为单位，逐个宏块编解码，然后组合为一帧图像。

    avc 许可政策每台产品0.20 美元

  1. H.264/AVC优越性

    1. 码率低：和MPEG-2等压缩技术相比，在同等图像质量下，采用H.264技术压缩后的数据量只有MPEG-2的1/2~1/3。

      1. 图像质量高：H.264能提供连续、流畅的高质量图像。

      2. 容错能力强：H.264提供了解决在不稳定网络环境下容易发生的丢包等错误的必要工具。

      3. 网络适应性强：H.264提供了网络适应层， 使得H.264的文件能容易地在不同网络上传输。

  1. 分辨率和实时性术语：

    完全实时：pal=25f/s、ntsc=30f/s
    基本实时（准实时）：pal：8~15f/s ntsc：10~17f/s
    非实时：pal<8f/s ntsc<10f/s

    d1为480i格式，和ntsc模拟电视清晰度相同
    d2为480p格式，和逐行扫描dvd规格相同，dvd的分辨率为480p，也就是720x480(ntsc)720x576(pal)
    d3为1080i格式，分辨率为1920×1080 i/60hz
    d4为 720p格式，分辨率为1280×720 p/60hz
    d5为1080p格式，分辨率为1920×1080逐行扫描，专业格式

  1. h.264 标准压缩系统

    视频编码层（ vcl ）：包括 vcl 编码器与 vcl 解码器，主要功能是视频数据压缩编码和解码。包括运动补偿、变换编码、熵编码等压缩单元。采用 dct 变换编码加 dpcm 的差分编码，即混合编码结构.
    网络提取层（ network abstraction layer ， nal ）：提供一个与网络无关的统一接口，它负责对视频数据进行封装打包后使其在网络中传送

  1. x264

    x264是一种免费的、具有更优秀算法的H.264/MPEG-4AVC视频压缩编码格式。它同xvid一样都是开源项目，但x264是采用H.264标准的，而xvid是采用MPEG-4早期标准的。X264最大的特点在于其出发点是为了H.264的实用性。与JM相比，X264并没有完整的对H.264标准给与实现，而是舍去了一些对编码性能贡献很小但计算复杂度极高的新特性，如多参考帧、帧间预测中不必要的块模式等技术。从而计算复杂度降低，编码效率却并没有明显降低。码流兼容H.264。在实用的系统中，比如比较普遍的在DSP芯片上实现H.264，大多是移植X264代码。X264只有编码端没有解码端.

  1. CIFF

    标准化图像格式（Common Intermediate Format），在H.323协议簇中，规定了视频采集设备的标准采集分辨率。电视图像的空间分辨率为家用录像系统(Video Home System，VHS)的分辨率，即352×288。

  1. YPbPr和YCbCr

    YPbPr表示逐行色差信号，是数字信号。
    YCbCr表示隔行色差信号，是模拟信号。

  1. CVBS

    CVBS 是复合视频广播信号 或 复合视频消隐和同步，也叫做基带视频或RCA视频。复合视频包含色差（色调和饱和度）和亮度（光亮）信息，并将它们同步在消隐脉冲中，用同一信号传输。　

    接口是音频、视频分离的视频接口，一般由三个独立的RCA插头(又叫梅花接口、RCA接口)组成的，其中的V接口连接混合视频信号，为黄色插口;L接口连接左声道声音信号，为白色插口;R接口连接右声道声音信号，为红色插口。

  1. TU-R BT 601/656

    TU-R BT 601: ；21芯，Y、U、V信号16位数据传输并行传输，行场同步单独输出。

    ITU-R BT 656: 9芯，8位数据串行传输，先传Y，后传UV，行场同步信号嵌入在数据流中不需要单独输出，传输速率是601的2倍。

    分量信号的编码数据流是很高的。以4：2：2编码标准为例，其比特流为：13.5×8+6.75×8×2=216Mb/S。若采用4：4：4编码方式，即对复合信号直接编码，其抽样频率取为13.3×8=106.4 Mb/S

  1. 计算机图形显示使用RGB逐行扫描方法，而广播视频则使用YCbCr隔行扫描方法

    • 复合视频信号 = 亮度通道 + 色度通道（色彩脉冲解调参考信号）

    • Gamma校正：人的视觉对亮度的灵敏度特性几乎恰好与CRT的输出特性相反

      • 显示器的亮度与输入信号电压成正比，一般为2.2~2.5

    • RGB并非图像处理的最佳选择，因为如果要变动一个通道，则也必须在另外两个通道进行更改，而且每个通道的带宽相同。

    • YPbPr用于分量化的模拟视频中，YUV则适用于复合的NTSC和PAL系统，YCbCr则与分量化数字视频有关。

    • 视频处理的一般过程：

    • 隔行扫描的模拟视频流
      -> 视频端口
      -> 去隔行操作
      -> 扫描速率转换
      -> 4:2:2 YCbCr视频流
      -> 像素处理（图像滤波(二维卷积、3X3掩膜)）
      -> 图像边界处理
      -> 色度的再采样（4:2:2 -> 4:4:4、插值）
      -> Gamma校正
      -> 颜色转换、缩放（下采样(减少像素和/或行的数量)或者上采样(增加像素和/或行的数量)）
      -> 显示处理（ALpha混合(多层缓冲区叠加时对内容处理)、合成、色度键控(层次颜色控制)）
      -> 输出格式化（RGB888、666、565）
      -> LCD平板上显示
