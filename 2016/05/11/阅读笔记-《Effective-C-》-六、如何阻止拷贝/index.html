六、如何阻止拷贝

May 11, 2016 | 阅读笔记《Effective C++》 | Hits
文章目录

##一、为什么需要阻止拷贝
某些情况下，不希望类实例化生成的对象被拷贝或者复制，这时需要阻止编译器生成默认拷贝构造函数和默认重载赋值操作符=，并避免对他们的调用。

##二、阻止编译器默认拷贝构造函数和默认重载赋值操作符=

  • 很简单，只要自己声明一个，编译器就不会生成默认的。并且声明为private，防止对它们的外部访问。如：

    class class_A 
{
public:
……
private:
class_A(const class_A &)；        //声明拷贝构造函数，阻止编译器生成默认版本
class_A& operator=(const class_A&);    //声明重载赋值操作符=，阻止编译器生成默认版本
}

这样的访问就会失败：
class_A a；
class_A b(a);    \\error
class_A b=a;    \\error

##三、阻止对自定义的拷贝构造函数和重载赋值操作符的访问
声明成private，阻止外部访问，但这样还是可以被类成员函数和有元访问。解决办法有两种：

  • 只声明，不定义，导致一个连接期错误。这个很简单，如果调用了函数，连接时报错，但如果做库怎么办。
  • 将连接期错误提前到编译期

##四、将连接期错误提前到编译期

  • 这是个经典做法，定义一个uncopyable class，在这个类中实现上面的做法，我们的类继承它。这样在派生类中因为继承了基类中定义为private的拷贝构造函数和重载赋值操作符, 在编译器为派生类生成默认的copy构造函数时无法访问它们，从而阻止编译器的生成。而又因为基类中为private，派生类成员也无法访问，所以编译器就能检查出错误。这是个经典做法：

    class Uncopyable {
    protected:
    Uncopyable();
    ~Uncopyable();

    private:
    Uncopyable(const Uncopyable&);
    Uncopyable& operator=(const Uncopyable&); 
    };

class Ourclass::public Uncopyable{    //派生类无法访问基类的拷贝构造函数和重载赋值操作符，也不会生成默认的
            ……
            };        

这个基类不含数据成员，符合empty base class optimization，或者也可以使用boost提供的版本：noncopyable基类。
