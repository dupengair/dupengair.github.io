条款34 注意哪个算法需要有序区间

May 11, 2016 | 阅读笔记《Effective STL》 | Hits
文章目录

##一、算法有自己适用的区间

  1. remove需要前向迭代器和可以通过这些迭代器赋值的能力，所以它不能应用于由输入迭代器划分的区间，也不能是map或multimap
  2. 很多排序算法需要随机访问迭代器，所以不可能在一个list的元素上调用这些算法

  3. 只能操作有序空间的算法：

    binary_search     
lower_bound
upper_bound     
equal_range
set_union         
set_intersection
set_difference     
set_symmetric_difference
merge             
inplace_merge
includes
    • 需要有序区间的算法为了比它们用于可能无序区间提供更好的性能而这么做

  4. 一般用于有序区间的算法

    unique 
unique_copy

##二、内部

  1. 搜索算法binary_search、lower_bound、upper_bound和equal_range
    • 因为它们使用二分法查找来搜索值，这些算法保证了对数时间的查找，但必须给它们已经排过序的值
  2. 算法set_union、set_intersection、set_difference和set_symmetric_difference
    • 提供线性时间设置的操作性能
  3. merge和inplace_merge
    • 执行有效的单遍合并排序算法：读取两个有序区间，然后产生一个包含了两个源区间所有元素的新有序区间
    • 以线性时间执行，如果它们不知道源区间已经有序就不能完成
  4. includes
    • 检测是否一个区间的所有对象也在另一个区间中
    • includes可能假设它的两个区间都已经有序，所以它保证了线性时间性能
  5. unique和unique_copy
    • 在无序区间上也提供了定义良好的行为
    • 标准描述unique的行为：从每个相等元素的连续组中去除第一个以外所有的元素
    • 如果要unique从一个区间去除所有重复值（也就是让区间中所有值“唯一”），必须先确保所有重复值一个接着一个
    • 所以总是要确保你传递给unique（或unique_copy）的区间是有序的

##三、排序一致性

  1. 因为有很多不同的方式来排序，所以保证给STL所使用的排序相关信息一致是很重要的

  2. 如果传一个区间给一个也带有比较函数的算法，确保传递的比较函数行为和用于排序这个区间的一样

    vector<int> v;
... 
sort(v.begin(), v.end(), greater<int>()); // 降序排列
bool a5Exists = binary_search(v.begin(), v.end(), 5); // 在这个vector中搜索5假设它是升序排列！

    • 使用一致的排序准则

      bool a5Exists = binary_search(v.begin(), v.end(), 5. greater<int>()); // 把greater作为比较函数
