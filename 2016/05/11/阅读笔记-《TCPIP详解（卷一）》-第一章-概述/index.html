##一、TCP/IP 四层模型

[ Ping ] [ Telnet ] [ FTP ] [ SMTP ]   :   [ DNS ] [ TFTP ] [ SNMP ] [ NFS ]    ( 应用层 )  
    |                  |                   |                   |                    [ 根据TCP/UDP首部的端口号进行分用 ]
    |            [ TCP ]                   |                [ UDP ]                    ( 传输层 )
   |                                      |                                        [ 根据IP首部的协议值进行分用 ]
[ ICMP ]                            —   [ IP ]  —            [ IGMP ]                ( 网络层 )
                                          |                                        [ 根据以太网首部的帧类型进行分用 ]
                  [ ARP ]          —  [ 硬件 ]  —             [ RARP ]              ( 链路层 )
  1. 链路层包括操作系统中的设备驱动和硬件网卡，用于处理与传输媒介的物理接口细节
  2. 网络层处理分组在网络中的活动
    • ICMP用于网络层与其它主机交换错误报文的其它信息
    • IGMP用于将一个UDP数据报多播到多个主机
  3. 传输层为两台主机提供端到端的通信
  4. 应用层处理特定的应用程序细节
    • ARP用于转换网络层和链路层使用的地址

##二、网络地址

A类：[0,127].255.255.255

B类：[128,191].[0,255].255.255

C类：[192,223].[0,255].[0,255].255

D类：[224,239].255.255.255

E类：[240,255]. —.—.—

IP地址有3类：单播、多播、广播

##三、封装

  • TCP传给IP的数据单元称为TCP数据段（TCP segment）
  • IP传给网卡的数据单元称为IP数据报（IP datagram）

  • 通过以太网传输的比特流称为帧（Frame）

            应用程序
               |                 ——                                           | 数据 |
              TCP
               | （segment）      ——                             | 20: TCP首部 | 数据 |
              IP
               | （datagram）      ——                   | 20: IP首部 | 20: TCP首部 | 数据 | 
              网卡
               | （frame）        ——    | 14: 以太网首部 | 20: IP首部 | 20: TCP首部 | 数据 | 4: 以太网尾部 |
----------以太网----------
  1. IP首部16bit的协议域，ICMP为1，IGMP为2，TCP为6，UDP为7
  2. TCP和UDP用16bit的端口号区分不同的应用程序
  3. ICMP和IGMP报文都被封装在IP数据报中，它们是IP的附属协议

##四、客户-服务器模型

  1. 服务器分两种类型：重复性和并发型
  2. 一般来说，TCP服务器是并发的，UDP服务器是重复的

##五、端口号

  1. 1~1023为保留端口
  2. 客户端通常对端口不关心，客户端口号又称临时端口号
  3. 当TCP和UDP提供相同的服务时，一般选择相同的端口号
