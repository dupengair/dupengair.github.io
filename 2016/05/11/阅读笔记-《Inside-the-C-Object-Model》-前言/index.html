#前言

##一、

  1. cfront——Bjarne Stroustrup的第一个C++编译器。
  2. Foundation项目——为构建大型系统定义的新开发模型，用C++进行完成大规模程序设计来解决问题的方法，其对象模型为Simplifier。
  3. Barbara Moo（团队管理者）——管理一个团队，就像放牧一群骄傲的猫。
  4. Simplifier是编译器的一部分，处于type codeing 和 code generation 之间，即将用户编写的代码用C++对象模型的方式为编译器生成代码。
  5. Simplifier的一个目标：提供一个对象模型体系，在其中对象的实现是一个虚拟接口，支持各种对象模型。
  6. Inside the C++ Object Model（中文名：深度探索C++对象模型）是一本由一位编译器设置者针对中高级C++程序员所写的书。

##二、什么是C++对象模型

  1. C++对象模型

    • 语言中面向对象程序设计的部分
    • 对各种支持的底层实现机制部分

  2. 不变量

    • 完整的virtual functions在编译器就固定下来，使调用操作得以快速派遣，但会失去执行期的弹性，用户不可以动态改变。
    • 目前编译器使用每个class的专属virtual table实现virtual functions，在编译器构造完成。

  3. 关于编程

    • 一个人不应该用猜的方式，或者等待某大师的宣判，对问题的解答应该来自于我们自身对对象模型的了解。

  4. 关于本书

    • 写本书目的在于消除对C++语言以及对面向对象支持的错误认识。
    • 本书尽可能将各种Object facilities（inheritance、virtual functions、指向class member的指针……）带来的额外负荷说明

  5. 关于读者

    • 中级以上的C++程序员
