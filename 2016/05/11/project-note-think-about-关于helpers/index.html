Helper

1) Pipe
@ 有读写状态标志，这个很好，在普通读取（非阻塞）的方式下（管道状态 IDEL）如果遇到管道读取不出来可能是因为没有写入（管道状态 PENDING），此时可以根据状态切换为阻塞式读取，用rolling或者select。
@ 缺乏多用户管理，统计系统创建的管道数
@ 没有针对PIPE_BUF的处理

2) Socket
@ 只有STREAM式的，没有UDP和原始套接字
@ Accept时为客户端socket分配内存，留给客户区回收，容易资源泄漏，一般情况下EdgeOS会用一个智能指针来管理

3) Event
@ 一对一，不具有管理功能，不能对外
@ 只能唤醒自己，不能唤醒指定线程
@ 线程启动停止执行客户端动作
@ 可以通过Event来向线程传递数据，如Event类型，这是优点，可以发挥

4) Xml
@ 不具有线程安全性，WriteXmlElement会改变全局fromatting

5) String
@ 缺乏全局空String对象：static const std::string s_empty;

6) Timer
@ ThreadTimer 采用轮询实现，没有真正使用系统定时器

7) Thread
@ 三个工厂函数Create可以用默认参数合成一个，避免重载
@ 多个get/setpriority函数功能重复
@ thread默认名可以为主函数名

8) System
@ 多个ToString()重载，可以用一个iostringstream模板替代
@ string转int用atoi效率更高，反之sscanf、sprintf效率高
@ 用空间名称代表转换类型，不符合一般操作习惯，用模板类型更地道

@@@ 所有的helper在使用系统资源时都没有做管理，系统无法了解当前的状况
