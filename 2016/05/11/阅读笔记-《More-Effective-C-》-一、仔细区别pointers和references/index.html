一、仔细区别pointers和references

May 11, 2016 | 阅读笔记《More Effective C++》 | Hits
文章目录

不论pointers还是references，都得间接参考其它对象。

##一、没有所谓的null references, references一定得代表某个对象，必须有初值

  1. 如果一个变量总是必须代表一个对象，不允许这个对象变为null，应该使用reference

  2. 写出这种代码的程序员应该被隔离：

    char *pc = 0;
char& rc = *pc;

  3. 这个事实意味着references可能比使用pointers更有效率，因为使用references前不需判断其有效性

    void printDouble(const double& rd)    
{
    cout << rd;
}

void printDouble(const double* pd)    
{
    if(pd) {
        cout << pd;
    }
}

##二、pointers可以被重新赋值，指向另一个对象，references总是指向它初始化的对象
string s1(“Nancy”);
string s2(“Clancy”);

string &rs = s1;
string *ps = &s1;

rs = s2;        //s1 的值变成了s2
ps = &s2;        //s1 没有变化
  • 当需要考虑“不指向任何对象”的可能性时，或考虑在“在不同时间指向不同对象”的能力时，应采用pointers。前一种情况可设为null，后者可改变其所指对象。当确定“总会代表某个对象”，而且“一但代表了某个对象就不能改变”，应选用references。

##三、某些操作符的实现需要使用references，如[]，需要能当做左值进行运算。
vector v(10);
v[5] = 10;

如果[]返回pointers，则需写为;
*v[5] = 10;

##四、结论
当知道需要指向某个东西，而且绝不会改变其指向，或实现一个需要references语法的操作符时，应该选择references，其它任何时候都采用pointers
