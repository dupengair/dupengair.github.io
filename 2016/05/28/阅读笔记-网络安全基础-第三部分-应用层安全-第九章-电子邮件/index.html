一、简单邮件传输协议 SMTP

  1. 协议

    • 使用TCP流服务，端口25
    • 消息是7bit ASCII格式
    • 采用请求-应答
    • 命令和回应由回车换行<CRLF>结束

  2. 攻击

    • 基于头部的攻击

      • 头部简单，然后无效的命令和应答都会忽略
      • 可能通过缓冲区溢出攻击

    • 基于协议的攻击

      • 不常见，协议时序可控，任何冲突都会被忽略

    • 基于验证的攻击

      • 协议缺乏验证，一般检查域名是否有效
      • 邮件欺骗、垃圾邮件、恶意邮件
      • 利用中继来伪造返回地址
      • 用户名探测

    • 基于流量的攻击

      • 嗅探
        • 协议不加密，可以读取内容
      • flood
        • 利用转发，效率低

  3. 一般对策

    • 利用协议的STARTTLS和AUTH命令，辅助安全

二、POP和IMAP

  1. 协议

    • 用户代理是远程的，用户通过POP或IMAP访问用户代理，以获取电子邮件
    • 邮局协议(Post Office Protocol, POP)
      • 用于服务器到客户端的传输，提供对远程电子邮件的预览和有限的管理
      • 客户端先通过服务器验证(明文的用户名密码)，然后查询并下载邮件
      • POP3基于TCP，使用端口110
      • 用于传输，而不是服务器上的邮件管理
    • 网际消息访问协议(Internet Message Access Protocol, IMAP)
      • 支持服务器邮箱管理电子邮件
      • 可以产生、管理和删除远程邮箱

  2. 攻击

    • 基于头部和协议的攻击
      • 头部很简单，是非限制性的自由文本格式，没有什么攻击
    • 基于验证的攻击
      • POP3允许用户 不受限制的登录，攻击者往往依赖字典
    • 基于流量的攻击
      • 用户名密码以纯文本形式传输，需要加密保护

三、多用途网际电子邮件扩展协议 MIME

  1. 协议

    • 用于对邮件内容进行格式化，可传输任何类型的数据，并呈现给客户
    • 头部自由，没有包交换

    • MIME头部：

      | SMTP headers | MIME Version | MIME Headers | Email Object | MIME Headers | Emil Object | ...

    • Header 字段

      • MIME Version:1.1
        • 表示当前MIME版本，用于指出余下的格式是MIME格式

      • Content-Type:type/subtype;parameters

        • 内容类型

        • 可被用来传输恶意代码

          type        subtype        disc
Text        Plain        非格式化文本        
            HTML        HTML格式
Multipart    Mixed        多个有序文本
            Parallel    多个无序对象
            Digest        多个有序RFC822对象
            Alternative    同一个对象的可替换方法
Message        RFC 822        封装后消息
            Partial        大尺寸消息的一部分
            External-body    
                        某个扩展消息的参考
Image        JPEG        JPEG图像
            GIF            GIF图像
Video        MPEG        MPEG视频
Audio        Basic        音频对象
Application    Postscript    脚本
            Octet-Stream
                        8bit二进制对象

      • Content-Transfer-Encoding:type

        • 内容编码格式

      • Content-ID

        • 可选标识符

      • Content-Description:

        • 提供内容描述
        • 可被用于屏蔽真实标识，如内容类型为可执行，描述为图像

      • Content-Disposition:type

        • 内容定位取消，告诉接收方如何显示内容
        • 正文和附件容易受到攻击，比如将恶意代码作为正文打开并执行

  2. 攻击

    • 基于头部的攻击

      • 头部可以隐藏消息的实际内容
      • 使用基于HTML的邮件也可以隐藏消息内容

    • 基于协议的攻击

      • MIME协议不涉及多个协议层交互
      • 主要是使用附件携带恶意代码，如蠕虫

    • 基于验证的攻击

      • MIME协议不支持用户验证，可以让攻击者产生受信的电子邮件以达到欺骗的目的

    • 基于流量的攻击

      • 可能产生大尺寸的邮件

四、一般对策

  1. 加密和验证
    • PGP(Pretty Good Privacy)
      • 拥有接收方秘钥的用户才能阅读消息
      • 阻碍来自秘钥分配
  2. 邮件过滤
    • 消息分类分析，通过训练提高系统辨识能力
    • 加入标注，识别垃圾邮件
    • 过滤列表
  3. 内容过滤
    • 过滤器需要理解MIME协议
    • 不能分析加密邮件
  4. 邮件取证
