十二、拷贝对象要全面

May 28, 2016 | 阅读笔记EffectiveC++ | Hits
文章目录

  2. 2. 二、不完整复制的情况
  3. 3. 三、完整复制
  4. 4. 四、总结

一、自定义拷贝构造函数和重载拷贝运算符时，编译器不会检查是否完全拷贝

  • default的拷贝构造函数和拷贝运算符会value-by-value地拷贝对象的每一个成员，但自定义这个操作时，编译器不会检查是否完整复制。

二、不完整复制的情况

如：
class Base {
    ……
private:
    int id;                //基类中有个id成员
};

class Derived::public Base{
public:
    Derived(const Derived& rhs);                //派生类的拷贝构造函数
    Derived& operator=(const Derived* rhs);        //派生类的重载赋值运算符
    ……
private:
    int number;
};

Derived::Derived(const Derived& rhs):number(rhs.number)    //只初始化了派生类对象的成员，基类成员由基类的defualt构造函数初始化
{
}

Derived& Derived::operator=(const Derived& rhs)        //同上
{
    number = rhs.number;
    return *this;
}
  • Derived派生类的拷贝构造函数并没有指定参数传给其基类Base构造函数，即初始化列表中没有初始化派生类对象中的基类成员，它们是由基类自己的default构造函数初始化的。重载赋值运算符时也一样。

三、完整复制

  1. 对象的所有local成员变量

  2. 基类中的所有成员变量

    如：
Derived::Derived(const Derived& rhs):Base(rhs):number(rhs.number)    //调用基类Base的拷贝构造函数
{
}

Derived& Derived::operator=(const Derived& rhs)    
{
    Base::operator=(rhs);                            //对基类成分进行赋值
    number = rhs.number;
    return *this;
}

四、总结

  1. 拷贝对象时，确保拷贝所有local成员变量和基类成员
  2. 如果拷贝构造函数和拷贝运算符有相同的部分，可以提取出一个私有的公共接口来调用，避免重复代码
