二、uptime

# uptime 
 19:49:51 up  1:18,  1 user,  load average: 0.12, 0.12, 0.18
  1. 系统负载 —— 可运行态进程和不可中断态进程总数
  2. 包含当前时间、已运行时间、当前登陆用户数，系统过去1、5、15分钟的平均负载
  3. 可运行态：正在使用或等待cpu。 不可中断态：正在等待IO访问
  4. 平均负载为1意味着单个cpu总是有负载的，对于单cpu来说1或2是可接受的，这个数字等于cpu个数为最佳
  5. 可由uptime确定问题在自身还是在网络，如果cpu负载不高，问题可能和自身无关

三、ps

  1. 以当前有效用户ID运行

  2. 显示

    • 显示单个进程： ps -C FwLoader
    • 显示单个进程某列： ps -C FwLoader -o pid=
    • 显示线程信息：ps -L $(pidof FwLoader)

  3. 参数

    -s：显示信号格式
-u：显示面向用户格式
-v：显示虚拟内存格式
-e：所有进程
-f：全格式列表
-F：额外的全格式

  4. 状态

    D：不可中断睡眠
R：正在运行或可运行
S：可中断睡眠
T：已停止
Z：已死亡，还未被父进程回收
BSD格式：
<：高优先级，其它用户不能nice
N：低优先级，其它用户可以nice
L：内存中锁定分页，用于实时或定制IO
s：session leader
l：多线程
+：前台进程组

四、free

# free -lm
             total       used       free     shared    buffers     cached
Mem:           587        176        410          0          0        111
Low:           587        176        410
High:            0          0          0
-/+ buffers/cache:         65        522
Swap:            0          0          0
  1. shared表示/proc/meminfo中Shmem字段（实测有差别）

  2. 含义

    Mem: used = 已使用 + buffer + cache
     free = 空闲
-/+ buffers/cache: used = 已使用
                   free = 空闲 + buffer + cache
  3. 64bit系统不再使用high内存
  4. 使用/proc/buddyinfo确定每个区域可用的内存块。列代表可用的分页数量，第一列为0，每列的可用分页数量为当前数字乘以2的列数次幂，再乘以分页大小。如第0列的数字为4，分页数为：4 * 2^0 * 4KB = 16KB

  5. buffer与cache

    • buffer: buffer cache，块设备写缓冲区

    • cache : page cache，文件系统读缓冲区，大说明cache住的文件多，如果频繁访问的文件被cache，磁盘IO效率低下

      A buffer is something that has yet to be “written” to disk.

      A cache is something that has been “read” from the disk and stored for later use.

五、vmstat

  1. 可用选项

    delay: 更新间隔秒数，不指定时为启动到目前的平均值
count: 更新次数，指定delay时默认无限
-a: 显示活跃和非活跃内存
-f: 启动到现在的fork数量，包含fork、vfork、clone
-m: SLAB模式 
-s: 显示内核事件统计信息
-d: 磁盘模式
-D: 磁盘活动统计数据
-p: 分区模式

State: tcp才有状态
    ESTABLISHED: 已建立连接

    SYN_RECV : 被动打开

    FIN_WAIT2: 等对方关闭
    TIME_WAIT: 等待2MSL

    CLOSE_WAIT: 被动关闭，对方已关闭，等待本地关闭
    LAST_ACK : 本地已执行关闭，等对方确认
    LISTEN   : 正在监听
    CLOSING  : 两端已执行关闭，仍然有数据没有发送完成
    UNKNOWN  : 未知状态

十三、numastat

  1. 显示每个NUMA(Non-Uniform Memory Architecture)节点的内存统计信息

    $ numastat
                           node0

numa_miss                      0
numa_foreign                   0
interleave_hit             17197
local_node               2419687
other_node                     0

  2. 输出项

    numa_hit:    预期的内存在这个节点上成功分配
numa_miss:    在这个节点上分配内存，但进程使用了不同的节点。每个numa_miss在另一个节点上有一个numa_foreign
numa_foreign: 为改节点准备的内存实际被分配到了别的节点上，每个numa_foreign在另一个节点上有一个numa_miss
interleave_hit: 预期的这个节点上成功交错分配的内存
local_node: 当一个进程在节点上运行的时候，在这个节点上分配的内存
other_node: 当一个进程运行在其它节点上，在这个节点分配的内存
