一、Set和MultiSet

  1. 关联式容器，自动排序

  2. 定义

    namespace std {
    template <typename T, typename Compare = less<T>, typename Allocator = allocator<T>>
    class Set;
    template <typename T, typename Compare = less<T>, typename Allocator = allocator<T>>
    class MultiSet;
}

  3. 关于排序准则

    • 非对称
      • 对operator<，如果x<y为true，则y<x为false
      • 对判断式op()，如果op(x, y)为true，则op(y, x)为false
    • 可传递
      • 对operator<，如果x<y为true， 且y<z为true，则x<z为true
      • 对判断式op()，如果op(x, y)为true，且op(y, z)，则op(x, z)为true
    • 非自反
      • 对operator<，x<x永远为false
      • 对判断式op()，op(x, x)永远为false

    • 等效传递

      • 对operator<，如果!(a<b)&&!(b<a)为true，且!(b<c)&&!(b<c)为true，则!(a<c)&&!(c<a)也为true
      • 对判断式op()，如果op(a, b)、op(b, a)、op(b, c)、op(c, b)都为false，则op(a, c)、op(c, a)也为false
      • (其实就是 a == b, b == c 得到 a == c)

    • 这意味着必需区分less和equal

二、能力

  1. 通常以平衡二叉树实现
  2. 由于自动排序，在查找元素时性能良好，具有对数复杂度
  3. 因为自动排序，所以不能直接改变元素值，必需先删除旧元素，再插入新元素，具体表现在：
    • 不提供任何操作可以直接访问元素
    • 从迭代器角度看，元素值是常量

三、操作

  1. 定义排序准则

    • 以template参数定义：
      • 此时排序准则就是类型的一部分，类型系统会确保只有排序准则相同的容器才能被合并。第二参数是排序准则的类型，容器构造函数会调用改类型的default构造函数，来生成函数对象
    • 以构造函数参数定义：
      • 可以运行期指定排序准则，同一个类型可以运用不同的排序准则，而排序准则的初始值或者状态也可能不同

  2. 排序准则也被用来检验元素相等性，默认准则——函数less<>

    if(! (elem1<elem2 || elem2<elem1))

    好处：

    • 只需要一个实参
    • 不必针对元素相等性提供operator ==，如果用operator==对两个容器比较，就需要容器的所有元素支持operator==的操作
    • 可以定义相等性
    • 可能更费时，如果要进行两次比较的话

  3. 元素比较是以字典顺序，只适用于类型相同的容器，元素和排序准则必需有相同的类型，如果要比较不同类型（排序准则不同）的容器，必需采用STL提供的比较算法

  4. Set和multiset在元素查找方面有优化设计，提供特殊查找函数，实现为对数复杂度，STL的普通函数为线性复杂度
  5. equal_range()将lower_bound()和upper_bound()的值组合成一个pair返回
  6. 对于insert和emplace操作，set返回一个pair，first表示新元素位置，second表示安插成功（因为set不允许重复元素，所以可能安插失败），MultiSet则只返回位置
  7. 具有“位置提示参数”的安插函数，采用位置作为提示来提高效率，如果被安插元素的位置正好位于提示位置后，安插操作为常量复杂度。这个函数的结果形式与普通安插函数相同（返回iterator），使得通用型安插迭代器得以实现

  8. erase返回被删元素的个数，set上只能是0或1，对于multiset，如果含有重复元素，则不能用erase(val)来返回这些元素的第一个，而是用成员find（效率比STL高）和erase(pos)

    std::multiset<int> S = {1, 2, 3, 3};
auto pos = S.find(3);
if (pos != S.end())
    S.erase(pos);

  9. C++11之前，erase操作出于效率的考虑，不返回任何东西，因为查找后继者需要时间

  10. C++11起，insert、emplace和erase会保留等价元素的相对次序，新元素会保证安插在等价元素形成的区间末端
