二十五、将constructor和non-member function虚化

May 28, 2016 | 阅读笔记MoreEffectiveC++ | Hits
文章目录
  1. 1. 一、虚化构造函数
  2. 2. 二、virtual copy constructor
  3. 3. 三、将Non-member Function的行为虚化

  • 假设需要根据输入参数构造不同的派生类对象：

    class NLComponent {                        //抽象基类，内含纯虚函数
public:            
    …
};

class TextBlock: public NLComponent {    //文本类
public:
    …
};

class Graphic: public NLComponent {        //图片类
public:
    …
};

class NewsLetter {                        //新闻类，内含一系列NLComponent类对象
public:        
    …
private:
    list<NLComponent*> components;
};

一、虚化构造函数

class NewsLetter {                //新闻类，内含一系列NLComponent类对象
public:
    NewsLetter(istream& str);
    /*
     * 从str读取NLComponent的数据，产生对象并返回一个指针
     */
    static NLComponent* readComponent(istream& str);
    …
};

NewsLetter::NewsLetter(istream& str)
{
    while(str)
    {
        /*
          * 将返回指针加入链表
          */
        components.push_back(readcomponent(str));
    }
}
  1. 由于readcomponent产生新对象，行为类型constructor，但能产生不同类型对象，所有称为一个virtual constructor
  2. 所谓virtual constructor是某种函数，根据其获得的不同输入，可产生不同的对象

二、virtual copy constructor

  1. virtual copy constructor会返回一个指针，指向其调用者的一个副本。

  2. 基于这种方法，virtual copy constructors常以copyself 或者 clone命名

    class NLComponent {                //virtual copy constructor
public:
    …
    virtual NLComponent * clone() const = 0;
};

class TextBlock: public NLComponent {        
public:
    …
    virtual TextBlock * clone() const { return new TextBlock(*this); }
};

class Graphic: public NLComponent {        
public:
    …
    virtual Graphic * clone() const { return new Graphic(*this); }
};

  3. class 的virtual copy constructor就只是调用真正的copy constructor而已，copy这层含义对两个函数而言是一样的。

  4. 上述实现是利用“虚函数返回值”规则的一个宽松点：当derived class重新定义base class的一个虚函数时，不再需要一定声明与原版本相同的返回类型。如果函数的返回类型是一个指针或者reference，指向baseclass，那么derived class的函数可以返回一个指针或者reference指向该base class 的一个derived class。

    • 有了上述的virtual copy constructor，现在可以为NewsLetter实现一个正常的copy constructor：

      class NewsLetter {                
public:
    NewsLetter(const NewsLetter& rhs);    //copy constructor    
    …
private:
    list<NLComponent*> components;
};

NewsLetter::NewsLetter(const NewsLetter& rhs)
{
    for(list<NLComponent*>::const_iterator it = rhs.components.begin();
            it != rhs.components.end(); ++it)
    {
        /*
          * 将 rhs 内含的NLComponent对象的副本clone到新NewsLetter对象中，
         * 这里需要一个virtual copy constructor，因为这个components list内含对象指针
          */
        components.push_back((*it)->clone());
    }
}

三、将Non-member Function的行为虚化

  1. 考虑让non-member functions的行为视其参数的动态类型不同而不同
  2. 一言以蔽之，在non-member functions中调用虚函数，使其行为表现出多态
