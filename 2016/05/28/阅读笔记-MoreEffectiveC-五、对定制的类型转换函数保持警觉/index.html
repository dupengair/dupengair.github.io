五、对定制的类型转换函数保持警觉

May 28, 2016 | 阅读笔记MoreEffectiveC++ | Hits
文章目录
  1. 1. 一、隐式类型转换符
  2. 2. 二、单自变量constructors
  3. 3. 三、proxy classes
  4. 4. 四、不要提供隐式类型转换
  • 比起C语言，C++有着令人害怕的转型，可能会导致信息遗失。特别是当定义自己的类型，提供可供编译器进行隐式类型转换的函数。
  • 两种函数允许编译器执行隐式类型转换，单自变量constructor和隐式类型转换符。

一、隐式类型转换符

  1. 隐式类型转换符是关键词operator后加上类型名。不能为此函数指定返回值类型，因为其返回值类型已经体现在函数名称上。

    class Rational {
public:
    …
    //operator(int numerator = 0, int denominator = 1);
    operator double() const;    //隐式转换为double
};

  2. 此函数可能被隐含调用，其结果可能是不正确、不直观的程序行为，很难调试。

    Rational r(1, 2);    
cout << r;        //隐式类型转换, 本应输出1/2

  3. 解决办法是以一个功能对等的函数取代类型转换符

    class Rational {
public:
    …        
    double asDouble() const;    //显式转换为double
};

cout<<r.asDouble();

  4. 一般而言，越有经验越会避免使用类型转换操作符，如标准程序库的string类不支持从string类隐式转换为C-style char，而是提供一个显式的member function c_str

二、单自变量constructors

  1. 指能以单一自变量成功调用的constructors。如此的constructor可能声明拥有单一参数，也可能拥有多个参数，并且除了第一个参数外都有默认值。这种情况更加复杂。 

    template<class T>
class Array {
public:
    Array(int low, int high);
    Array(int size);

    T& operator[](int index);
    …
}；

bool operator==( const Array<int>& lhs, const Array<int>& rhs );

Array<int> a(10);
Array<int> b(10);
…
for (int i=0; i<10; ++i)
{
    if(a == b[i]) {        //a应该是a[i], b[i]被转化为Array<int> object
    …
    }
    else {
    }
}

  2. 只要不声明隐式类型转换符就可以避免害处，但单自变量的constructor很多时候是需要的。此时可以采用explicit关键字，让编译器阻止隐式转换的发生。

  3. 如果不支持explicit关键字，可以通过限制转换发生的次数来避免隐式类型转换。

    • C++ Standard规定，任何一个转换程序，都不能内含一个以上的用户定制转换行为（即上面说的两种转换）。可以利用这条规则，让对象的构造行为合法化，并且不运行隐式转换行为发生。

      如，将上述Array类的构造行为改写如下:
template<class T>
class Array {
public:                
    class ArraySize {            //以一个嵌套类实现    
    public:
        ArraySize(int numElement) : theSize(numElement) {}
        int size() const { return theSize; }
    private:
        int theSize;
    };

    Array(int low, int high);
    Array(ArraySize size);
}；

      • 这里把ArraySize嵌套进Array里，强调一个事实：永远与Array类搭配使用。
        此时，如果出现如下的表达式：

        Array<int> a(10);
Array<int> b(10);

if ( a == b[i] )

      • 要想将b[i]转为Array，需要通过int->ArraySize->Array的转换过程，这已经超过了一次的用户定制转换行为，所以编译器会阻止隐式类型转换的发生。

      • 而对于正常的构造过程：

        Array<int> a(10);

      • 编译器只需要将通过int->ArraySize的转换过程即可产生一个临时的

        ArraySize对象，用来构造Array类，达到上述目的。

三、proxy classes

  1. 类似ArraySize这样的classes，往往被称为proxy classes，因为它的每一个对象都是为了其它对象而存在的，好像其它对象的代理（proxy）一般。如本例中，ArraySize只是用来指定Array大小的整数的替身而已。
  2. Proxy objects让程序以超越外观的形式，如本例中的隐式类型转换，进而控制程序的行为，是非常有价值的技术。

四、不要提供隐式类型转换

允许编译器执行隐式类型转换，害处多多，所以不要提供隐式类型转换函数，除非确定需要他们
