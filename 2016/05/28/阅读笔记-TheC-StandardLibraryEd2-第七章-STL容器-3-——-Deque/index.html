第七章 STL容器(3) —— Deque

一、Deque

  1. 和vector一样采用dynamic array来管理元素，接口几乎相同

  2. deque实现通常为一组独立区块，头尾不同的区块向不同方向扩展，因此能在头尾两端快速安插和删除元素

    namespace std {
    template <typename T, typename Allocator = allocator<T>>
    class deque
}

二、能力

与vector的差异：

  1. 两端都能快速安插或删除元素，可以在常量时间完成
  2. 访问元素时多一个间接过程，会稍慢一些
  3. 迭代器要在不同区块跳转，所以是个smart pointer
  4. 使用不止一块内存，所以可能容纳更多元素，max_size可能更大
  5. 不支持对容量或者内存重新分配时机的控制，由实现决定，但重分配性能优于vector，因为不用复制所有元素。
  6. 内存大小可缩减，会释放不再使用的内存块，由实现决定
  7. 任何地点安插或删除元素，都可能使reference、pointer和迭代器失效

三、操作

  1. 不提供容量操作reserve和capacity
  2. 可直接在头部安插和删除元素push_front、pop_front
  3. 对于shrink_to_fit()操作，deque本身就会释放不再用到的内存块，但deque内部用来存放指向不同独立内存块的pointer的空间一般不会释放，如果使用这个函数，就有可能会被缩小，但不具强制性
  4. 只有at()抛出异常

四、补充deque与vector

  1. 因为内存分区块，重分配效果优于vector，适用于元素频繁增长
  2. 删除效率和访问效率都低于vector
  3. 需要在头部插入元素
