一、进程管理

1、什么是创建进程的写时复制？它是如何完成的？（exec 进程共享 页错误 分配新页）
2、创建线程的开销和进程有什么不同？（资源）
3、详述Linux内核对进程优先级的操作
4、进程上下文切换频繁会引发什么问题？为什么？（刷新reg&cache）
5、linux有哪两类中断？怎样提高中断的性能？（软硬，绑定CPU）
6、进程有哪些状态？怎么理解每个进程在很短时间内都是Zombie？僵尸进程是怎样产生的，如何杀死？
7、内核主要的调度算法有哪些？有什么特点？

二、内存管理

8、虚拟内存是否全部被映射到物理内存？（no，磁盘交换空间swap）
9、为何linux内存管理更高效？（虚拟内存分配所有有效的空闲内存空间为磁盘缓存）
10、linux虚拟内存管理是什么机制实现的，做了哪些事情？怎么查看？（伙伴系统，维护空闲分页，给请求分配分页，保证内存区域的连续性，/proc/buddyinfo）
11、分页回收是谁实现的？基于什么原则？目的是什么？通过什么查看？（kswapd，LRU，分页缓存是分页被映射到磁盘上的一个文件，分页属于一个进程地址空间，被用于堆和栈，称为匿名内存，/proc/sys/vm/swappiness）
12、swap是怎么工作的？为什么效率高？（当虚拟内存管理器发现进程的内存分页已经分配，但大量时间没有使用时，会将其移动到swap空间，以释放一个昂贵的主内存分页）

三、文件系统

13、日志文件系统与非日志文件系统的区别？（非日志文件系统中，内核先改变文件系统的元数据，再写实际的用户数据，如果元数据写入中发生崩溃，会损坏数据完整性，日志文件系统中，内核先将改变数据写入日志区）
14、XFS适用于什么场合？（适合大文件的快速存储，不适合单线程密集操作巨大数量的小文件，这方面Ext4更好）

四、IO子系统

15、cache的实现及原理（缓存快存储器数据减少慢存储器读写，数据局部性原理）
16、局部性原理的目的和内容（快存储器上实现更高的缓存命中率，时间局部性：最近使用过的数据，空间局部性：数据周围的数据）
17、dirty page是如何产生的？（当一个进程试图改变数据时，首先将数据写入内存，此时内存和磁盘上数据不同，内存中的数据称为dirty page）
18、什么是IO电梯调度？默认的调度算法是哪个？（文件系统将写操作以bio的形式提交到io队列的过程，CFQ完全公平队列）

五、网络子系统

19、哪个目录是调整socket buffer性能的？（/proc/sys/net/）
20、NAPI(Network API)起什么作用？（减少CPU中断次数）
21、内核Netfilter有哪3个主要功能？（数据包过滤、地址转换、数据包改写）
22、TCP滑动窗口是哪一端进行控制的？什么是慢启动？（接收端，从1个数据段开始成倍增加）

六、性能度量

23、下面几个CPU方面的值，高好还是低好，为什么？
    cpu usage    （持续时间内超过80%，cpu存在瓶颈）
    user time    （高好，系统执行效率高）
    sys time     （低好，内核被中断的时间少）
    waiting      （低好，不应过多等待IO）
    load average    （低好，否则得不到cpu时间的进程多）
    可运行的进程    （低好，不应超过cpu数量的10倍）
    阻塞的进程    （低好，反映出IO瓶颈）
    context switch    （低好，大量的上下文切换如果与大量中断相关，意味着驱动或应用程序可能有问题）
    中断        （低好，否则系统存在软件瓶颈，硬中断对性能影响更大，包括CPU时钟）

24、空闲内存由哪些部分组成？（free+used中的buffer(缓冲)和cache(缓存)）
25、哪些值可以体现出内存瓶颈？（Swap In/Out）
26、哪些值可以体现出块设备瓶颈？（IO等待时间、平均队列长度）
