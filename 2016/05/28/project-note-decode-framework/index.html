3. 模块初始化

  • 加载模块库配置，读取ModuleLibrary.xml

    CLibrary::LoadLibrary()
CModuleLibrary::LoadModuleSetup()
    • 将模块的属性放入CLibrary::ModuleLibrary中。

  • 加载模块配置，读取Setup.xml

    CFramework::LoadSetup()
CFramework::RestoreSetup()
CHost::LoadHostSetup()
CProcess::LoadProcessSetup()
CModule::LoadModuleSetup()
    • 当读取配置时解析到Process下面的Module时，实例化一个CModule对象，从之前加载的ModuleLibrary找到对应的模块属性，保存在CModule对象中。

  • 从配置解析到的模块中找到拥有资源的模块，检查这些模块是否NeedAutoconfig，需要的话，把这些模块从初始化解析到的模块中去掉

    CFramework::LoadSetup()
CFramework::CheckAutoconfigModules()

  • 更新系统元素状态

    CFramework::UpdateElements()

    • 如果以前存在，资源：拷贝状态，释放以前OwnedResources，通过所属模块把以前资源销毁

      • 模块：拷贝状态和所属进程，释放OwnedResources
        • Host：直接拷贝所有数据

    • 如果以前不存在，资源：找到资源Owner，删除Owner原来同一个Guid的资源的拥有

      • 模块：更新Process和Host信息
      • Host：添加进Env
      • 最后加入Elements

  • 解析引用关系

    CFramework::ResolveReferences()
    • 原有的模块：新加入的资源属于该模块，建立从属关系
    • 原有的资源：新加入的模块拥有它，设置为属主模块，如果拥有者是资源，设置属主为资源的属主模块，设置自己为child，属主为parent
      • 新加入的资源被它拥有：设置为child，设置新加入资源的属主模块为这个资源的属主
      • 新加入的资源被它引用：设置为引用者

  • 写入到配置文件中，并备份配置文件

    CFramework::SaveSetup()
