第十五章 Unix域套接字

一、概述

  1. UNIX域支持套接字
    • 字节流套接字
    • 数据报套接字
  2. 用途
    • 比TCP套接字快一倍
    • 可以传递描述符
    • 可以传递用户凭证(用户ID和组ID)，以提供额外的安全措施
  3. Unix域用文件系统的路径名来标识客户端和服务端的协议地址
    • 需要bind到套接字上
    • 如果路径已存在，会bind失败，需要先unlink路径名

二、套接字地址结构

#include <sys/un.h>
struct sockaddr_un {
    sa_family_t sun_family;
    char        sun_path[14];
};
  • sun_path数组中的路径名必需以空字符结尾

三、socketpair

#include <sys/socket.h>
int socketpair(int family, int type, int protocol, int sockfd[2]);
成功返回0，失败返回-1
  • 创建两个随后连接起来的套接字，只用于Unix域套接字
  • family必需为AF_LOCAL，protocol必需为0
  • type可以为SOCK_STREAM或者SOCK_DGRAM
  • 新创建的描述符作为sockfd[0]和sockfd[1]返回
  • 这样得到的结果为流管道，与PIPE的区别为流管道是全双工的
  • 书上说Berkeley的实现是半双工(？)

四、套接字函数

  1. bind绑定后的路径名默认权限是0777，并按照当前umask值进行修正
  2. 路径名需要为绝对路径，不然对端就需要知道本端的应用程序路径才能成功通信
  3. connect指定的路径必需是已绑定的Unix域套接字且已打开，type也需要一致，错误条件如下：
    • 路径未绑定到套接字
    • 路径绑定的套接字未打开(没监听)
    • 监听的套接字类型不符
  4. connect连接域套接字对路径的权限测试，相当于open写打开
  5. Unix域字节流套接字类似TCP
    • 提供无记录边界的字节流服务
    • 监听队列满时，立即返回ECONNREFUSED错误（注意TCP套接字上，监听队列满时，忽略新到达的SYN，等待客户端重试）
  6. Unix域字数据报套接字类似UDP
    • 提供有记录边界的不可靠数据报服务
    • 在未绑定的Unix域套数据报接字上发送数据报不会自动绑定一个临时路径(TCP或UDP套接字则会自动绑定一个临时端口进行发送)
    • 所以除非数据报发送端已绑定路径名到其套接字，否则接收端无法发送应答，因为服务端的recvfrom会返回空路径做对端地址，导致sendto错误

五、描述符传递

  1. 自然的传递

    • fork调用返回以后，子进程共享父进程所有打开的描述符
    • exec调用执行以后，所有描述符的打开状态不变

  2. 过程

    • 创建Unix字节流套接字，不推荐数据报套接字，可能被丢弃
    • 可被传递的描述符不限类型(文件、套接字…)
    • 发送进程穿件msghdr结构，通过msg_control成员作为辅助数据，通过sendmsg发送
    • 接收进程通过recvmsg接收消息，这个消息由内核来专门处理
    • 在sendmsg后，即使发送进程在接收进程调用recvmsg前闭了描述符，对于接收进程仍然处于打开状态
    • 接收到的描述符值与发送的不同是正常的，传递描述符的过程是创建新的描述符，指向内核中相同的文件表项
    • 如果接收进程调用recvmsg时没有可用空间，这个被传递的描述符就会被关闭
    • 接收进程调用recvmsg的过程中避免使用MSG_PEEK标志，否则后果不可预料
      • MSG_PEEK：允许查看已可读取的数据，而且系统不在recv或recvfrom返回

六、接收发送者的凭证(uid、pid、gid …)

  1. 用于服务器验证客户端是否有权限请求相应服务

    #include <sys/socket.h>
struct cmsgcred {
    pid_t cmcred_pid;
    uid_t cmcred_uid;
    uid_t cmcred_eupid;
    gid_t cmcred_gid;
    short cmcred_ngroups;
    gid_t cmcred_gourps[CMGROUP_MAX];
};
    • FreeBSD使用
    • CMGROUP_MAX通常为16，cmcred_groups至少为1，cmcred_ngroups第一个元素是有效组ID
