唤醒

<1> hub_thread()->hub_events()初始化被唤醒的过程：

hub_probe()->hub_configure()->hub_activate()->kick_khubd()->wake_up(&khubd_wait);


struct usb_interface intf->struct device dev->struct device_driver driver
struct usb_driver drv->struct device_driver driver
container_of(ptr, type, member) :  drv = container_of(intf->dev.driver, struct usb_driver, driver)

用法: 通过ptr指向的driver结构体，返回包含driver结构体的usb_driver结构体，即driver结构体是usb_driver结构体的成员。

设备插入

<2> 设备插入的处理过程：

hub_events() -> hub_port_connect_change() -> hub_port_debounce()
                         <1> usb_alloc_dev()
                         <2> usb_set_device_state()
                         <3> choose_address()
                         <4> hub_port_init()  -> hub_port_reset()、hub_set_address()、usb_get_device_descriptor()
                         <5> 当前连接的又是hub
                         <6> check_highspeed()
                         <7> usb_new_device() -> usb_get_configuration()、usb_cache_string()、device_add、、、
                         <8> hub_power_remaining()

端口变化事件发生,hub_irq就会被调用,而hub_irq()最终会调用kick_khubd(),触发hub的event_list,于是再次调用hub_events().

调用hub_quiesce() 的只有两个函数, 一个是hub_suspend , 一个是hub_pre_reset

STR和STD,这是两种Suspend的状态.STR即Suspend to RAM,挂起到内存,STD就是Suspend to Disk,挂起到磁盘.ACPI的状态一共有五种,分别是S1,S2,S3,S4,S5,实际上S4就是STD,而STR就是S3,只不过S1,S2,S3差别不大,不过,在Linux中,S1被叫做Standby,而S3被叫做STR.而S5就是Shutdown.在Linux中说挂起,主要说的就是S1,S3和S4.关于S1,S3,S4这三种状态,在/sys/power/state文件里可以有所体现,cat /sys/power/state,你会看到”standby”,”mem”,”disk”的字样.
