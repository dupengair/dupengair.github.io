附录A 密码学

一、哈希函数

  1. 哈希函数是将任意长度数据转换成固定长度数据的单向函数
  2. 多对一，不同的输入可能产生相同的输出
  3. 用途：
    • 验证用户名密码
    • 签名：验证数据是否被篡改
      • 随数据一起发送哈希值
      • 数字签名：确保哈希值没被修改，并且属于某个需要加密的文件

二、对称加密

  1. 又称共享密钥，加解密双方公开密钥，加解密方法相同
  2. 密钥发布存在安全问题
  3. 常见用法是双方应用之间新的连接产生新的密钥，并采用非对称密钥进行传递
  4. 攻击者往往从密钥产生方法和发布系统入手

三、非对称加密

  1. 又称公钥加密，采用数学上相关的两个密钥，分别用于加密和解密
  2. 公钥传给对方，私钥自己保存，私钥安全性很重要，通常采用对称加密进行保护

  3. 数字签名

    • 明文产生哈希值，用私钥加密，随明文数据一起发送

    • 接收方用私钥解密得到哈希值，与收到的明文进行哈希的结果进行比较

      发送：
明文 --(哈希运算)--> 哈希值 --(私钥加密)--> 数字签名 -> 发送
                                         + 明文  ->

接收：
数字签名 --(公钥解密)--> 哈希值  ->  比较
明文     --(哈希运算)--> 哈希值  ->

  4. 交换秘钥

    • 私钥和消息产生数字签名，并且用随机产生的对称秘钥加密，随机秘钥采用接收方的公钥加密

    • 接收者用私钥解密，得到对称秘钥，并解密出数字签名和消息，用数字签名进行验证

      发送：
私钥 + 消息 --> 数字签名 ++(对称加密)-> 密文

                    随机产生对称秘钥 --(公钥加密)-> 加密秘钥   

接收：

                          个
