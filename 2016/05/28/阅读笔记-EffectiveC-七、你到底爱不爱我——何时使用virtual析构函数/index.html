七、你到底爱不爱我——何时使用virtual析构函数

*”你到底爱不爱我？爱不爱~我？”（零点乐队《》）

我们经常看到有的类会声明virtual析构函数，这样的好处是不言而喻的的，可以通过多态的方式通过基类指针析构指向的派生类对象，那么是不是所有的析构函数都应该声明为virtual呢？这里面涉及到什么技术内幕呢？

一、派生类与virtual析构函数

  1. C++明确规定：“当派生类对象通过一个基类指针被删除，而这个基类的析构函数是non-virtual的，那么其执行结果未定义——通常只析构基类部分，派生类对象没有被销毁”
  2. 任何类只有带有virtual函数，都需要有一个virtual析构函数

二、何时不需要virtual析构函数

  1. 当一个类不用作基类被继承时，不要声明virtual析构函数。
  2. 当一个类设计目的不是为了用作多态的用途，不要声明virtual析构函数，如Uncopyable类和STL的input_iterator_tag，这样的类不需要。

三、深入了解virtual function的实现内幕

  1. 当C++实现virtual function时，对象必须携带一些信息，主要用来运行期决定哪一个virtual function被调用。这份信息有一个vptr（virtual table pointer）指针指出。vptr执行一个由函数指针构成的数组vtbl（virtual table）；每一个带有virtual function的类都有一个vtbl。当对象调用某个virtual函数，实现被调的函数取决于该对象vptr所指向的vtbl，编译器在其中寻找合适的函数指针。
  2. 如果一个类含有virtual function，其对象的体积不受控制，可能增加50%~100%。这时这个对象不再和其它语言的相同声明（如C语言的struct）有一样的结构，也就不可能以地址的方式传给（或接受）不在具有可移植性。

  3. 而一个类不含virtual function时，它的存储结构和声明相同，只是数据成员所占的空间，如：

    class pointer
{
public：
    int x；
    int y；
public：
    int function() {};
}

pointer pA；    //此时pA的大小为两个int的大小，32位系统下为8bytes
    • 所以，随意将一个类的析构函数声明为virtual，与从不把它声明为virtual一样，是不负责任的。只有类中至少含有一个virtual function（且被继承）才为类声明virtual析构函数。

四、virtual 析构函数使用上的注意事项

  1. 不要继承一个标准容器或者带有non-virtual析构函数的类
  2. 希望将类作为抽象类使用，却没有virtual function时，可以声明一个virtual析构函数
  3. virtual析构函数必须有定义（防止被继承时调用出错）
  4. 还要注意如果在拷贝构造函数中以bitwise的形式拷贝对象，则memcpy或者memset可能出问题（见《Inside C++ object model》P73）

五、析构函数的运行方式

  • 析构函数的运行方式是，最深层次的派生类的析构函数最先被调用，然后是其它基类的析构函数被调用，如果某个基类的virtual 析构函数没有定义，会遇到连接期错误。
