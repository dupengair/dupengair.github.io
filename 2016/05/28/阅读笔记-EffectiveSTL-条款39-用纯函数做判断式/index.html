一、判断式与纯函数

  1. 差异

    • 判断式是返回bool，或者其他可以隐式转化为bool的东西
      • 判断式在STL中广泛使用。标准关联容器的比较函数是判断式，判断式函数常常作为参数传递给算法
        *　纯函数是返回值只依赖于参数的函数
      • 在C++中，由纯函数引用的所有数据不是作为参数传进的就是在函数生存期内是常量
      • 如果一个纯函数引用的数据在不同次调用中可能改变，在不同的时候用同样的参数调用这个函数可能导致不同的结果，那就与纯函数的定义相反
      • 注意，如果函数含有类似局部静态变量、全局变量，就是有状态的，就不是一个纯函数

  2. 一个判断式类是一个仿函数类，它的operator()函数是一个判断式，也就是，它的operator()返回true或false，或其他可以隐式转换到true或false的东西

  3. 算法可能拷贝仿函数，在使用前暂时保存它们，而且有些算法实现利用了这个自由，所以判断式函数必须是纯函数

二、分析

  1. 问题

    • 一个带来问题的仿函数实现

      // 第三次被调用的时候。其他时候它返回假
class BadPredicate: public unary_function<Widget, bool> { 
    public:
        BadPredicate(): timesCalled(0) {} // 把timesCalled初始化为0
        bool operator()(const Widget&)
        {
            return ++timesCalled == 3;
        }
    private:
        size_t timesCalled;
};

//从一个vector<Widget>中除去第三个Widget：
vector<Widget> vw; 
...
vw.erase(remove_if(vw.begin(), Widget; vw.end(), BadPredicate()), vw.end());
      • 不仅会从vw中除去第三个元素，它也会除去第六个

  2. remove_if的实现

    template <typename FwdIterator, typename Predicate>
FwdIterator remove_if(FwdIterator begin, FwdIterator end, Predicate p)
{
    begin = find_if(begin, end, p);
    if (begin == end) 
        return begin;
    else 
    {
        FwdIterator next = begin;
        return remove_copy_if(++next, end. begin, p);
    }
}
    • 意判断式p先传给find_if，被调用了3次，删除第三个元素，然后传给remove_copy_if时，由于是个拷贝，新的BadPredicate的timesCalled成员初始化为0，然后在remove_copy_if中又被调用了3次，删除了第六个元素

三、使用纯函数作判别式

  1. 把operator()函数声明为const

    class BadPredicate:public unary_function<Widget, bool> {
    public:
        bool operator()(const Widget&) const
        {
            return ++timesCalled == 3; // 错误！在const成员函数中不能改变局部数据
        } 
};
    • const成员函数可以访问multable数据成员、非const局部静态对象、非const类静态对象、名字空间域的非const对象和非const全局对象，所以还不够好

  2. 小心局部静态变量

    bool anotherBadPredicate(const Widget&, const Widget&)
{
    static int timesCalled = 0;
    return ++timesCalled == 3; // Bomb! 判断式应该是纯函数，纯函数没有状态
}

  3. 总之，应该使用纯函数做判断式
