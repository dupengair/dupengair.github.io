二十、协助完成“返回值优化(RVO)”

May 28, 2016 | 阅读笔记MoreEffectiveC++ | Hits
文章目录
  1. 1. 一、效率
  2. 2. 二、无优化
  3. 3. 三、RVO优化
  4. 4. 四、总结

一、效率

  • 从效率的眼光来看，不应该在乎函数返回了一个对象，应该在乎的是返回对象的成本。需要做的，是努力找出某种方法以降低被返回的对象的成本，而不是想尽办法消除对象本身。

二、无优化

  • 没有优化的实现方式

    const Rational operator*(const Rational& lhs, const Ratinal& rhs)
{
    Rational result(lhs.numerator() * rhs.numerator(), 
            lhs.demoninator() * rhs.demoninator());
    return result;
}

三、RVO优化

  1. 以constructor arguments取代对象

    const Rational operator*(const Rational& lhs, const Ratinal& rhs)
{
    return result(lhs.numerator() * rhs.numerator(), 
            lhs.demoninator() * rhs.demoninator());
}    

Ratinal a = 10;
Ratinal b(1, 2)
Ratinal c = a * b;

  2. C++允许编译器将临时对象优化，使他们不存在

  3. 编译器得以消除“operator*内的临时对象”及“被operator*返回的临时对象”。它们可以将return表达式所定义的对象构造于c的内存内。
  4. 此时，构造临时对象的成本为0，取而代之的是只需付出一个constructor的代价，因为c是一个命名对象，而命名对象时不能被消除的

四、总结

  1. 这个特殊的优化行为——利用函数的return点消除一个局部临时对象（并可能用函数调用端的对象取代），是普遍实现出来的——return value optimization
  2. 经过实际测试，编译器已经能在大部分情况下优化，所以不用刻意这么做。
