一、基于终端的远程访问（telnet、rlogin、X-Window）

  1. telnet
    • 端口23
    • 客户在服务器上输入的每个字符作为一个单独的数据包PUSH到客户端，以使客户感觉是在服务器上直接操作
    • 目的在于连接服务器与客户端，依赖于服务器验证，不直接支持用户或主机验证
    • 流程：
      • 客户端打开连接
      • 参数协商
      • 服务器验证程序启动，客户端提示登录信息
      • 完成用户验证，开始应用通信交互
  2. rlogin
  3. X-Windows
  4. 漏洞与攻击
    • 基于头部的攻击
      • telnet和rlogin没有头部
      • X-Windows易受缓冲区溢出
    • 基于协议的攻击
      • telnet和rlogin只是基于TCP的直接连接
      • X-Windows事件驱动，也没有太多漏洞
    • 基于验证的攻击
      • telnet依赖远程计算机的验证机制来验证用户名密码
      • rlogin使用IP地址和用户名验证远程用户的身份，客户端提供了身份作为协议交换的一部分，使服务器容易受到欺骗，攻击者可以伪装成一个已经受信任的客户端
      • X-Windows客户端的IP地址验证，没有基于用户的验证，使得攻击者可以从客户端获得对服务器主机的访问
    • 基于流量的攻击
      • 都是明文，容易被嗅探

二、文件传输协议

  1. 文件传输协议 FTP
    • 支持用户验证并提供异种计算机间有限的数据转换
    • 提供通用命令结构支持目录和文件管理
    • 使用简单的ASCII命令集进行交互
    • 命令交互独立连接，端口21，每一条数据传输打开新的连接
    • 支持匿名FTP，为阻止命名用户更改文件，服务器应该设定目录为只读
  2. 轻量级文件传输协议 TFTP
    • 基于UDP，无验证

  3. 远程服务制协议RCP

    • rlogin的一部分，不支持验证

  4. 漏洞与攻击

    • 基于头部的攻击
      • FTP和SCP易受到，TFTP头部简单，无效时协议产生错误
    • 基于协议的攻击
      • TFTP协议简单，出错时终止传输
      • RCP只是使用TCP流进行数据传输
      • FTP协议复杂，易受到攻击，主要集中与使用数据连接来绕过防火墙
    • 基于验证的攻击
      • 匿名FTP可被扫描，如果目录可写就悲剧了
    • 基于流量的攻击
      • 都是明文，易于嗅探
      • FTP易受DoS攻击

三、对等网络

  1. 对等网络
    • 允许用户彼此相连传输数据而无须验证，目的是方便文件传输
    • 集中式：
      • 使用一个中心服务器进行检索，数据文件存储在用户计算机上
      • 中心索引服务也可以构成对等网络，以负载均衡
      • Napster和KaZaA
    • 分布式：
      • 每个计算机都是网络的一部分，叶节点是用户，超级节点与大量叶节点相连
      • BT

  2. 漏洞与攻击

    • 基于头部和协议
      • 集中式易受对中心索引服务器的攻击
    • 基于验证的攻击
      • 在匿名的基础上进行设计，内容易导致问题
    • 基于流量的攻击
      • 鼓励大文件传输并且自身产生大量数据包，所以易受到攻击

  3. 对策

    • 检查TCP载荷，以确定TCP连接是否被用于传输对等流量

四、常用对策

  1. 加密远程访问
  2. 安全外壳协议 SSH
    • 支持机器验证和流量加密
    • 使用公钥和对称加密
      • 公钥用于验证服务端合法性
      • 对称秘钥用于加密流量
    • 流程：
      • 版本协商
      • 功能协商
      • 公钥验证
      • 会话秘钥协商
      • 用户验证
      • 加密数据交换
    • 弱点是中间人攻击，攻击者伪装成服务端使客户端相信

  4. 安全文件传输(SFTP、FTPS、HTTPS)
