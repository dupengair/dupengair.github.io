在社会生活中，公共管理是政府的工作，如垃圾回收、公共资源的管理等工作，我们小P民只需要按照政府的规定和某种社会契约好好生活就行。

编程也是一样，不该你做的，就交给编译器吧，我们是小P民，编译器是我们的政府。

一、资源管理

仙人曰：所谓资源，一旦用了，将来必须还给系统。

如果社会资源被不断占用而得不到补偿，等待我们的就是经济衰败、社会崩溃，如次贷危机。

我们占有的资源不仅仅是程序中的一个对象、一个变量（个人财产），还有整个内存（社会资源）。

二、手动管理资源

如：
void f()
{
    invest* pInv = createInvest();
    ……
    delete pInv;            //手动管理资源
}
  • 潜在的无法delete对象的因素：
    • …区域内有过早的return语句
    • 当delete位于某个循环内，循环由于continue或者goto语言过早退出时
    • …预期内抛出异常
      因此，不能依赖于手动执行的管理资源语句总是会被执行

三、基于对象的资源管理

  1. 建立在C++构造函数、析构函数、拷贝构造函数基础上进行对象管理。
  2. 将资源放进对象内，当控制流离开作用域时析构函数自动释放资源，依赖C++“析构函数自动调用机制”，让C++的政府——编译器来管理资源。

四、操作方法

  1. 获得资源后立即放进管理对象，即RAII，Resource Acquisition Is Initialization。
  2. 管理对象的析构函数确保资源释放，这个一般不用担心，由智能指针（政府的环卫工）帮我们完成。

五、STL的auto_ptr

如：
void f()
{    
    std::auto_ptr<invest> pInv(createInvest()); //由auto_ptr管理对象资源；
    ……;
}            //退出作用域时自动析构

  1. auto_ptr被销毁时自动删除指向对象，所以不能多个auto_ptr指向同一个对象，因此，通过拷贝构造函数或者赋值操作符复制他们，原来的指针会变成NULL，资源的唯一所有权在复制所得的指针。

    如：
std::auto_ptr<invest> pInv2(pInv);    // pInv=NULL，pInv2指向资源

  2. 因为auto_ptr没有正常的copy行为，所以无法在STL容器中使用

六、引用计数型智能指针RCSP reference-counting smart ptr

tr1::shared_ptr

七、对C++动态分配数组设计的指针

  1. auto_ptr和shared_ptr的析构函数只做delete动作，而不是delete[]，所以它们不能用于动态分配的数组。
  2. vector和 string几乎总是可以取代C++动态分配的数组。
  3. 如果必要，可以考虑boost的 boost::scoped_array 和 boost::shared_array 类，它们专为此设计。
