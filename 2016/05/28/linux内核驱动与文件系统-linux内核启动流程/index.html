内核启动初始化汇编代码段

<<<<<<<<<<<<<<< 内核启动初始化汇编代码段流程: >>>>>>>>>>>>>>

内核启动流程的程序段定义在\arch\arm\boot\compressed\vmlinux.lds中：

.text : {
    _start = .;
    *(.start)
    *(.text)
    *(.text.*)
    *(.fixup)
    *(.gnu.warning)
    *(.rodata)
    *(.rodata.*)
    *(.glue_7)
    *(.glue_7t)
    *(.piggydata)
    . = ALIGN(4);
  }

1、设置内核映像的入口地址
   位置：\arch\arm\boot\compressed\head.S
   入口： Line 134 - 157 -> .start段, 取得内核magic number和映像地址


2、解压缩内核代码
   位置：\arch\arm\boot\compressed\head.S
   入口：Line 263 - 276 -> 检测解压缩地址，调用decompress_kernel()解压缩内核
   位置：\arch\arm\boot\compressed\head.S
   入口：Line 307 - 310 -> 检测是否会覆盖当前运行地址
   位置：\arch\arm\boot\compressed\Misc.c
   入口：Line 345 - 361 -> 调用gnuzip()解压缩内核
   位置：\arch\arm\boot\compressed\head.S
   入口：Line 287 - 299
      Line 511 - 530    -> 存放解压后的内核

3、进入内核代码、初始化处理器，进入C代码
   位置：\arch\arm\kernel\head.S
   入口：Line 72 - 94  -> 初始化处理器模式，调用下述4个子程序，进入C代码
   位置：\arch\arm\kernel\head-common.S
   入口：Line 146 - 216  -> 检测处理器类型
   位置：\arch\arm\mm\proc-arm926.S
   入口：Line 467 - 489  -> 初始化arm926处理器信息
   位置：\arch\arm\kernel\head.S
   入口：Line 147 - 189 -> 打开MMU
   位置：\arch\arm\kernel\head-common.S
   入口：Line 14 - 55 -> 进入C代码start_kernel（main.c）

内核启动init进程代码

<<<<<<<<<<<<<<< 内核启动init进程代码流程: >>>>>>>>>>>>>>

1、start_kernel()：CPU初始化完成，调用rest_init()
   位置：\init\main.c
   入口： Line 461 - 601 -> start_kernel(), 初始化完成CPU后调用rest_init()创建内核init进程

2、rest_init()：调用kernel_thread()创建init进程
   位置：\init\main.c
   入口： Line 394 - 411 -> rest_init(), 调用kernel_thread()创建init进程，建立内存管理策略，cpu进入空闲状态

3、kernel_thread()：调用init()创建init进程
   位置：\arch\arm\kernel\Process.c
   入口： Line 430 - 447 -> kernel_thread(), do_fork()调用init()创建init进程


4、init()：启动init进程，初始化系统，执行启动脚本
   位置：\init\main.c
   入口： Line 707 - 846 -> init(), 调用populate_rootfs()     -> 初始化文件系统映像
                    do_basic_setup()      -> 设置各种驱动，
                                        prepare_namespace()   -> 挂载根文件系统到VFS，                                                                                   free_initmem()        -> 释放启动内存，
                    run_init_process("")  -> 执行启动脚本，
                                        启动init进程

5.1、do_basic_setup()：注册系统设备、设置驱动函数
   位置：\init\main.c
   入口： Line 684 - 672 -> do_basic_setup(), 调用driver_init()注册系统设备，调用do_initcalls()设置各种驱动

5.1.1、driver_init()：注册系统设备
   位置：\drivers\base\init.c
   入口： Line 23 - 40 -> driver_init(), 调用地xxx_init()函数，注册系统设备

5.1.2、do_initcalls()：设置各种驱动
   位置：\init\main.c
   入口： Line 616 - 663 -> do_initcalls(), 调用地址位于__initcall_start~end段的初始化函数，其地址定义在\arch\arm\kernel\vmlinux.lds.S链接脚本.init段中__initcall_start~end部分（Line 47 - 55）：
    __initcall_start = .;
        *(.initcall1.init)
        *(.initcall2.init)
        *(.initcall3.init)
        *(.initcall4.init)
        *(.initcall5.init)
        *(.initcall6.init)
        *(.initcall7.init)
    __initcall_end = .;


5.2、prepare_namespace()：挂载根文件系统到虚拟文件系统
   位置：\init\Do_mounts.c
   入口： Line 396 - 432 -> prepare_namespace(), 挂载根文件系统到VFS的过程

5.3、free_initmem()：释放启动内存
   位置：\arch\arm\mm\init.c
   入口： Line 470 - 477 -> free_initmem(), 释放启动内存

挂载文件系统

<<<<<<<<<<<<<<< 挂载文件系统的过程: >>>>>>>>>>>>>>

1、临时挂载根文件系统
   位置：\init\main.c
   入口： Line 580 -> start_kernel() 中调用 vfs_caches_init(num_physpages) -> mnt_int()
->位置：\init\namespace.c
   入口： Line 580 -> mnt_int(num_physpages)中调用init_rootfs()注册根文件系统，调用init_mount_tree()挂载根文件系统

1.1、注册根文件系统
   位置：\fs\ramfs\inode.c
   入口： Line 225 -> init_rootfs() 中调用 register_filesystem(&rootfs_fs_type)注册根文件系统

1.2、挂载文件系统
   位置：\fs\namespace.c
   入口： Line 1790 -> init_mount_tree() 中调用do_kern_mount()临时挂载根文件系统，然后分配名字空间，最后设置当前目录为根目录，此挂载具体的文件系统

2、检测是否是initramfs或initrd格式
   位置：\init\main.c
   入口： Line 740 -> ini() 中调用 populate_rootfs()检测是否是initramfs，或者initrd格式。前者直接放到根目录，如果是initrd则放到 /initrd,image，其它格式的根文件系统一律不作处理.然后在Line809 free_initmem()中释放initrd映像文件所占内存。
