十二、抛出异常与函数调用、参数传递之间的差异

May 28, 2016 | 阅读笔记MoreEffectiveC++ | Hits
文章目录
  1. 1. 一、控制权问题
  2. 2. 二、复制行为
  3. 3. 三、异常捕捉
  4. 4. 四、异常抛出与捕捉——类型匹配
  5. 5. 五、总结，异常传递与虚函数调用的区别
class Widget {……};

void f1(Widget w);
void f1(Widget& w);
void f1(const Widget& w);
void f1(Widget* w);
void f1(const Widget* w);

catch(Widget w);
catch(Widget& w);
catch(const Widget& w);
catch(Widget* w);
catch(const Widget* w);
  • 咋看之下，异常处理和函数调用非常类似，都可以以by value, by reference, by pointer的方式传递，其实形似而神远

一、控制权问题

  1. 对于函数调用，控制权最终会回到调用端，除非函数失败无法返回
  2. 对于抛出异常，控制权不会回到抛出端

二、复制行为

  1. 无论被捕捉的异常是以by value或by pointer 的方式传递，都会发生被抛出对象的复制（pointer不会复制对象本身，复制的是指针），区别只是被复制了几次和所复制的内容。catch 所接收到的总是一个副本，local对象总是在拷贝后被destructor析构掉。这就是C++特别声明，抛出异常对象时，总是发生copy行为的原因
  2. exception 必然会发生复制行为也说了一个事实：异常总是比函数调用慢

  3. 由于复制行为由对象的copy constructor执行，复制动作总是作用于对象的静态类型，而非动态类型（指throw）

    class SpecialWidget: public Widget {……};

void passAndThrow ()
{
    SpecialWidget sw;
    Widget& rw = sw;        // rw代表一个SpecialWidget
    throw rw;            // 抛出类型总为Widget
}

  4. 不要在catch的throw子句中抛出特定类型，除非需要

    catch (Widget& w)
{
    ...
    throw;        // 继续传播异常
}

catch (Widget& w)
{
    ...
    throw w;    // 抛出异常w的一个副本
}

    • 两个catch子句乍看之下非常相似，唯一的差异就是前者重新抛出当前的exception，后者抛出当前exception的副本。两者的差异在于：

      • 前者会重新抛出当前的异常，不论是什么类型，如SpecialWidget
      • 后者抛出一个新异常，类型总是Widget

    • 所以应尽量按照前者的方式重新抛出异常，不要改变原来的异常

三、异常捕捉

  1. 可以简单的用by reference的方式捕捉异常，不需要以by reference-to-const的方式，但函数调用过程中将一个临时对象传递给一个non-const reference 参数是不允许的，对exception则合法

  2. 如果以by value 的方式传递exception，需要付出构造“被抛出物”的“两个副本”的代价，一个用于构造“任何exception都会产生的临时对象”上，另一个构造“将临时对象复制到w”

    catch (Widget w) ...

  3. 如果以by reference 的方式传递exception，只需要付出构造“被抛出物”的“一个副本”的代价，用于构造“任何exception都会产生的临时对象上

    catch (Widget& w) ...
catch (const Widget& w) ...

  4. 千万不要抛出一个指向局部对象的指针，因为该局部对象会在控制权离开原函数时被销毁，而这个指针则被作为异常对象复制，因此catch子句会获得一个指向“已被销毁的局部”对象的指针，这是需要设计时避免发生的

四、异常抛出与捕捉——类型匹配

  1. 一般而言、异常捕捉时不进行类型转换，仅有两种转换发生：

    • 继承关系中的类型转换，即多态。针对base 的exception可以捕捉derived 类型

    • 可以从一个有类型的pointer 转换为一个void的pointer

      catch (const void*)

  2. catch子句总是以出现顺序做匹配尝试

    • 虚函数采用best fit的策略，exception采用first fit的策略
    • 捕捉异常时要先捕捉derived，再捕捉base，先抓典型

五、总结，异常传递与虚函数调用的区别

  1. exception object总是被复制，如果以by value形式捕捉，会被复制两次。而传递给参数的对象不一定被复制。
  2. 异常捕捉时进行的类型转换非常有限，仅支持多态和转换到空指针类型
  3. catch子句总是以出现顺序做匹配尝试，虚函数则采用最佳匹配
