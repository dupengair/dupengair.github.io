四十四、将与参数无关的代码抽离templates

May 28, 2016 | 阅读笔记EffectiveC++ | Hits
文章目录

  2. 2. 二、改进：以函数参数替换template参数
  3. 3. 三、完善：添加额外参数保存矩阵数据
  4. 4. 四、讨论：最佳代码
  5. 5. 五、模板类型参数带来的膨胀
  6. 6. 六、总结

使用template可能会导致代码膨胀（code bloat）。在non-template中，代码重复是明确的，在template代码中，重复是隐晦的，当实例化时才可能发生。

一、实例：由template参数带来的代码重复

template<typename T, std::size_t n>        //支持n阶矩阵的template，n是一个非类型参数
class SquareMatrix {
public:
    ……
    void invert();                        //矩阵求逆操作
};

考虑这些实例化：
SquareMatrix<double, 5> sm1;            //实例化
sm1.invert();                    

SquareMatrix<double, 10> sm2;            //再次实例化
sm2.invert();
  • 即使是非类型template参数，也会导致实例化，在上面的应用中，很明显的两次实例化的类代码膨胀。

二、改进：以函数参数替换template参数

template<typename T>                    //支持n阶矩阵的template非类型参数由函数参数取代
class SquareMatrixBase {
public:
    ……
    void invert(std::size_t n);            //以给定的矩阵尺寸进行求逆
};

再给模板派生一个子类：
template<typename T, std::size_t n>        //这是一个实现继承，不是is-a
class SquareMatrix : private SquareMatrixBase<T> {        
private:
    using SquareMatrixBase<T>::invert;    //使base class中的名称可见，条款33、43
public:
    ……
    void invert() {                        //inline 调用base class的invert
        this->invert(n); 
    }    
};
  • 这样，在以上面例子的形式实例化派生类SquareMatrix时，如果只是非类型参数n导致的实例化，只要类型T不变，基类只会被类型T实例化出一个类型, 因为只有类型T会导致基类的实例化。也就是说，对应给定类型T的派生类，无论n导致了多少个实例化，所有的派生类都共享唯一一个由类型实参T决定的基类SquareMatrixBase，因此共享这个基类中的invert方法，也就避免了代码膨胀。

三、完善：添加额外参数保存矩阵数据

  1. 现在为类添加保存矩阵数据的参数，使派生类能够和基类通讯。

    template<typename T>                //由矩阵参数初始化基类对象
class SquareMatrixBase {
public:
    SquareMatrixBase(std::size_t n, T* pMem)
        : size(n), pData(pMem) {}    //以阶数n和矩阵数据pData来初始化
    void setDataPtr(T* ptr);        //设置矩阵数据pData
    ……
private:
    std::size_t size;                //矩阵大小
    T* pData;                        //指向矩阵数据的pImp指针

};

  2. 在派生类中为矩阵数据分配内存：

    template<typename T, std::size_t n>
class SquareMatrix : private SquareMatrixBase<T> {
public:
    SquareMatrix() : SquareMatrixBase<T>(n, 0), pData(new T[n*n])
    {
        this->setDataPtr(pData.get());    //将矩阵数据传递给base class
    }
    ……
private:
    boost::scoped_array<T> pData;
};
    • 通过避免在定义时使用矩阵数据对象，可以缩小对象体积；

四、讨论：最佳代码

  1. 改进前的实例：这个导致代码膨胀的版本，可能生成比改进版本更佳的代码。原因是非类型参数是个编译器常量，“因为编译期常量的广传达到最优化，包括把它们折进被生成指令中成为直接操作数”，这在改进版中是无法办到的。
  2. 从另一个角度看，相同类型的不同大小的派生类对象只实例化单一版本的基类，可以减少执行文件的大小，因此降低程序的working set（指在一个虚拟内存环境下运行的进程，所使用的那一组内存页），并强化指令cache内的引用集中化（locality of reference）。这些都可以让程序执行更快速。
  3. 哪一个影响更明显，只有实际测试才知道。
  4. 从对象大小的角度看，改进版将模板参数转换成类成员，会使对象体积增大，并且由于这个参数需要由派生类传递到基类中，增加类的复杂度。

五、模板类型参数带来的膨胀

  1. 许多平台上int和long有相同的二进制表述，所以类型vector<int>和vector<long>的成员函数可能完全相同，某些链接器会合并完全相同的函数实现代码。
  2. 大多数平台上，所有的指针类型都有相同的二进制表述。凡template持有指针者，如果实现某些操作强类型指针（Strong typed pointers，即 T*）的成员函数，应让它们调用一个操作无类型指针（untyped pointers，即 void*）的函数来完成实际操作。

六、总结

  1. template生成多个class和多个函数，所以任何template代码都不该与某个造成膨胀的template参数产生关系。
  2. 对非类型模板参数引起的代码膨胀，可以用函数参数或者类成员变量代替，以消除膨胀。
  3. 对类型模板参数引起的代码膨胀，可以让带有完全二进制表述（binary representation）的实例化类型（instantiation type）共享其实现，以消除膨胀。
