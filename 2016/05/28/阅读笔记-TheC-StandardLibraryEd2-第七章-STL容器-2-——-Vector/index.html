一、Vector

  1. vector是一个dynamic array

  2. 第二个参数定义了所使用的内存模型，默认是C++标准库提供的allocator

    namespace std {
    template<typename T, typename Allocator = allocator<T>>
    class vector;
}

二、能力

  1. 是有序集合
  2. 支持随机访问，可以在常量时间内访问任意元素
  3. 提供随机访问迭代器，所以适用于任何STL算法
  4. 在末端添加或删除元素效率高，其它地方差，因为要搬运元素

  5. 大小和容量

    • vector性能高的原因之一，在于分配更大的内存
    • capacity()方法返回当前内存下可容纳的元素量，一旦超过就要重新分配内存，此时所有的reference、Iterator和pointer都失效

    • 可以使用reserve()保留适当容量，避免重新分配，或者初始化时传入数值实参作为vector的初始大小

      std::vector<int> v;
v.reserve(80);
std::vector<int> v(80);        // 等价于上面的操作

    • vector的容量不会缩减，即使调用reserve()，如果指定容量小于当前值，对vector不会有任何操作，这点与string不同。因此，对vector来说，安插元素可能导致reference、Iterator和pointer失效，删除元素却不会

    • C++11引入方法shrink_to_fit()缩减容器大小到元素个数，但不具强制力，只是为优化提高可能性，注意所有的reference、Iterator和pointer失效

      v.shrink_to_fit()

    • 交换两个vector时，容量也会互换，因此可以间接缩减容器大小，注意所有的reference、Iterator和pointer失效

      std::vector<int> tmp(v);    // 以容器v的元素创建新容器tmp，大小和v的元素个数相等
v.swap(tmp);                // v的容量与tmp交换
std::vector<T>(v).swap(v);    // 等价于以上两条

三、操作

  1. 构造时可以指定大小或初值，不指定初值时通过default构造初始化，对于基础类型，初始值默认为0
  2. 与array相同，访问元素时只有at()会执行范围检查，索引越界时会抛出异常out_of_range()。对一个空vector调用[]、front()、end()方法会引起不明确行为
  3. vector提供随机访问迭代器，所以理论上可以适用于所有的STL算法
  4. 引起vector迭代器失效的两种情况：
    • 在较小索引上安插或移除元素
    • 由于容量变化引起的内存重新分配
  5. 效率较高的安插或移除操作
    • 在容器尾部
    • 容器够大
    • 安插多个元素时一次性操作

  6. Vector没有提供方法直接移除与某值相等的元素，需要借助于算法

    v.erase(remove(v.begin(), v.end(), val), v.end());

  7. 移除与某值相等的第一个元素，也要借助于算法

    std::vector<int>::iterator pos = find(v.begin(), v.end(), val);
if(pos != v.end())
    v.erase(pos);

四、与C-style array

  1. C++标准保证vector内的所有元素一定位于连续且相邻的内存，使得对于vector a的任何游戏索引i，以下结果必为true

    &a[i] == &a[0] + i;

  2. 这使得在任何使用C-style array的地方，都可以使用vector

  3. 直接访问vector元素时，&a[0]与data()方法效果一样，使用data()时注意vector大小

    std::vector<char, 32> a;
strcpy(&a[0], "hello");
strcpy(a.data(), "world");

  4. 不能以迭代器体现首个元素地址，迭代器的类型取决于实现，并不是一个pointer，其行为不明确

    printf("%s\n", a.begin());    // ERROR
printf("%s\n", a.data());    // OK

五、异常处理

  1. 与array一致，at()是唯一C++标准认可抛出异常的方法
  2. 异常保证（略）

六、vector<bool>

  1. C++标准库针对bool的特化版本，使其空间得到优化，内核使用1bit存放一个元素。
  2. 由于C++最小可寻址单位为byte，所以特化版本对reference和iterator要特殊处理
  3. 因此vector<bool>::reference不是一个真正的lvalue，vector<bool>::iterator也不是真正的随机访问迭代器，某些template代码可能不适用
  4. vector<bool>也比一般的vector慢，因为内部要将元素转化为bit操作
  5. 提供某些特殊的bit操作，如flip()，可以对单一元素或所有元素调用flip操作
  6. 如果要用固定大小的bitfield，应该用bitset
