二十七、尽量少做转型动作

May 28, 2016 | 阅读笔记EffectiveC++ | Hits
文章目录

  2. 2. 二、C++风格

一、转型

  • C style:

    (T)expression

  • 函数 style:

    T(expression)

  • 两种形式没有差别，只是括号位置不同

二、C++风格

  1. const_cast：常量性转除
  2. dynamic_cast：安全向下转型，基->派
  3. reinterpret_cast：低级转型，效果取决于编译器，如pointer to int -> int
  4. static_cast：强迫隐式转换
  5. 优点
    • 容易辨识
    • 转型动作目标越窄化，编译器越能检测出错误

三、少用转型

  1. 任何一个类型转换，往往令编译器编译出运行期执行的代码

    如：
int x, y;
double d = static_cast<double>(x)/y;
    • int的底层表述不同于double，所以会产生一些代码

  2. 对类来说，单一对象可能拥有一个以上的地址

    如：
class Base { ... };
class Derived: public Base { ... };
Derived d;
Base* pb = &d;
    • 有个偏移量offset在运行期被施加于Derived 指针身上，以取得正确的Base 指针值，所以这时，“以Base 指向对象d”和“以Derived 指向对象d”时的地址并不同。

  3. 对象的布局方式和它们的地址计算方式，随编译器的不同而不同。通常应该避免作出“对象在C++中如何布局”的假设，更不应该以此执行任何转型动作。如将对象地址转型为char*然后进行指针算术，总是会导致无定义行为。那些“由于知道对象如何布局”而设计的转型动作，不一定在每个平台都行得通。

  4. 关于基类与派生类的转型

    class Window {
public:
    virtual void onResize() { ... }                    //基类中的onResize实现
    ...
};
class SpecialWindow: public Window {
public:
    virtural void onResize() {                        //派生类的实现
        static_cast<Window>(*this).onResize();        //将*this转型问window
        ...                                            //然后调用其onResize
    }
    ...
}
    • 这里派生类对*this进行转型后，调用的并不是当前对象上的onResize()函数，而是一个进过转型动作建立的一个”*this对象的base class”的临时副本上的onResize() !
    • 也就是说，如果Window::onResize()修改了对象内容，改动的是副本，当前对象其实没有被改动。

    • 解决的办法是去掉转型动作：

      class SpecialWindow: public Window {
public:
    virtural void onResize() {                
        Window::onResize();                //直接调用基类的实现
        ...                        
    }
    ...
}

  5. dynamic_cast的实现版本执行速度相当慢，深度继承或多重继承的成本更高，会过多耗用strcmp调用来比较class名称。

    • 避免的方法有两个：

      • 使用容器，并在其中直接存储执行derived class的（智能）指针

        如：
class Window { ... };
class SpecialWindow: public Window {
public:
    void blink();
    ...
};

typedef std::vector<std::tr1::shared_ptr<Window> > VPW;
VPW winPtrs;
...
for (VPW::iterator iter = winPtrs.begin(); iter != vinPtrs.end(); ++iter) {
if (SpecialWindow *psw = dynamic_cast<SpwcialWindow *>(iter->get()) )
    psw->blink();
}

应改为：
typedef std::vector<std::tr1::shared_ptr<SpecialWindow> > VPSW;
VPSW winPtrs;
...
for (VPSW::iterator iter = winPtrs.begin(); iter != vinPtrs.end(); ++iter) {        
    (*iter)->blink();
}

+ 在base class中提供virtual函数做派生类的事情

        如：
        class Window {
        public:
            virtual void blink() {}            //在base class中提供缺省实现
            ...
        };
        class SpecialWindow: public Window {
        public:
            virtual void blink() { ... }        //派生类中完成实际工作
            ...
        };

        typedef std::vector<std::tr1::shared_ptr<Window> > VPW;
        VPW winPtrs;
        ...
        for (VPW::iterator iter = winPtrs.begin(); iter != vinPtrs.end(); ++iter) {        
            (*iter)->blink();
        }
  1. 避免连串dynamic_cast，这样的代码又大又慢，且基础不稳，因为每次继承体系一有改变，所有这一类代码都必须再次检阅是否要做修改。

四、总结

1、尽量避免转型，追求效率就少用dynamic_cast
2、将转型隐藏在函数中，不要让客户将转型放入自己的代码内
3、用C++ style转型，少用C style
