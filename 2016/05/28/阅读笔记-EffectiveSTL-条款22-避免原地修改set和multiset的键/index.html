条款22 避免原地修改set和multiset的键

一、改变容器值

  1. 对于map和multimap

    • 为试图改变这些容器里的一个键值的程序将不能编译，因为map<K, V>或multimap<K, V>类型的对象中元素的类型是pair<const K, V>，因为键的类型const K

      map<int, string> m;
m.begin()->first = 10;  // 错误！map键不能改变
multimap<int, string> mm;
mm.begin()->first = 20; // 错误！multimap键也不能改变

  2. 对set和multiset

    • 对于set或multiset类型的对象来说，储存在容器里的元素类型是T，所以随便改

      class Employee {
public:
    ...
    const string& name() const;         // 获取雇员名
    void setName(const string& name);     // 设置雇员名
    const string& getTitle() const;     // 获取雇员头衔
    void setTitle(string& title);         // 设置雇员头衔
    int idNumber() const;                 // 获取雇员ID号
    ...
}

struct IDNumberLess：public binary_function<Employee, Employee, bool> { 
    bool operator()(const Employees lhs, const Employee& rhs) const
    {
        return lhs.idNumber() < rhs.idNumber();
    }
};

typedef set<Employee, IDNumberLess> EmpIDSet;
EmpIDSet se;

    • 如果STL的实现让用于set<T>::iterator的operator*返回一个常数T&。即set的迭代器解引用的结果是set元素的常量引用。在这样的实现下，将没有办法修改set或multiset的元素

      EmpIDSet::iterator i = se.find(selectedID);
if (i != se.end()){
    i->setTitle("Corporate Deity"); // 有些STL实现会拒绝这行
}

二、正确的做法

  1. 如果不关心移植性，想要改变set或multiset中元素的值，而且STL能实现。只是要确定不要改变元素的键部分，即会影响容器有序性的元素部分，就没有问题

  2. 如果在乎移植性，就认为set和multiset中的元素不能被修改，至少不能在没有映射的情况

    • 将set元素映射为引用，通过const_cast修改其值

      if (i != se.end()) { // 映射掉*i的常量性
    const_cast<Employee&>(*i).setTitle("Corporate Deity"); 
}
      • 通过映射到引用，我们避免了建立一个新对象。取而代之的是，映射的结果是一个现有对象的引用，i指向的对象

    • 错误的做法：

      if (i != se.end()){ // 把*i映射到了匿名对象上，无法修改
    static_cast<Employee>(*i).setTitle("Corporate Deity"); 
}

三、改变元素的通常做法

  1. 定位你想要改变的容器元素。如果你不确定最好的方法，条款45提供了关于怎样进行适当搜寻的指导。
  2. 拷贝一份要被修改的元素。对map或multimap而言，确定不要把副本的第一个元素声明为const。毕竟，你想要改变它！
  3. 修改副本，使它有你想要在容器里的值。
  4. 从容器里删除元素，通常通过调用erase（参见条款9）。

  5. 把新值插入容器。如果新元素在容器的排序顺序中的位置正好相同或相邻于删除的元素，使用insert的“提示”形式把插入的效率从对数时间改进到分摊的常数时间。使用你从第一步获得的迭代器作为提示。

    EmpIDSet se;             // 同前，se是一个以ID号排序的雇员set
Employee selectedID;     // 同前，selectedID是一个带有需要ID号的雇员        
...
EmpIDSet::iterator i = se.find(selectedID); // 第一步：找到要改变的元素
if (i!=se.end()) {
    Employee e(*i);                 // 第二步：拷贝这个元素
    se.erase(i++);                     // 第三步：删除这个元素；注意自增这个迭代器以保持它有效            
    e.setTitle("Corporate Deity");     // 第四步：修改这个副本
    se.insert(i, e);                 // 第五步：插入新值；提示它的位置和原先元素的一样
}
