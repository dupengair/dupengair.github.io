第七章 STL容器(4) —— List

一、List

采用doublely linked list管理元素
namespace std {
    template <typename T, typename Allocator = allocator<T>>
    class List;
}

二、能力

  1. List对象自身提供两个pointer指向第一个和最后一个元素，成为anchor，每个元素都有pointer指向前一个和后一个元素
  2. 不支持随机访问，所以不提供下标操作符，访问任意元素需从头遍历，只有头尾很快，所以只提供front和back访问元素，迭代器也只是双向迭代器
  3. 任意位置安插和删除元素很快，在常量时间完成，并且安插和删除元素不会造成pointer、reference和迭代器失效
  4. 对于异常，要么成功，要不无任何操作发生
  5. 不提供容量、分配空间的操作，因为元素有自己的内存
  6. list本身提供的移动和删除元素的操作，比STL提供的同名方法效率更高，因为考虑了采用指针操作
  7. 对于元素安插与删除，list提供deque的所有有功能，另外提供remove与remove_if以提高效率
  8. 提供自带的排序操作sort与reverse

  9. remove算法效率比STL更高，应尽可能采用自身的算法，

    std::list<int> l = {1, 2, 3};
l.remove(2);

  10. 如果只是删除第一个为某值的元素，需要采用通用的方法

    std::list<int>::iterator pos = find(l.begin(), l.end(), val);
if(pos != l.end())
    l.erase(pos);

  11. remove_if以一个函数或函数对象做为移除准则，移除使之返回true的所有元素

    l.remove_if([](int i){
        return i % 2 != 0;        // 移除list中的奇数
    });

  12. 不但可以在一个list内，也能在两个list间快速移动元素，因为都是基于指针的操作，提供splice/merge操作

三、异常处理

  1. 不成功就成仁——要么成功，要不无任何操作发生
