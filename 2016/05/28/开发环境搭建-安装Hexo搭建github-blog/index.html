2. 手动建立node的link

* 不能建立link是因为homebrew不在root用户组，没有写入系统文件夹的权限



* 查看brew的用户组：







        ➜  ~ ls -al `which brew`



        -rwxr-xr-x  1 dupengair  admin  791  8 13  2015 /usr/local/bin/brew







* 将其改为root用户组：







        ➜  ~ sudo chown root:admin `which brew`







* 执行brew建立链接命令：



        ➜  ~ sudo brew link --overwrite node







* 执行完成后改回原来的用户组：



        ➜  ~ sudo chown dupengair:admin `which brew`

3. 安装npm

* node -v 正常后，发现找不到npm命令，按网上的solution，重装node解决







        $ brew update



        $ brew uninstall node



        $ brew install node



        $ sudo brew postinstall node 

* 按同样方法建立链接后，终于得到了正确的输出：



        ➜  ~ sudo chown root:admin `which brew`

        ➜  ~ sudo brew link --overwrite node

        Linking /usr/local/Cellar/node/0.12.7... 6 symlinks created

        ➜  ~ sudo chown dupengair:admin `which brew`

        ➜  ~ node -v

        v0.12.7

        ➜  ~ npm -v

        2.12.1

6. 此时可以启动本地server进行验证

    hexo new "test.md"
    hexo g
    hexo s

* 生成的文件在./source/__post/下

7. 安装deploy插件

npm install hexo-deployer-git --save

8. 修改配置文件_config.xml

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: git@github.com:dupengair/dupengair.github.io.git
  branch: master

9. 部署到github上即可看到效果

hexo d
