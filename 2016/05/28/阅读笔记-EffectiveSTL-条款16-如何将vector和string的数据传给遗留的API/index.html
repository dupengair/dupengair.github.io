条款16 如何将vector和string的数据传给遗留的API

一、对于vector

  1. 内存连续，所以可以直接转换

    vector<int> v;
&v[0]；

  2. 要注意容器可能是空的

    void doSomething(const int* pInts, size_t nmInts);
if (!v.empty()) {
    doSomething(&v[0], v.size());
}

  3. 关于用v.begin()

    • begin返回指向vector内部的迭代器，而对于vector，其迭代器实际上是指针，通常是正确的，但并不总是如此，不该依赖于此
    • begin的返回类型是iterator，而不是一个指针，当需要一个指向vector内部数据的指针时绝不该使用begin。如果基于某些原因需要使用v.begin()，就应该以这样的形式&*v.begin()，从而产生和&v[0]相同的指针，只是书写上比较晦涩

二、对于string

  1. 不能使用指针进行转换
    • string中的数据不保证内存独立且连续
    • string的内部表示形式不保证以一个null字符结束

  2. 因此存在c_str()返回一个C风格的指针指向string的值

    void doSomething(const char *pString);
doSomething(s.c_str());
    • 字符串长度为0时返回一个指向null的指针
    • string对象不关心null字符，当做普通字符处理，转换为char*的C风格会认为字符串结束

三、灵活性

  1. vector和string的数据只能传给只读取而不修改它的API
  2. 对于vector，有更多一点点灵活性。如果将v传给一个修改其元素的C风格API的话，典型情况都是没问题，但被调用的函数绝不能试图改变vector中元素的个数
    • 注意如果对顺序有要求，则修改后不会保持顺序

  3. C风格API初始化vector

    • 利用vector和数组潜在的内存分布兼容性将存储vecotr的元素的空间传给API

      size_t fillArray(double *pArray, size_t arraySize);
vector<double> vd(maxNumDoubles);          // 建立一个vector，它的大小是maxNumDoubles
vd.resize(fillArray(&vd[0], vd.size())); // 让fillArray把数据写入vd，然后调整vd的大小为fillArray写入的元素个数

    • 只能工作于vector，因为只有vector承诺了与数组具有相同的潜在内存分布

  4. 如果要用于string，则需要先建立一个vector并初始化，然后把vector内容拷贝到string中

    size_t fillString(char *pArray, size_t arraySize);
vector<char> vc(maxNumChars);                          // 建立一个vector，它的大小是maxNumChars
size_t charsWritten = fillString(&vc[0], vc.size()); // 让fillString把数据写入vc
string s(vc.begin(), vc.begin()+charsWritten);          // 从vc通过范围构造函数拷贝数据到s

  5. 让C风格API把数据放入一个vector，然后拷到你实际想要的STL容器中的主意总是有效的，这也是用C风格API初始化所有容器的一般通行做法

    size_t fillArray(double *pArray, size_t arraySize); // 同上
vector<double> vd(maxNumDoubles);         // 一样同上
vd.resize(fillArray(&vd[0], vd.size()));
deque<double> d(vd.begin(), vd.end());     // 拷贝数据到deque
list<double> l(vd.begin(), vd.end());     // 拷贝数据到list
set<double> s(vd.begin(), vd.end());     // 拷贝数据到set

四、注意

  • vector<bool>除外，因为它们依赖于能够取得指向容器中包含的元素类型的指针，vector<bool>中不保存bool值
