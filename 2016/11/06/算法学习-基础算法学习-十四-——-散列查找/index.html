基础算法学习(十四) —— 散列查找

一、散列查找

  1. 散列技术

    • 建立存储位置与关键字之间的对应关系，存储在一块连续空间中，不需要比较就可获得记录的位置

      存储位置 = f(关键字)

  2. 查找

    • 散列技术既是存储方法，也是查找方法，是一种面向查找的存储结构
    • 适用于查找与给定值相等的记录，即关键字与记录间是一对一的关系
    • 不适用于一对多的关系，范围查找，以及进行排序
  3. 冲突
    • key1 != key2时，f(key1) == f(key2)
    • key1和key2称为f()的同义词

二、构造散列函数

  1. 原则
    • 计算简单
      • 效率要高，不能比其它查找和比较的时间长
    • 散列地址分布均匀
      • 有效利用存储空间
  2. 选择时应考虑
    • 计算时间
    • 关键字长度和分布
    • 散列表大小
    • 查找频率

  3. 直接定址

    • 取关键字的线性函数值作为散列地址前

      f(key) = a*key + b

    • 简单、均匀，没有冲突

    • 需要事先知道关键字分布，适合查找表较小且连续
  4. 数字分析
    • 抽取关键字的一部分计算散列位置
      • 反转、左右环移位、首尾叠加等
      • 如手机号
    • 适合处理关键字位数较大的情况，需要事先知道关键字分布均匀
  5. 平方取中
    • 先平方，再抽取中间几位
    • 适用于不知道关键字分布，位数不大的情况
  6. 折叠法
    • 将关键字分割成位数相等的几部分，进行叠加求和，取最后几位作为地址
    • 可以通过来回折叠保证地址均匀
    • 适用于不知道关键字分布，位数较多的情况

  7. 除留取余

    • 对关键字(可先折叠、平方取中后再)取模

      f(key) = key % p (p <= m)
      • 若散列表长m，通常p为小于等于表长m的最小质数
    • 最常用

  8. 随机数法

    • 选择一个随机函数

      f(key) = rand(key)

    • 适用于关键字长度不等

三、处理散列冲突

  1. 开放定址

    • 冲突发生时，寻找下一个空闲的散列表地址，只要表足够大就总能找到

      f'(key) = (f(key) + d) % m （d = 1, 2, 3 ...）

    • 是一种线性探测法

    • 会造成堆积，本来不是同义词，却要争夺同一个地址

    • 可以增加平方运算，防止关键字聚集，即二次探测

      f'(key) = (f(key) + d) % m （d = 1^2, 2^2, 3^2 ...）

    • 可以对d采用随机数，即随机探测

      f'(key) = (f(key) + d) % m （d = random()）
  2. 再散列法
    • 采用多个散列函数，每当冲突时就换一种
  3. 链地址法
    • 将关键字为同义词的记录用链表保存成同义词子表，散列表中只保存表头指针
    • 绝不会出现找不到地址
    • 查找时需要遍历链表，有一定性能损失
  4. 公共溢出区法
    • 为所有关键字建立公共溢出区
    • 冲突少的情况下，性能非常高

四、实现

  1. 定义

    #define SUCCESS      1
#define UNSUCCESS 0
#define HASHSIZE 12        // 散列表长
#define NULLKEY -32768

typedef struct
{
    int *elem;
    int count;
}THashTbl;        // 散列表

int m = 0;

  2. 初始化散列表

    bool InitHashTable(THashTbl *H)
{
    int i;
    m = HASHSIZE;
    H->count = m;
    H->elem=(int *)malloc(m*sizeof(int));
    if(!H->elem)
        return false;

    for(i=0; i<m; i++)
        H->elem[i] = NULLKEY;
    return true;
}

  3. 散列函数

    int Hash(int key)
{
    return key % m;
}

  4. 计算散列结果，存储到散列表中

    void InsertHash(THashTbl *H, int key)
{
    int addr = Hash(key);

    while (H->elem[addr] != NULLKEY)
        addr = (addr + 1) % m;        // 冲突时采用线性定址;

    H->elem[addr] = key;
}

  5. hash查找

    bool SearchHash(THashTbl *H, int key, int *addr)
{
    *addr = Hash(key);

    while( H->elem[*addr] != key) 
    {
        *addr = (*addr + 1) % m;
        if( H->elem[*addr] == NULLKEY || *addr == Hash(key))
            return UNSUCCESS;
    }

    return SUCCESS;
}

  6. 输出

    int a[HASHSIZE] = { 12, 67, 56, 16, 25, 37, 22, 29, 15, 47, 48, 34 };

THashTbl h;
if(InitHashTable(&h) == false)
    printf("init hashtable failed. \n");

for(int i=0; i<HASHSIZE; i++)
    InsertHash(&h, a[i]);

int addr = NULLKEY;
int ret = -1;

ret = SearchHash(&h, 37, &addr);
printf("search 37: %s, at: %d\n", ret?"SUCCESS":"FAILED", addr);
ret = SearchHash(&h, 27, &addr);
printf("search 27: %s, at: %d\n", ret?"SUCCESS":"FAILED", addr);

./test_hash 
search 37: SUCCESS, at: 2
search 27: FAILED, at: 3

五、总结

  1. 性能
    • 效率最高，复杂度O(1)
  2. 影响性能的因素
    • 散列函数是否均匀
    • 处理冲突的方法
    • 散列表的饱和程度
      • 越饱和，冲突可能性越大
      • 通常表空间比查找集大
