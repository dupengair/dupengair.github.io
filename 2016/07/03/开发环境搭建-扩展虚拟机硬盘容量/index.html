一、虚拟机管理工具中扩展容量

  1. vmware版本10.0以上
  2. 虚拟机设置->硬盘->高级工具->扩展->扩展容量到40G

二、查询分区

  1. 查询容量，总容量已扩展到40G，但分区没变

    sudo fdisk -l

三、新建分区

  1. 使用fdisk新建分区

    sudo fdisk /dev/sda
m->n->p->default->n->p->default->w

  1. 注意第一次建立的主分区很小，再np一次建第二个新分区多时候就会大了，最后的分区如下：

    command (m for help): p
Disk /dev/sda: 40 GiB, 42949672960 bytes, 83886080 sectors
Units: sectors of 1 * 512 = 512 bytes
Sector size (logical/physical): 512 bytes / 512 bytes
I/O size (minimum/optimal): 512 bytes / 512 bytes
Disklabel type: dos
Disk identifier: 0x99cca55b

Device     Boot    Start      End  Sectors  Size Id Type
/dev/sda1  *        2048 33554431 33552384   16G 83 Linux
/dev/sda2       33556478 41940991  8384514    4G  5 Extended
/dev/sda3       33554432 33556477     2046 1023K 83 Linux
/dev/sda4       41940992 83886079 41945088   20G 83 Linux
/dev/sda5       33556480 41940991  8384512    4G 82 Linux swap / Solaris

Partition table entries are not in disk order.

四、重启后格式化分区

  1. 格式化新分区

    sudo mkfs -t ext4 /dev/sda3
sudo mkfs -t ext4 /dev/sda4

  2. 手动挂载

    sudo mkdir /mnt/sda3
sudo mkdir /mnt/sda4
chmod -R 777 /mnt/sda*
sudo mount /dev/sda3 /mnt/sda3
sudo mount /dev/sda4 /mnt/sda4

五、配置启动自动挂载

  1. 查各分区uuid

    $ ls -al /dev/disk/by-uuid/
total 0
drwxr-xr-x 2 root root 160 Jul  3 18:59 .
drwxr-xr-x 6 root root 120 Jul  3 18:55 ..
lrwxrwxrwx 1 root root  10 Jul  3 18:55 0bc7ae73-c795-4537-8ccc-1e81ca23ae20 -> ../../sda1
lrwxrwxrwx 1 root root  10 Jul  3 18:59 0e69f98f-e9b1-4f79-8584-dc5765ac8824 -> ../../sda3
lrwxrwxrwx 1 root root   9 Jul  3 18:55 2015-10-21-17-39-25-00 -> ../../sr1
lrwxrwxrwx 1 root root   9 Jul  3 18:55 2016-03-13-16-24-32-00 -> ../../sr0
lrwxrwxrwx 1 root root  10 Jul  3 18:59 b6bdd617-daff-4357-bce9-645e83fba23f -> ../../sda4
lrwxrwxrwx 1 root root  10 Jul  3 18:55 cf85869d-fa3e-494b-9860-6ef7364570ea -> ../../sda5

  2. 修改/etc/fstab，以uuid的方式自动挂载

    almond@bogon:~$ cat /etc/fstab
# /etc/fstab: static file system information.
#
# Use 'blkid' to print the universally unique identifier for a
# device; this may be used with UUID= as a more robust way to name devices
# that works even if disks are added and removed. See fstab(5).
#
# <file system> <mount point>   <type>  <options>       <dump>  <pass>
# / was on /dev/sda1 during installation
UUID=0bc7ae73-c795-4537-8ccc-1e81ca23ae20 /               ext4    errors=remount-ro 0       1
UUID=b6bdd617-daff-4357-bce9-645e83fba23f /mnt/sda4       ext4    errors=remount-ro 0       1
# swap was on /dev/sda5 during installation
UUID=cf85869d-fa3e-494b-9860-6ef7364570ea none            swap    sw              0       0
/dev/fd0        /media/floppy0  auto    rw,user,noauto,exec,utf8 0       0
